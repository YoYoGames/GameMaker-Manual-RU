Показать виртуальную клавиатуру

Это действие можно использовать для отображения виртуальной клавиатуры на устройстве, на котором запущена игра. При вызове этого действия вам будет предложено указать одну из следующих констант для каждого из первых трех аргументов:

  • Тип клавиатуры: Эта константа используется для установки того, какой набор клавиш будет доступен на виртуальной клавиатуре. Доступными типами клавиатуры являются:
    Тип клавиатуры Описание
    kbv_type_default Тип клавиатуры по умолчанию для текущей системы.
    kbv_type_ascii Клавиатура только ASCII.
    kbv_type_url Обычная клавиатура, оптимизированная для ввода URL-адресов. Обычно имеет клавиши ".com" или другие доменные клавиши, а также клавиши "/" и ".".
    kbv_type_email Обычная клавиатура, оптимизированная для ввода электронной почты. Обычно содержит символы "@" и ".".
    kbv_type_numbers Клавиатура только для ввода цифр, обычно отображается как цифровая панель.
    kbv_type_phone Клавиатура телефонной панели. Обычно содержит только цифры и клавиши "*" и "#".
    kbv_type_phone_name Клавиатура, оптимизированная для ввода как телефонного номера, так и имени. Обычно похожа на клавиатуру ASCII, но с ограниченным выбором специальных символов.
  • Тип возврата: Эта константа используется для установки того, что будет отображаться на клавише возврата/действия виртуальной клавиатуры. Доступны следующие типы возврата:
    Тип возврата Описание
    kbv_returnkey_default Заголовок клавиши возврата по умолчанию для текущей системы.
    kbv_returnkey_go Устанавливает заголовок клавиши возврата на "Go".
    kbv_returnkey_google Устанавливает заголовок клавиши возврата на "Google", или на общий значок поиска в некоторых случаях.
    kbv_returnkey_join Устанавливает заголовок клавиши возврата на "Перейти".
    kbv_returnkey_next Устанавливает заголовок клавиши возврата на "Далее".
    kbv_returnkey_route Устанавливает заголовок клавиши возврата на "Маршрут".
    kbv_returnkey_search Устанавливает заголовок клавиши возврата на "Поиск", или на общий значок поиска в некоторых случаях.
    kbv_returnkey_send Устанавливает заголовок клавиши возврата на "Отправить".
    kbv_returnkey_yahoo Устанавливает заголовок клавиши возврата на "Yahoo", или на значок общего поиска в некоторых случаях.
    kbv_returnkey_done Устанавливает заголовок клавиши возврата на "Готово".
    kbv_returnkey_continue Устанавливает заголовок клавиши возврата на "Продолжить".
    kbv_returnkey_emergency Устанавливает заголовок клавиши возврата на "Экстренный вызов".
  • Тип автокапитализации: Эта константа используется для определения способа автокапитализации слов, набираемых с помощью виртуальной клавиатуры. Доступны следующие типы автокапитализации:
    Тип автокапитализации Описание
    kbv_autocapitalize_none Автокапитализация отключена.
    kbv_autocapitalize_words Слова будут автозаглавными.
    kbv_autocapitalize_sentences Предложения будут автозаглавными.
    kbv_autocapitalize_characters Все символы будут набраны заглавными буквами.

Последним аргументом является разрешение/запрет предикативного текста, и этот параметр должен иметь значение true, чтобы разрешить его, и false в противном случае, но обратите внимание, что то, что он разрешен, не означает, что он будет использоваться, так как это зависит от предпочтений пользователя устройства. В режиме предикативного текста виртуальная клавиатура не будет генерировать обычные события нажатия клавиш GameMaker. Вместо этого она будет обновлять только последний нажатый символ и переменные строки клавиатуры. Это связано с тем, что невозможности определить, произошло ли изменение во внутреннем текстовом поле, используемом для обнаружения нажатия клавиш, от фактической клавиши виртуальной клавиатуры или от текстового предложения. В таких случаях необходимо считывать ввод keyboard_string, а не чем считывать любой необработанный ввод клавиш.

Важно также отметить, что пользователь получит разные клавиатуры с разными возможностями в зависимости от ОС платформы, со следующими оговорками для использования по назначению:

  • UWP: Реализация UWP не поддерживает return_key_type, autocapitalization_type или predictive_text_enabled. Передача этих параметров в функцию не вызовет ошибок, они просто не будут иметь никакого влияние на клавиатуру.
  • Android / Amazon Fire: На платформе Android тип клавиатуры kbv_type_phone_name не поддерживается и вместо него отображается клавиатура ASCII, а тип клавиши возврата может быть только kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send или kbv_returnkey_done. Если на этой платформе используются другие типы клавиш возврата, система по умолчанию будет использовать заменяющие клавиши возврата, перечисленные ниже:
    Недоступная клавиша возврата Клавиша замены
    kbv_returnkey_google
    kbv_returnkey_yahoo    		 kbv_returnkey_search
    kbv_returnkey_join
    kbv_returnkey_route
    kbv_returnkey_emergency    		 kbv_returnkey_go
    kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV / FireTV: Пользовательские типы клавиш возврата не поддерживаются на клавиатурах ASCII - всегда будет отображаться клавиша возврата по умолчанию.
  • tvOS: предиктивный текст и автокапитализация не поддерживаются на tvOS, а высота клавиатуры, возвращаемая системными событиями и функцией keyboard_virtual_height() будет по умолчанию равна высоте экрана, поскольку она занимает весь экран и функции прямоугольного размера клавиатуры отключены на tvOS. Также обратите внимание, что события физической (Bluetooth) клавиатуры не будут транслироваться, пока виртуальная клавиатура не будет была открыта.

Вызов этой функции сгенерирует системное асинхронное событие, в котором карта async_load  DS будет заполнена следующими парами ключ/значение:

  • "event_type" - это будет "статус виртуальной клавиатуры", когда срабатывает от действий виртуальной клавиатуры.
  • "screen_height" - высота виртуальной клавиатуры (в пикселях). Это значение будет равно 0, если клавиатура невидима.
  • "keyboard_status" - текущее состояние клавиатуры, возвращается в виде одной из следующих строк:
    • "скрытие"
    • "скрытый"
    • "показывающий"
    • "видимый"

Синтаксис действия:

Аргументы:

Аргумент Описание
keyboard_type Определяет набор клавиш, доступный на виртуальной клавиатуре.
return_key_type Определяет, что будет отображаться на клавише возврата/действия виртуальной клавиатуры.
autocapitalization_type Определяет, как/если слова, набранные с помощью виртуальной клавиатуры, будут автокапитализированы.
predictive_text_enabled Установите значение true/false, чтобы включить/выключить предикативный ввод текста.

Пример:

Приведенный выше код блока действий проверяет наличие действия нажатия кнопки мыши, и если таковое обнаружено, то выполняется проверка, отображается ли виртуальная клавиатура ОС. Если нет, то она устанавливается на отображение - с использованием цифрового типа клавиатуры - а если уже отображается, то скрывается.
