Получить переменную экземпляра

С помощью этого действия вы можете получить любой из перечисленных встроенный переменных. Все экземпляры всех объектов имеют ряд переменных, уже созданных внутри них для хранения важной информации, такой как текущее положение x/y, скорость, направление, видимость и многое другое. Переменные экземпляра, значения которых вы можете получить, следующие:

  • Координата X - положение вдоль оси x в помещении.
  • Y Coordinate - Положение вдоль оси y в помещении.
  • Direction - Направление движения экземпляра (по умолчанию 0, где вправо - 0°, вверх - 90°, влево - 180° и вниз - 270°).
  • Скорость - скорость (в пикселях на игровой кадр), с которой движется экземпляр.
  • Горизонтальная скорость - горизонтальная скорость (в пикселях на игровой кадр), с которой движется экземпляр. Отрицательная скорость означает, что он движется влево, положительная скорость - вправо, по умолчанию - 0.
  • Вертикальная скорость - вертикальная скорость (в пикселях на игровой кадр), с которой движется экземпляр. Отрицательная скорость означает, что он движется вверх, положительная скорость - вниз, а значение по умолчанию равно 0.
  • Visible - Переменная переключения видимости (по умолчанию true, если в редакторе объектов не установлено иное, установите значение false, чтобы сделать невидимым). Обратите внимание, что невидимый экземпляр по-прежнему выполняет все свои события, кроме события Draw.
  • Persistent - Переменная переключения постоянства (по умолчанию false, если в редакторе объектов не установлено иное, установите значение true, чтобы сделать его постоянным). Это позволит экземпляру "сохраняться" из комнаты в комнату, т.е. экземпляр не будет очищен в конце комнаты, а будет перенесен в следующую комнату. Обратите внимание, что когда экземпляр персистентен он не будет вызывать события Create и Destroy, но будет вызывать события Clean Up, Room End и Room Start.
  • Layer - Уровень, к которому был назначен экземпляр.
  • Глубина - Значение глубины, присвоенное слою.
  • Sprite - Ресурс sprite, назначенный экземпляру.
  • Индекс изображения - индекс изображения назначенного sprite. Это значение устанавливает кадр sprite, назначенный данному экземпляру, и если значение больше, чем количество кадров в анимации, она будет "зацикливаться", так что sprite с 5 кадрами, если установить его на кадр 7, на самом деле покажет кадр 2.
  • Угол изображения - угол, под которым должен быть установлен экземпляр (по умолчанию 0, где вправо - 0°, вверх - 90°, влево - 180° и вниз 270°).
  • Альфа изображения - значение альфа для экземпляра (по умолчанию 1) от 0 до 1, где 0 - полностью прозрачный, а 1 - полностью непрозрачный.
  • Image Blend - Цвет, который будет смешиваться с экземпляром sprite (по умолчанию белый).
  • Скорость изображения - Скорость анимации для данного экземпляра.
  • In Sequence - Эта переменная возвращает true, если экземпляр в настоящее время включен в последовательность, и false в противном случае.
  • Горизонтальный масштаб - горизонтальный масштаб, в котором должен быть нарисован экземпляр sprite (это также влияет на ограничивающую рамку экземпляра и столкновения). рамки и коллизии). Масштаб 1 означает отсутствие масштабирования (1:1), меньшие значения уменьшают масштаб (0.5, например, уменьшает ширину sprite вдвое), большие значения увеличивают масштаб, а отрицательные значения зеркально отражают sprite и масштабируют его, за исключением случаев, когда используется значение -1 (в этом случае просто зеркально отражается). если только используемое значение не равно -1 (в этом случае sprite просто зеркально отображается по оси y без масштабирования).
  • Vertical Scale - вертикальный масштаб, в котором должен быть нарисован экземпляр sprite (это также влияет на ограничивающую рамку экземпляра и столкновения). рамки и коллизии). Масштаб 1 означает отсутствие масштабирования (1:1), меньшие значения уменьшают масштаб (например, 0.5 уменьшает высоту sprite в два раза), большие значения увеличивают масштаб, а отрицательные значения переворачивают sprite и масштабируют его, за исключением случаев, когда используется значение -1 (в этом случае просто переворачивается вдоль вертикали). если только используемое значение не равно -1 (в этом случае sprite просто переворачивается по оси y без масштабирования).
  • Mask - индекс маски для экземпляра. Значение по умолчанию -1 (это означает, что экземпляр будет использовать маску столкновений для назначенного в данный момент индекса sprite ), но если установить значение для любого другого ресурса sprite, то будет использоваться маска столкновений для этого экземпляра вместо sprite, назначенного для индекса sprite.
  • Friction - Трение, которое будет использоваться в каждом игровом кадре игры для замедления экземпляра. Это значение будет просто вычитаться из текущего скорости каждый игровой кадр (или прибавляться, если значение скорости отрицательное).
  • Гравитация - Гравитация будет прикладывать постоянную силу в направлении гравитации (см. ниже) экземпляра, влияя на его скорость (по умолчанию установлено значение 0). Обратите внимание, что гравитация является кумулятивной силой и будет ускорять объект, если вы решите не ограничивать конечную скорость.
  • Направление гравитации - можно использовать для задания направления движения, когда гравитация экземпляра больше 0 (по умолчанию 270). Обратите внимание, что направления в GameMaker рассчитываются как 0° - вправо, 90° - вверх, 180° - влево и 270° - вниз.
  • Score - Установить переменную score.
  • Lives - Установите переменную lives.
  • Здоровье - Установите переменную health.

Выбрав встроенную переменную для получения, вы затем устанавливаете целевую переменную для хранения возвращаемого значения. Если вы отметите эту переменную как временную, то она будет создана для вас в действии и будет доступна только до конца события или сценария действия. события или сценария действия. Вы также можете получить значения дополнительных переменных в одном и том же действии, нажав на значок плюс рядом с действием, выбрать другую переменную экземпляра и указать сопутствующую переменную для хранения возвращаемого значения.
