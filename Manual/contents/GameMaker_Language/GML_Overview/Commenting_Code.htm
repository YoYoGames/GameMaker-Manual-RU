Код комментирования

При совместной работе с другими людьми, при работе с большим проектом или даже просто для собственного ознакомления и отладки очень важно оставлять комментарии в коде событий и скриптах. В GameMaker есть различные механизмы, позволяющие оставлять заметки и комментарии к участкам кода и даже целым блокам кода, которые можно увидеть в списке событий объекта, который вы в данный момент редактируете. Таким образом, вы можете оставлять заметки для себя и своих коллег, объяснять особенно сложный кусок кода или даже просто оставлять напоминания о том, что что-то делает или что вы хотите, чтобы что-то делалось в будущем.

Чтобы оставить простой однострочный комментарий, первое, что вы можете сделать, это использовать // перед текстом. Например:

//initialize variables
sr = 10;
hp = 100;

Вы также можете оставлять многострочные комментарии, чтобы отдать должное, опустить целый раздел кода для отладки или даже оставить заметки о том, как вы хотите изменить код в будущем. Для этого вы можете использовать /* ... */ например:

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

Помимо комментариев вы также можете использовать специальные теги #region, чтобы отметить участок кода для сворачивания кода, и каждый участок может содержать комментарий. Это работает следующим образом: вы добавляете тег #region - вместе с любым текстом комментария - в верхнюю часть раздела кода, а затем добавляете соответствующий тег #endregion в нижнюю часть раздела, после чего вы можете "свернуть" код, то есть спрятать его, чтобы сократить количество строк кода, видимых в редакторе в любой момент времени:

Фактический код выглядит примерно так:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Наконец, стоит упомянуть, что при работе с событиями в редакторе объектов вы можете использовать специальный комментарий "description" в начале события, чтобы сказать, каково общее содержание события, и это будет показано в окне событий редактора объектов. Этот комментарий должен располагаться в самой первой строке редактора кода и оформляется следующим образом:

/// @description <This Is An Event Description>

Обратите внимание, что вы можете определить комментарий по умолчанию, который будет создаваться для всех событий, в настройках редактора объектов.

Наконец, если вы пишете скриптовые функции и хотите иметь некоторый контроль над тем, как они обрабатываются IDE (например, иметь автозаполнение или показывать параметры скрипта), то вам следует писать комментарии, используя нотацию JSDoc, как описано здесь.
