network_send_broadcast

С помощью этой функции вы можете транслировать данные из буфера локально на диапазон IP-адресов (более подробную информацию о буферах см. в разделе Справочник - Буферы). Диапазон ограничен диапазоном устройства, на котором работает сервер, например, если устройство имеет IP-адрес 92.168.11.130, то данные будут транслироваться в диапазоне 92.168.11.*. Функция возвращает количество байт отправленных данных или число меньше 0, если отправка не удалась.

Эта функция будет работать только при использовании с UDP - ваш сервер должен быть TCP, а ваш клиент должен иметь клиентский сокет UDP, созданный с помощью функции network_create_socket_ext() для того, чтобы получать любые широковещательные сообщения, отправляемые с сервера.

Эта функция не будет работать при использовании в проекте, работающем на HTML5-цели.

Синтаксис:

network_send_broadcast(socket, port, bufferid, size);

Аргумент Тип Описание
socket Идентификатор сетевого сокета The id of the socket to use.
port Реально The port that the server will use.
bufferid Идентификатор буфера The id of the buffer to get the data from.
size Реально The size (in bytes) of the data.

Возвращается:

Real

Пример:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Приведенный выше код записывает строку имени текущего сервера (хранится в "global.ServerName"), затем записывает ее в двоичный буфер с идентификатором "broadcast_buffer". Затем эти данные локально транслируются на ряд IP-адресов (IP-адрес устройства в настоящее время подразумевается как базовый диапазон вещания) на порт 6511.
