виртуальный_ключ_добавить

Эта функция позволяет привязать "прикосновения" к области экрана к событиям клавиатуры. Это означает, что если вы назначили область виртуальной клавише, все прикосновения к этой области будут вызывать событие клавиатуры, соответствующее клавише, которую вы назначили этой области. Каждой виртуальной клавише можно присвоить переменную, которая затем может быть использована в дальнейших функциях виртуальных клавиш для их отображения, скрытия и удаления. Эти клавиши назначаются для каждой комнаты и будут автоматически удаляться программой GameMaker при смене комнаты.

Фактическое положение виртуального ключа основано на положении экрана, а не комнаты, поэтому значения x/y являются абсолютными на экране. Это означает, что вам не нужно беспокоиться об использовании представлений или относительных координат комнаты, а можно просто определить ключи в событии Create объекта (вам нужно определить виртуальный ключ только один раз для каждой комнаты), затем нарисовать ключ sprites в событии Draw GUI Event, установив размер слоя GUI равным размеру экрана.

Синтаксис

virtual_key_add(x, y, w, h, keycode);

Аргумент Тип Описание
x Реально The x coordinate (left side) of the virtual key on the screen
y Реально The y coordinate (top side) of the virtual key on the screen
w Реально The width of the virtual key
h Реально The height of the virtual key
keycode Константа виртуального ключа (vk_*) Which keyboard key event should be triggered by touching this area

Возвращается:

Virtual Key ID

Пример:

global.Left = virtual_key_add(32, 32, 64, 64, vk_left);

Приведенный выше код создает виртуальную клавишу размером 64x64 пикселя, расположенную на экране по адресу (32, 32), которая будет вызывать событие vk_left при касании, и присваивает индекс этой виртуальной клавиши глобальной переменной.
