Виртуальные клавиши и виртуальные клавиатуры

При разработке игр для устройств с сенсорным экраном часто бывает важно получить от пользователя клавиатурный ввод. При использовании GameMaker это можно сделать одним из двух способов:

  • Создайте свои собственные виртуальные клавиши на экране для ввода информации
  • Всплывающая виртуальная клавиатура ОС и получение ввода с нее

То, что вы будете использовать, во многом зависит от игры и того, что вам нужно, но в целом вы будете использовать виртуальные клавиши, когда вам нужны экранные кнопки для управления игроком или для меню, в то время как вы будете использовать виртуальную клавиатуру, когда вам нужен ввод игрока, например, для получения имени или установки значения.

Для того чтобы ваши игры работали должным образом на мобильных устройствах с сенсорным экраном, вам может понадобиться возможность назначать области экрана, реагирующие на определенные действия, когда к ним прикасается пользователь. Это делается путем назначения "виртуальной клавиши" на внутренне сопоставленную стандартную клавишу клавиатуры и последующего использования генерируемых ею событий для управления вашим приложением. Стоит отметить, что эти "виртуальные клавиши" привязаны к положению экрана, а не к положению комнаты, поэтому значения x/y являются абсолютными, основанными на размере дисплея. Это означает, что вам не нужно беспокоиться об использовании представлений или относительных координат комнаты, а можно просто определить клавиши в событии Create объекта (вам нужно определить виртуальную клавишу только один раз для каждой комнаты), затем нарисовать клавишу sprites в событии Draw GUI, установив размер слоя GUI равным размеру экрана.

ПРИМЕЧАНИЕ Эти функции не будут работать ни на Windows, Ubuntu (Linux), ни на MacOS, целевых платформах, использующих сенсорный экран, хотя они будут воспринимать щелчки мыши как "прикосновения".

Эти функции поддерживаются в HTML5 и Opera GX.

Доступны следующие функции виртуальных клавиш:

  • virtual_key_add
  • virtual_key_show
  • virtual_key_hide
  • virtual_key_delete

Если вам требуется ввод текста или цифр в игре на устройстве с сенсорным экраном, то вы можете использовать функции виртуальной клавиатуры, предоставленные для использования клавиатуры ОС устройства. Эти функции будут вызывать или скрывать клавиатуру ОС и принимать ввод с нее, однако они не будут вызывать обычные события клавиатуры, а будут обновлять переменную keyboard_string с вводом пользователя (это связано с тем, что виртуальная клавиатура может вводить целые слова, а не только отдельные символы).

Открытие или закрытие виртуальной клавиатуры будет генерировать системные асинхронные события, в которых карта async_load DS map будут заполнены следующими парами ключ/значение:

  • "event_type" - тип полученного системного события, для виртуальных клавиатур это будет "virtual keyboard status".
  • "screen_height" - высота виртуальной клавиатуры (в пикселях). Это значение будет равно 0, если клавиатура невидима.
  • "keyboard_status" - текущее состояние клавиатуры, возвращается в виде одной из следующих строк:
    • "скрытие"
    • "скрытый"
    • "показывающий"
    • "видимый"

Доступны следующие функции виртуальной клавиатуры:

ПРИМЕЧАНИЕ Эти функции действительны только для целевых платформ Android (включая AndroidTV), iOS (включая tvOS) и UWP .

  • keyboard_virtual_show
  • keyboard_virtual_hide
  • keyboard_virtual_status
  • keyboard_virtual_height
