keyboard_set_map

Иногда при создании игры вы можете пожелать, чтобы одна клавиша выполняла те же действия, что и другая. Например, многие люди используют клавиши WASD для перемещения, но многие также используют клавиши со стрелками! Что же делать? Ну, вы можете закодировать систему движения дважды, но это немного сложно и, к счастью, излишне, поскольку эта функция позволяет "сопоставить" одну клавишу с другой, и тогда любой ввод с любой клавиши будет интерпретироваться как один и тот же. Для этого вы выбираете клавишу, которую хотите сопоставить (key2 - это будет та клавиша, для которой вы пишете код) и клавишу, на которую вы хотите ее сопоставить (key1). После этого нажатия на любую из клавиш будут интерпретироваться GameMaker как нажатия на клавишу key2. Вы также можете использовать эту функцию для разработки системы, в которой пользователь может определять свои собственные клавиши для игры, просто сопоставляя клавишу ввода пользователя с клавишей, которую вы закодировали в игре.

Функция принимает значение кода клавиш, возвращаемое функцией ord() (только заглавные буквы от A-Z или цифры от 0-9), или любую из констант vk_*, перечисленных на главной странице " Ввод с клавиатуры ".

ВАЖНО! После использования этой функции клавиша, к которой вы привязываете символ (клавиша1), больше не будет использоваться в качестве фактического символа клавиши. Например, если вы назначите стрелку вверх на "W", то вы больше не сможете определить нажатие клавиши клавиши "W" как "W", она всегда будет восприниматься как стрелка вверх. Чтобы отменить это, либо привяжите клавишу к самой себе - так, чтобы и клавиша1, и клавиша2 были ord("W") - либо используйте функцию keyboard_unset_map().

Синтаксис:

keyboard_set_map(key1, key2);

АргументТип Описание
key1 This is the key that key1 is to be mapped to
key2 This is the key that is to be mapped

Возвращается:

Пример:

keyboard_set_map(ord("A"), vk_left);

В приведенном выше примере кода клавиша "A" будет сопоставлена с клавишей стрелки влево. Это означает, что игрок может использовать либо клавишу "A ", либо клавишу со стрелкой влево, и что весь код, написанный для стрелки влево, также будет реагировать на клавишу "A". вместо нее.
