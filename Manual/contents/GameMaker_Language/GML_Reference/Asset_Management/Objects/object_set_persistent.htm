object_set_persistent

С помощью этой функции вы можете установить постоянство определенного объекта. Постоянный объект "переносится" из комнаты в комнату и сохраняет свои свойства в любое время (более подробную информацию о постоянстве см. в разделе " Редактор объектов"). Установка персистентности объекта означает, что все экземпляры этого объекта, созданные после изменения персистентности, будут созданы персистентными, в то время как экземпляры, уже находящиеся в комнате, могут быть не затронуты.

Когда вы создаете экземпляр с включенной персистентностью в комнате, ему будет назначен слой или глубина (в зависимости от того, какую функцию вы использовали для создания экземпляра объекта). При смене комнаты, если в следующей комнате нет слоя с тем же именем или глубиной, что и назначенный, то для экземпляра будет создан новый слой, сохраняющийся между комнатами. Если при создании экземпляра вы присвоили слою имя, то новый слой также будет называться так же, как и исходный слой, а если вы присвоили экземпляру глубину, то новый слой будет называться "_layer_XXX", где "XXX" - шестнадцатеричное значение, используемое для присвоения слою уникального имени. Эти слои будут удалены из комнаты при завершении работы комнаты. Еще один момент, связанный со слоями и постоянными экземплярами: если при создании экземпляра вы назначили ему слой с именем, а в следующих комнатах есть другой слой с тем же именем, то постоянный экземпляр будет назначен слою с тем же именем, независимо от глубины слоя. Наконец, если сохраняемый экземпляр перемещается в комнату со слоем на той же глубине, на которой он был создан, он не будет назначен этому слою, а вместо него будет создан новый слой на той же глубине (в соответствии с соглашением об именовании, описанным ранее).

Обратите внимание, что это не функция экземпляра! Вы можете установить постоянный флаг отдельных экземпляров с помощью переменной persistent и таким образом иметь десять экземпляров, помеченных как постоянные, даже если у данного объекта этот флаг установлен в false на сайте IDE, и даже если вы измените флаг постоянства объекта на true с помощью этой функции, все экземпляры, которые в данный момент находятся в комнате, останутся такими, какими они были, и только экземпляры, созданные после вызова функции, будут начинаться с флагом постоянства, помеченным как true.

Синтаксис:

object_set_persistent(index, pers);

Аргумент Тип Описание
index Объект Активы The index of the object to change.
pers Булево Whether the object should be flagged as persistent (true) или нет (false).

Возвращается:

N/A

Пример:

if (room == rm_final)
{
    object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

Приведенный выше код проверяет, в какой комнате в данный момент находится экземпляр, и если он находится в указанной "rm_final", то он устанавливает постоянство объекта "obj_Player" на false, а затем создает экземпляр.
