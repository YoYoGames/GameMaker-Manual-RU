object_is_ancestor

Эта функция может быть использована для проверки объекта и выяснения, является ли он предком или нет другого объекта. Таким образом, оба аргумента должны быть индексами объектов, а не идентификаторами экземпляров, и первый аргумент всегда является объектом, который вы хотите выяснить, является ли он дочерним для второго аргумента, который всегда является объектом, который вы хотите проверить как предка (родителя).

Синтаксис:

object_is_ancestor(obj, par);

Аргумент Тип Описание
obj Объект Активы The object that is being checked as the child.
par Объект Активы The object that is being checked as the ancestor (parent).

Возвращается:

Boolean

Пример:

if object_is_ancestor(object_index, obj_Enemy)
{
    instance_destroy();
}

Приведенный выше код проверяет, является ли экземпляр, выполняющий код, дочерним объектом объекта "obj_Enemy", и если да, то он уничтожается.
