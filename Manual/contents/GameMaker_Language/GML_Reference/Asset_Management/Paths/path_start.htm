путь_начало

Путь создается из серии определяющих точек, которые соединяются вместе и затем используются для планирования движения экземпляра. Они могут быть созданы с помощью кода или в редакторе путей и назначены экземпляру для использования в игре. С помощью этой функции можно указать экземпляру, по какому пути следовать, с какой скоростью (измеряется в пикселях за шаг), как вести себя, когда он достигнет конца пути, и следовать ли абсолютному или относительному положению пути. Последняя часть означает, что экземпляр либо начинает и следует по пути точно так, как вы спроектировали и разместили его в редакторе путей (абсолютный), либо начинает и следует по пути с позиции, в которой был создан экземпляр (относительный).

Как видно из приведенного выше изображения, два экземпляра следуют по одному и тому же пути, начатому с path_start(), но один из экземпляров следует по абсолютному пути, а другой - по относительному. Следует отметить, что при установке относительного пути на false (абсолютный) экземпляр будет "прыгать" к началу пути независимо от того, в каком месте комнаты он находится.

Константы, используемые для определения поведения экземпляра при достижении им конца пути, приведены на этой странице.

Синтаксис:

path_start(path, speed, endaction, absolute);

=======

path_start

This function tells the calling instance to start the given path. The path started by the instance is stored in the variable path_index.

A path is created from a series of defining points that are linked together and then used to plan the movements of an instance. They can be created with code, or in The Path Editor and they are assigned to an instance to use in the game. You would then use this function to tell your instance which path to follow, what speed to follow the path (measured in pixels per step), how to behave when it reaches the end of the path, and whether to follow the absolute or relative path position. This last part means that it either starts and follows the path exactly as you designed and placed it in The Path Editor (absolute), or it starts and follows the path from the position at which the instance was created (relative).

As you can see in the above image, the two instances are following the same path started with path_start, but one of the instances is following the absolute path while the other is following the relative path. It should be noted that when absolute is set to false (i.e. relative) the instance will "jump" to the start of the path no matter where in the room it is placed.

The constants used to define the behaviour of the instance when it reaches the end of the path are given on this page.

 

Syntax:

path_start(path, speed, endaction, absolute);

speed Реально The speed of which to follow the path in pixels per step, negative meaning going backwards.

Возвращается:

N/A

Пример:

path_start(path, 4, path_action_reverse, 0);

Это заставит вызывающий экземпляр следовать по пути, индексированному в переменной "path", со скоростью 4 пикселя за шаг. Он будет следовать по пути относительно своего положения, а когда достигнет конца, изменит направление и вернется назад.

Example:

This will make the calling instance follow the path indexed in the variable "path" at a speed of 4 pixels per step. It will follow the path relative to its own position, and when it reaches the end it will reverse direction and go back the way it came.
