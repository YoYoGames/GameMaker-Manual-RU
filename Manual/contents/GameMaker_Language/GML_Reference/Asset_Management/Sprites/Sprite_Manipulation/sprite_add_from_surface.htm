sprite_add_from_surface

Эта функция работает точно так же, как sprite_create_from_surface() только вместо создания нового sprite из выбранной вами области индексированной поверхности, она добавляет определенную область поверхности как новое подизображение к ранее созданному sprite (это означает, что вы не можете добавить его непосредственно к sprite из дерева ресурсов, а только к ранее созданному из поверхности или дублированному из ресурса sprite с помощью функции ). sprite_duplicate()).

Следует отметить, что память, используемая при добавлении sprite таким способом, будет больше, чем вы ожидаете. Это связано с тем, что GameMaker будет хранить sprite как текстурную страницу , и она также будет храниться в памяти GPU, поэтому общий объем памяти будет больше, чем ожидалось бы для файла изображения того же sprite.

Синтаксис:

sprite_add_from_surface(index, surface, x, y, w, h, removeback, smooth);

Аргумент Тип Описание
index Актив спрайта The index of the sprite to add the new image to.
surface Идентификатор поверхности The index of the surface from which we get the image.
x Реально The x position to copy from.
y Реально The y position to copy from.
w Реально The width of the area to be copied (from the x position).
h Реально The height of the area to be copied (from the y position).
removeback Булево Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Булево Indicates whether to smooth the edges.

Возвращается:

N/A

Пример:

spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
var i;
for (i = 1; i < 8; i += 1)
{
    sprite_add_from_surface(spr_custom, surf, i, 0, 32, 32, true, true);
}

Приведенный выше код создает sprite из поверхности, индексированной в переменной "surf", присваивая ее индекс переменной "spr_Custom", а затем использует цикл for для перемещения по поверхности и захвата различных участков, которые добавляются в sprite в качестве суб-изображений.
