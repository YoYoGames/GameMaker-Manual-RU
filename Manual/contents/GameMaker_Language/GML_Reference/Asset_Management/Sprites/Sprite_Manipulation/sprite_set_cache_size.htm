sprite_set_cache_size

При выполнении смешивания изображений HTML5 не может делать это динамически, как это может делать исполняемый файл. Поэтому GameMaker вынужден временно сохранять копию изображения и загружать ее. Эта функция устанавливает, сколько копий данного sprite может быть кэшировано, прежде чем старые копии будут перезаписаны. По умолчанию - 4. Это применяется ко всем вложенным изображениям sprite.

Синтаксис:

sprite_set_cache_size(ind, max);

АргументТип Описание
ind The index of the sprite to change the cache size of.
max The maximum number of cached copies of the sprite that can be stored.

Возвращает

Пример:

sprite_set_cache_size(sprite0, 2);

Это установит кэш sprite всех суб-изображений sprite0 в 2 копии.
