sprite_set_cache_size_ext

При выполнении смешивания изображений HTML5 не может делать это динамически, как это может делать исполняемый файл. Поэтому GameMaker вынужден временно сохранять копию изображения и загружать ее. Эта функция устанавливает, сколько копий данного sprite можно кэшировать, прежде чем старые копии будут перезаписаны. По умолчанию - 4. Это применяется к одному заданному подизображению sprite.

Синтаксис:

sprite_set_cache_size_ext(ind, index, max);

Аргумент Тип Описание
ind Актив спрайта The index of the sprite to change the cache size of.
index Реально The sub-image of ind to change the cache size of.
max Реально The maximum number of cached copies of the sprite that can be stored.

Возвращает

N/A

Пример:

sprite_set_cache_size_ext(sprite0, 0, 2);

Это установит кэш sprite первого суб-изображения sprite0 в 2 копии.
