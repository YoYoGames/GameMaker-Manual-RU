sprite_merge

Эта функция объединит sprite, индексированный в аргументе 1 ("ind2"), с тем, который индексирован в аргументе 0 ("ind1"). Сами изображения НЕ объединяются, а объединяются индексы изображений. объединяются, при этом суб-изображения из sprite "ind2" добавляются к изображениям из sprite "ind1", т.е. добавляются в конце. Обратите внимание, что если sprites разного размера, то добавленные sprites растягиваются, чтобы соответствовать размеру изображения для "ind1".

Это изменение является постоянным, и с момента использования этой функции до закрытия игры или удаления sprite, sprite, с которым происходит слияние, будет изменен, однако sprite, с которым происходит слияние, останется прежним.

ПРИМЕЧАНИЕ: Ни ресурс изображения, в который происходит слияние, ни ресурс изображения, который предназначен для слияния, не могут быть одним из исходных ресурсов игры. Вы должны использовать sprite_duplicate() первый для создания двух новых ресурсов (если вы объединяете включенные игровые ресурсы), а затем объединить их.

ПРИМЕЧАНИЕ: Эта функция полезна только для растровых изображений sprites и не работает с SWF или JSON (Spine) sprites.

Синтаксис:

sprite_merge(ind1, ind2);

АргументТип Описание
ind1 The index of the sprite to merge.
ind2 The index of the sprite that ind1 is to be merged with.

Возвращает

Пример:

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

Приведенный выше код загружает sprite в локальную переменную, затем объединяет его с другим sprite, который был продублирован. Наконец, объединенный sprite присваивается экземпляру sprite_index, а загруженный sprite удаляется. чтобы освободить используемую память. Обратите внимание, что в какой-то момент после этой операции, например, в событиях Destroy или Room End, вам также нужно будет использовать sprite_delete() чтобы удалить объединенный спрайт из памяти.
