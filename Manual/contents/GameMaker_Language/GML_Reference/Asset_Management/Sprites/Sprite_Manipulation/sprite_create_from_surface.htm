sprite_create_from_surface

С помощью этой функции можно создать sprite из ранее инициализированной поверхности (значение идентификатора индекса поверхности возвращается при создании поверхности с помощью функции surface_create()). Координаты x и y, которые вы затем вводите в функцию, должны быть относительно позиции (0,0) поверхности (левый верхний угол), а не окна игры (или вида, если он активен). Аргументы width и height задаются в пикселях и определяют ширину и высоту используемой части поверхности.

Установка аргумента "removeback" в true приведет к удалению цвета из sprite, проверяя левый нижний пиксель sprite на наличие там цвета и затем используя его в качестве удаляемого цвета.

Если вы выбрали опцию "removeback", вы также можете захотеть, чтобы GameMaker сгладил края sprite, установив аргумент "smooth" на true. Это создаст полупрозрачную границу по краям sprite после удаления фона.

Наконец, вы также можете указать начало координат x и y для sprite. Это точка, в которой sprite "фиксируется" на использующем его экземпляре, и она всегда рассчитывается как относительно левого верхнего угла 0,0 одного подизображения sprite. Так, например, sprite размером 32 x 32 пикселя с этими значениями, установленными на (16,16), будет иметь начало в центре.

По умолчанию для всех новых sprites автоматически рассчитываются ограничивающие рамки (точный размер рамки зависит от размера и прозрачности sprite), однако вы можете захотеть настроить это, в этом случае вам также следует использовать функцию sprite_collision_mask().

ПРИМЕЧАНИЕ Когда вы создаете sprite в GameMaker с помощью этого метода, вы должны не забыть удалить его снова (с помощью кнопки sprite_delete()), когда он больше не нужен, иначе есть риск утечки памяти, что приведет к замедлению и, в конечном итоге, к краху вашей игры.

Синтаксис:

sprite_create_from_surface(index, x, y, w, h, removeback, smooth, xorig, yorig);

Аргумент Тип Описание
index Идентификатор поверхности The index of the surface to create from.
x Реально The x position to copy from.
y Реально The y position to copy from.
w Реально The width of the area to be copied (from the x position).
h Реально The height of the area to be copied (from the y position).
removeback Булево Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Булево Indicates whether to smooth the edges.
xorig Реально Indicates the x position of the origin in the sprite.
yorig Реально Indicates the y position of the origin in the sprite.

Возвращает

Sprite Asset

Пример:

var surf;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);

Приведенный выше код создает поверхность и сохраняет ее индекс в локальной переменной "surf". Затем он нацеливается на эту поверхность, очищает ее и рисует несколько sprites друг на друга. Наконец, он создает новый sprite из составного изображения, нарисованного на поверхности, и присваивает его индекс "spr_Custom", после чего освобождает память, используемую поверхностью.
