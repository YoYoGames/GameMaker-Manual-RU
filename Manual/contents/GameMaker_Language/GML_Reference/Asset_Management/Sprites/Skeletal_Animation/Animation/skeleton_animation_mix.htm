skeleton_animation_mix

Вы можете легко переключать наборы анимаций с помощью функции skeleton_animation_set() но это может привести к пропуску или заиканию при смене одной анимации на другую. Чтобы предотвратить это, вы можете установить значение mix между двумя наборами анимации, и sprite будет интерполировать между ними. Обычно это нужно сделать в событии Create Event экземпляра со скелетной анимацией, так как его нужно установить только один раз, а GameMaker будет интерполировать все дальнейшие изменения sprite, используя наборы анимации этого экземпляра. Обратите внимание, что значение длительности варьируется от 0 до 1, при значении 0,5 интерполяция от одного набора к другому будет "половинчатой".

ПРИМЕЧАНИЕ В Spine 4.0 изменен принцип работы анимации вращения, поскольку она больше не интерполирует вращение по кратчайшему пути, а использует абсолютные значения в ключевых кадрах для интерполяции. Это может привести к неожиданному поведению, если вы обновляете анимацию Spine с более старой версии, и для того, чтобы вращение работало так, как нужно, могут потребоваться изменения в проекте Spine.

Синтаксис:

skeleton_animation_mix(animfrom, animto, duration);

Аргумент Тип Описание
animfrom Строка The name (a string) of the first animation set to interpolate from.
animto Строка The name (a string) of the second animation set to interpolate to.
duration Реально The duration of the interpolation (from 0 to 1)

Возвращается:

N/A

Пример:

skeleton_animation_set("walk");
skeleton_animation_mix("walk", "jump", 0.2);
skeleton_animation_mix("jump", "walk", 0.4);

Приведенный выше код входит в событие Create Event экземпляра со скелетной анимацией sprite и устанавливает длительность анимационного микса для интерполяции между двумя анимациями "ходить" и "прыгать".
