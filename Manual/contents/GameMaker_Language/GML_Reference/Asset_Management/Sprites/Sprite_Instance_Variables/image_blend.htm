смешивание изображений

Эта переменная управляет "тонированием" экземпляра sprite. Значение по умолчанию равно -1 (но может быть и c_white). Любое другое значение (включая внутренние цветовые константы, такие как c_red, или c_aqua) будет смешивать указанный цвет с исходным sprite. Обратите внимание, что для того, чтобы изменения в этой переменной были видимыми, экземпляр должен либо не иметь события рисования (и тогда GameMaker по умолчанию будет рисовать sprite), либо рисоваться с помощью одной из расширенных функций рисования, таких как draw_self() или draw_sprite_ext().

Обратите внимание, что на платформе HTML5 следует ограничить смешивание (если не используется WebGL), поскольку каждый смешиваемый sprite должен кэшироваться отдельно, поэтому наличие большого количества смешиваемых sprites может отрицательно сказаться на производительности (вы также можете установить размер кэша с помощью функции sprite_set_cache_size()).

Синтаксис:

image_blend;

Возвращается:

Colour Constant

Пример:

image_blend = make_colour_hsv(255, 255, random(255));

Приведенный выше код установит image_blend для sprite, определенного для данного экземпляра, в случайный цвет.
