индекс_изображения

Сайт sprite состоит из одного или нескольких суб-изображений, которые могут заставить sprite выглядеть анимированным при переходе от одного к другому, или они могут переключаться между собой в коде для создания различных "состояний", подобно кнопке в windows. Если sprite анимирован, то вы можете получить текущий кадр анимации, проверив переменную image_index, или если вы хотите изменить состояние статичного sprite, вы можете выбрать новое подизображение, установив эту переменную на нужное подизображение sprite. Обратите внимание, что для того, чтобы изменения в этой переменной были видны, экземпляр должен либо не иметь события рисования (и тогда GameMaker по умолчанию нарисует sprite), либо быть нарисованным с помощью одной из функций рисования, таких как draw_self() или draw_sprite_ext() (передав image_index в соответствующем аргументе).

Обратите внимание, что установка этой переменной непосредственно на кадр не вызовет никаких широковещательных сообщений, которые могут присутствовать на этом кадре.

Также обратите внимание, что при использовании скелетной анимации sprites вы все еще можете получать и устанавливать значения image_index - см. функцию skeleton_animation_get_duration() для примеров того, как это сделать.

Синтаксис:

image_index;

Возвращается:

(single precision floating point value)

Пример:

if (image_speed > 0)
{
    if (image_index >= image_number - 1) instance_destroy();
}

Приведенный выше код проверяет, анимируется ли sprite, и если да, то проверяет, находится ли текущий image_index на последнем кадре, и в этом случае уничтожает экземпляр.
