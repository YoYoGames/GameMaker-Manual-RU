shader_set_uniform_f

С помощью этой функции можно установить значение (или значения) константы шейдера. Предварительно вы должны получить "хэндл" константы с помощью функции shader_get_uniform()и вы должны знать тип константы, чтобы передать ей правильное количество значений с плавающей запятой, т.е. если у вас есть vec2, вам нужно передать в функцию два значения.

Исключением из этого правила является работа с 32-битными значениями цвета. Вы можете передать в шейдер до восьми 32-битных значений цвета, и они будут автоматически преобразованы в значения vec4 0 - 1. Однако для этого вы должны использовать имя константы шейдера in_Colour для одного цвета, или in_Colour0, in_Colour1 и т.д. вплоть до in_Colour7 для нескольких входных значений. Следующие примеры показывают, как это будет выглядеть в самом шейдере:

    //Single colour
    attribute vec2 in_Position;
    attribute vec4 in_Colour;
    attribute vec2 in_TextureCoord;

    //Multiple colours
    attribute vec2 in_Position;
    attribute vec4 in_Colour0;
    attribute vec4 in_Colour1;
    attribute vec2 in_TextureCoord;

Обратите внимание, что если вы хотите сделать сжатые нормали и т.д., используя 4-байтовый формат цвета, то вам также придется использовать формат "in_Colour", как показано выше.

ПРИМЕЧАНИЕ: Все формы должны быть установлены после вызова функции shader_set(), и до вызова shader_reset().

Синтаксис:

shader_set_uniform_f(handle, value1 [, value2, value3, value4]);

Аргумент Тип Описание
handle Равномерная обработка шейдера The handle of the shader constant to set.
value1 ... value4 Реально The floating point value (or values) to set the shader constant to.

Возвращается:

N/A

Пример:

shader_set(shader_Glass);
shader_params = shader_get_uniform(shader_glass, "u_vParams");
shader_set_uniform_f(shader_params, 0.1, 0.8, 0.25);
draw_self();
shader_reset();

Приведенный выше код получит хэндл шейдерной константы "u_vParams" (a vec3), затем установит эту константу в заданные значения.
