script_execute

Эта функция изначально была разработана для динамического вызова активов сценария, но с введением пользовательских функций ее теперь можно использовать двумя способами:

  • Первый способ - это поддержка унаследованных скриптов, когда каждый скрипт представлял собой отдельную функцию, а название скрипта было также названием функции. Используемый таким образом, он будет работать, как и раньше, и будет запускать созданный пользователем актив скрипта и передавать в него заданные аргументы. Обратите внимание, что в данном случае вызывается не скрипт, а одноименная функция внутри скрипта.
  • Второй способ - вызвать функцию из актива сценария, содержащего несколько функций. В этом случае нужно указать имя функции и аргументы, которые она требует, а не имя актива сценария. Обратите внимание, что это работает только для функций сценария, но не для методов, поскольку только функции сценария имеют присвоенный им идентификатор сценария.

Несмотря на то, что эта функция предназначена в основном для поддержки старых версий, она не является устаревшей, так как все еще имеет свое место в сетевых ситуациях и ситуациях удаленного вызова процедур. Стоит также отметить, что эта функция медленная, и ее частое использование может негативно сказаться на производительности.

ПРИМЕЧАНИЕ Эту функцию нельзя использовать со встроенными функциями.

Синтаксис:

script_execute(scr, arg0, arg1, arg2..., etc...);

Аргумент Тип Описание
scr Актив сценария или Функция сценария The function/script that you want to call
arg0, arg1, arg2..., etc... Переменная The different arguments that you want to pass through to the function/script

Возвращается:

Variable (Will depend on the return value from the script/function being called)

Пример:

script_execute(choose(move_up, move_down, move_left, move_right), irandom(5));

Приведенный выше пример кода будет использовать script_execute для выбора одной из четырех определенных пользователем функций (в данном случае связанных с движением) и передавать ей случайное целое число в качестве первого аргумента.
