постоянный

Эту переменную можно прочитать, чтобы узнать, помечен ли экземпляр как persistent или нет, или использовать ее для установки постоянства true (постоянный) или false (не постоянный) для данного экземпляра. Постоянный экземпляр - это экземпляр, который будет "переноситься" из комнаты в комнату, то есть (например) его нужно создать только один раз в начале игры, и он будет присутствовать во всех последующих комнатах. С постоянными экземплярами следует быть осторожными, так как их легко потерять, что может привести к проблемам на более поздних этапах разработки игры.

Когда вы создаете экземпляр с включенной персистентностью в комнате, эта переменная будет установлена в true, и ему будет назначен слой или глубина (в зависимости от того, какую функцию вы использовали для создания экземпляра объекта). При смене комнаты, если в следующей комнате нет слоя с тем же именем или глубиной, что и назначенный, то для экземпляра будет создан новый слой, сохраняющийся во всех комнатах. Если при создании экземпляра вы присвоили слою имя, то новый слой также будет называться так же, как и исходный слой, а если вы присвоили экземпляру глубину, то новый слой будет называться "_layer_XXX", где "XXX" - шестнадцатеричное значение, используемое для присвоения слою уникального имени. Эти слои будут удалены из комнаты при завершении работы комнаты. Еще один момент, связанный со слоями и постоянными экземплярами: если при создании экземпляра вы назначили ему слой с именем, а в следующих комнатах есть другой слой с тем же именем, то постоянный экземпляр будет назначен слою с тем же именем, независимо от глубины слоя. Наконец, если сохраняемый экземпляр перемещается в комнату со слоем на той же глубине, на которой он был создан, он не будет назначен этому слою, а вместо него будет создан новый слой на той же глубине (в соответствии с соглашением об именовании, описанным ранее).

У постоянного объекта по-прежнему будут срабатывать события Game Start, Game End, Room Start и Room End, однако если вы перезапустите игру (например, с помощью функции game_restart() ), все постоянные объекты будут удалены и будут существовать только тогда, когда будут созданы игрой заново. Также обратите внимание, что если вы деактивируете постоянный объект, он больше не будет переходить из одной комнаты в другую, если не будет повторно активирован до срабатывания события Room End.

Синтаксис:

persistent;

Возвращается:

Пример:

if !persistent
{
    persistent = true;
}

Приведенный выше код проверит экземпляр, чтобы убедиться, что постоянство отмечено как false, и если экземпляр НЕ постоянный, то флаг постоянства будет установлен в true.
