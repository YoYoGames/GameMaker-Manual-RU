Аудио

GameMaker имеет полноценный звуковой движок, основанный на звуковых форматах *.ogg, *.mp3 и *.wav. Звуки этих типов, добавленные в IDE, могут быть использованы в вашей игре с помощью базовых аудиофункций, показанных ниже. Для более сложных вещей, чем базовые звуковые эффекты или воспроизведение одного музыкального фрагмента, вы можете обратиться к расширенным аудиофункциям, которые позволяют изменить способ воспроизведения звука. Есть также ряд более специализированных функций, предназначенных для потоковой передачи звука, позиционирования звука - для создания 3D-звука - и группировки звука.

ВАЖНО! При использовании звука в HTML5-цели следует помнить, что не все браузеры поддерживают Web Audio, поэтому при запуске ваш проект может не воспроизводить звук. Вы можете получить общее представление о поддержке Web Audio по следующей ссылке: Могу ли я использовать Web Audio?

  • audio_get_type
  • audio_play_sound
  • audio_play_sound_at
  • audio_pause_sound

  • audio_resume_sound

Поэтому рекомендуется, чтобы вы хорошо знали, как работает остальная часть звукового движка GameMaker, прежде чем использовать любую из следующих функций:

Веб-аудио

При создании игр для целевой платформы HTML5 звуковой движок требует поддержки Web Audio, а это, в свою очередь, означает, что иногда звук не будет воспроизводиться, когда или как вы ожидаете. Это происходит потому, что контекст Web Audio может не работать или перестать работать во время воспроизведения игры. Причины этого сильно различаются в разных браузерах и даже в разных версиях одного и того же браузера, поэтому обнаружение изменения состояния контекста веб-аудио очень важно, например: для обнаружения изменения контекста и приостановки/запуска зацикленного аудио, например фоновой музыки.

Чтобы решить эту проблему, GameMaker предлагает два разных способа обнаружения изменения состояния контекста Web Audio, либо с помощью следующей функции:

  • audio_system_is_available

Вы также можете использовать асинхронное системное событие, которое будет срабатывать всякий раз при изменении состояния Web Audio. В этом событии вы получите встроенную async_load DS map, заполненная ключом "event_type", который, в свою очередь, будет содержать строку "audio_system_status", если это аудиособытие. Когда этот ключ существует, также будет существовать дополнительный ключ "status", который будет либо "available", либо "unavailable". Обратите внимание, что это событие будет срабатывать на всех платформах, но на всех, кроме HTML5, оно будет срабатывать только один раз при запуске игры, когда аудиодвижок впервые инициализируется.
