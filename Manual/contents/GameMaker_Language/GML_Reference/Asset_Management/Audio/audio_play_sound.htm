audio_play_sound

С помощью этой функции вы можете воспроизводить любой звуковой ресурс в вашей игре. Вы предоставляете индекс звука и присваиваете ему приоритет, который затем используется для определения того, как звуки будут обрабатываться, когда количество воспроизводимых звуков превысит предел, установленный функцией audio_channel_num(). Звуки с более низким приоритетом будут остановлены в пользу звуков с более высоким приоритетом, а значение приоритета может быть любым реальным числом (фактическое значение произвольно и может быть от 0 до 1 или от 0 до 100, поскольку GameMaker будет определять их приоритет одинаково). Обратите внимание, что при работе с приоритетами, чем выше число, тем выше приоритет, так что звук с приоритетом 100 будет предпочтительнее звука с приоритетом 1. Третий аргумент предназначен для зацикливания звука, и если установить его в true, то звук будет повторяться до тех пор, пока его не остановят, а если установить его в false, то звук будет воспроизводиться только один раз.

Эта функция также возвращает уникальный индексный номер воспроизводимого звука, который можно сохранить в переменной, чтобы затем поставить его на паузу или остановить с помощью соответствующих функций. Это означает, что если у вас есть несколько экземпляров одного и того же звука, воспроизводимого в любой момент времени, вы можете выбрать только один экземпляр этого звука для работы с ним с помощью аудиофункций.

Синтаксис:

audio_play_sound(index, priority, loop, [gain], [offset], [pitch], [listener_mask]);

Аргумент Тип Описание
index Звуковой актив The index of the sound to play.
priority Реально Set the channel priority for the sound.
loop Булево Sets the sound to loop or not.
gain Реально OPTIONAL Ценность выигрыша.
offset Реально OPTIONAL Время (в секундах), на которое нужно установить начальную точку. Значения, превышающие длину заданного звука, игнорируются.
pitch Реально OPTIONAL Множитель шага (по умолчанию 1).
listener_mask Реально OPTIONAL Данные битовой маски, которые нужно установить для звука. Для цели HTML5 это не будет иметь никакого эффекта, поскольку цель не поддерживает более одного слушателя.

Возвращается:

Sound Instance ID

Пример 1:

if health <= 0
{
    lives -= 1;
    audio_play_sound(snd_PlayerDead, 10, false);
}

Приведенный выше код проверяет глобальную переменную "health", и если она меньше или равна 0, то удаляет 1 из глобальной переменной "lives" и воспроизводит звук.

Пример 2:

if bbox_left > room_width
{
    audio_play_sound(snd_Goodbye, 10, false, 1.1, 0, 2);
}

Приведенный выше код проверяет, находится ли экземпляр за правой границей комнаты, и воспроизводит звук с коэффициентом усиления 1,1 и высотой тона 2 (в два раза выше обычного).
