аудио_отладка

Эта функция может использоваться для отображения отладочной информации об аудиосистеме, при этом true включает ее, а false - выключает. При отладке аудио таким образом на оверлее будет отображаться следующая информация:

  • Handle - хэндл источника OpenAL (уникальный для каждого воспроизводимого звука)
  • Data Handle - Ручка буфера OpenAL (показывает одно и то же значение для нескольких звуков с одинаковой ручкой источника).
  • Обработанные буферы - сколько буферов было воспроизведено
  • Очередь буферов - сколько буферов было поставлено в очередь (и все еще ожидает воспроизведения).
  • Усиление - Значение усиления для аудио (0 - 65536).
  • Play Offset - позиция сэмпла (текущая позиция воспроизведения) в текущем буфере
  • Бары буферов в очереди - небольшая диаграмма обработанных/очередных буферов с смещением воспроизведения (текущая позиция), нарисованная синей линией.

Для синхронизированных групп аудио также имеется функция audio_sync_group_debug().

ПРИМЕЧАНИЕ Эта функция не работает на целевой платформе HTML5.

Синтаксис:

audio_debug(enable);

АргументТип Описание
enable Enable (set to true) или отключить (false) отладку звука.

Возвращается:

Пример:

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

Приведенный выше код будет включать или выключать наложение отладки звука в зависимости от того, запущена игра в режиме отладки или нет.
