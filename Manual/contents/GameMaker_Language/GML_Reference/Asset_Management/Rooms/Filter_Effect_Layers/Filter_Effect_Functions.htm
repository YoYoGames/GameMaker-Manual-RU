Функции фильтров и эффектов

Обзор

Существуют различные функции GML, которые можно использовать для создания, изменения и удаления фильтров/эффектов из слоев в помещении, что позволяет легко управлять эффектами в режиме реального времени для создания реалистичных и динамичных фильтров/эффектов.

Последний метод (указание фильтра/эффекта в коде) работает только со строковыми литералами, непосредственно указанными в аргументе функции, что означает, что если вы используете переменную или какую-либо логику, чтобы придумать строку имени фильтра/эффекта, то компилятор активов не обнаружит этого, и фильтр/эффект не будет загружен.

Рассмотрим следующие примеры:

// This will work on its own
var _fxshake = fx_create("_filter_screenshake");

// This will NOT work on its own
var _myfilters = { screenshake: "_screenshake" }
var _filter_to_use = "_filter" + _myfilters.screenshake;
var _fxshake = fx_create(_filter_to_use);

Чтобы убедиться, что последний метод работает, вы можете просто добавить фильтр хотя бы в одну комнату в вашем проекте, или убедиться, что fx_create() вызывается в любом месте вашего проекта с именем фильтра в качестве строковой константы (а не переменной).

Список функций

Следующие функции используются для создания и управления "FX Structs", содержащими данные эффектов:

  • fx_create
  • fx_get_parameter
  • fx_get_parameters
  • fx_get_name
  • fx_get_parameter_names
  • fx_get_single_layer
  • fx_set_parameter
  • fx_set_parameters
  • fx_set_single_layer

Следующие функции используются для модификации слоев, которые могут содержать фильтры/эффекты, с помощью структур FX:

  • layer_set_fx
  • layer_get_fx
  • layer_clear_fx
  • layer_enable_fx
  • layer_fx_is_enabled

Изменение FX во время выполнения

Вы можете изменять фильтры/эффекты во время выполнения, выполнив следующие действия:

  • Извлеките структуру FX из слоя, который вы хотите изменить, с помощью вызова layer_get_fx()
    • Или создайте новую структуру FX, вызвав команду fx_create(), и примените ее к слою с помощью команды layer_set_fx()
  • Получите его структуру параметров, вызвав fx_get_parameters()
  • Измените параметры по мере необходимости, присвоив значения переменным struct
    • Получите названия параметров отсюда: Типы и параметры FX
  • Примените модифицированную структуру обратно к структуре FX вызовом fx_set_parameters()
    • Вам не нужно вызывать layer_set_fx() здесь, поскольку модификация структуры FX непосредственно влияет на слой, которому она уже назначена.

Вот пример кода для вышеупомянутого рабочего процесса:

Создать событие

// Store the FX struct, and its parameters struct, in variables
pixelate_fx = layer_get_fx("Effect_1");
pixelate_fx_params = fx_get_parameters(pixelate_fx);

Шаговое мероприятие

// Change param as variable
pixelate_fx_params.g_CellSize = round((mouse_x / room_width) * 64);

// Or, change param as string
pixelate_fx_params[$ "g_CellSize"] = round((mouse_x / room_width) * 64);

// Apply updated parameters struct to the FX struct
fx_set_parameters(pixelate_fx, pixelate_fx_params);

Параметры времени выполнения FX

На странице FX Types & Parameters перечислены все фильтры/эффекты вместе с их параметрами времени выполнения.

Имена параметров Runtime можно использовать следующими тремя способами (на примере параметра "g_CellSize" ):

  • Измените параметр в структуре FX, вызвав fx_set_parameter(): fx_set_parameter(fx_struct, "g_CellSize", 8);
  • Измените переменную в структуре параметров: params_struct.g_CellSize= 8;
    • ПРИМЕЧАНИЕ Убедитесь, что сначала получили структуру параметров, вызвав fx_get_parameters().
  • Измените переменную в структуре параметров, используя аксессор $ и строку: params_struct[$ "g_CellSize"] = 8;

Однослойный режим

По умолчанию фильтр/эффект применяется к слою, которому он назначен, и ко всем слоям ниже этого слоя, однако вы можете воспользоваться функцией fx_set_single_layer() чтобы включить режим "Один слой " для фильтра/эффекта и убедиться, что он применяется только к тому слою, которому назначен.
