font_add_sprite_ext

С помощью этой функции вы можете использовать "спрайтовую полосу" (сам sprite должен быть активом sprite из браузера активов или sprite, который вы добавили в игровые ресурсы с помощью sprite_add) для создания нового актива шрифта, где каждое подизображение будет отдельным символом или буквой. В отличие от обычного шрифта font_add_sprite() которая имеет определенный порядок для субизображений sprite, эта функция отображает субизображения sprite на основе аргумента "string_map" функции. Этот аргумент представляет собой строку, которую вы можете использовать, чтобы сообщить GameMaker, в каком порядке расположены подрисунки шрифта sprite, и он будет автоматически отображать их при написании текста. Например, если вы задали строку "AaBbCcDdEeFfGgHh", то ваш шрифт sprite должен иметь именно такой порядок расположения субизображений. Вы можете определить "пробел" как любой символ, например, одну строку такого размера, каким вы хотите видеть пробел, и все пробелы в тексте будут отображаться с такой шириной (но выбранное изображение никогда не будет отображаться), или, если вы не указали sprite для пробела, вместо него будет использоваться ширина самого широкого символа в шрифте sprite.

Вы также можете задать пропорциональный (true) или непропорциональный (false) интервал для шрифта. При пропорциональном шрифте интервал определяется фактической шириной букв (например, буква "i" занимает меньше места, чем буква "w"), а при непропорциональном - шириной подрисуночного изображения, как при моноширинном шрифте. Наконец, вы можете определить пространство, которое нужно оставить между каждой буквой при написании, и это может быть любое целое значение, при этом 0 означает отсутствие пространства (при пропорциональном шрифте буквы будут "соприкасаться").

Функция возвращает значение индекса, которое должно быть сохранено в переменной, так как оно будет необходимо во всех дальнейших кодах, которые будут ссылаться на этот шрифт, или возвращает -1, если функция не работает по какой-либо причине.

Стоит отметить, что для работы функций выравнивания шрифтов (например, таких как draw_set_halign()), у шрифта sprite начало координат должно быть установлено в левом верхнем углу. Если вы используете другие значения, то при рисовании текста с помощью этого шрифта вам придется учитывать смещение начала координат.

ПРИМЕЧАНИЕ Когда вы загружаете шрифт в GameMaker, вы должны не забыть удалить его снова (с помощью кнопки font_delete()), если он больше не нужен, иначе существует риск утечки памяти, что приведет к замедлению и, в конечном итоге, к краху вашей игры.

Синтаксис:

font_add_sprite_ext(spr, string_map, prop, sep);

Аргумент Тип Описание
spr Актив спрайта The sprite to add a font based on.
string_map Строка String from which sprite sub-image order is taken.
prop Булево Set as proportional font or not.
sep Реально The space to leave between each letter.

Возвращается:

Font Asset

Пример:

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

Приведенный выше код создаст новый актив шрифта из sprite, индексированного в переменной "spr_CalcFont", и сохранит индекс нового шрифта в переменной "global.Font". Подобразы шрифта sprite будут отображены на указанную строку.
