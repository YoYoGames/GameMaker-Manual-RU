рабочий_каталог

Working_directory может возвращать два разных значения в зависимости от того, для чего вы его используете.

Если вы записываете файл на диск, working_directory указывает на область локального хранилища, зарезервированную для вашей игры на целевом устройстве (в Windows это "%LOCALAPPDATA%/gamedir/", где "gamedir" - каталог с именем вашей игры).

Однако при чтении с диска working_directory может быть либо локальным хранилищем (game_save_id) , либо областью, где хранятся Включенные файлы.

ПРИМЕЧАНИЕ Переменная working_directory возвращает путь, включая последний обратный слэш.

Например, если вы включили в игру файл по умолчанию *.txt и считываете его в память с помощью working_directory, то этот файл будет взят из области, где хранятся файлы Included Files. Если затем вы решите записать эту информацию в текстовый файл с помощью working_directory, то файл будет записан в локальное хранилище целевой платформы, откуда он будет считан в следующий раз, если только вы не дадите GameMaker указание удалить файл, в этом случае working_directory снова будет указывать на область, где хранятся Включенные файлы.

Стоит отметить, что в общем случае вам не нужно использовать это и достаточно просто указать само имя файла (плюс любую дополнительную информацию о пути), чего вполне достаточно. Например, это:

file_text_open_read(working_directory + "my_file.txt");

было бы лучше сделать так:

file_text_open_read("my_file.txt");

ПРЕДУПРЕЖДЕНИЕ Эта функция может работать не так, как вы ожидаете, из-за того, что GameMaker находится в "песочнице"! Пожалуйста, обратитесь к разделу о файловой системе для получения дополнительной информации.

Синтаксис:

working_directory

Возвращается:

String

Пример:

ini_open(working_directory + "temp_ini.ini");

Это откроет ini-файл из рабочего каталога игры (создав его, если он еще не существует). Это может быть локальное хранилище или область, где находятся включенные файлы, в зависимости от того, существует ли искомый файл в любом из этих мест.
