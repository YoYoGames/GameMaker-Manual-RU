рабочий_каталог

Working_directory может возвращать два разных значения в зависимости от того, для чего вы его используете.

Если вы записываете файл на диск, working_directory указывает на область локального хранилища, зарезервированную для вашей игры на целевом устройстве (в Windows это "%LOCALAPPDATA%/gamedir/", где "gamedir" - каталог с именем вашей игры).

Однако при чтении с диска working_directory может быть либо локальным хранилищем (game_save_id) , либо областью, где хранятся Включенные файлы.

ПРИМЕЧАНИЕ Переменная working_directory возвращает путь, включая последний обратный слэш.

Например, если вы включили в игру файл по умолчанию *.txt и считываете его в память с помощью working_directory, то этот файл будет взят из области, где хранятся файлы Included Files. Если затем вы решите записать эту информацию в текстовый файл с помощью working_directory, то файл будет записан в локальное хранилище целевой платформы, откуда он будет считан в следующий раз, если только вы не дадите GameMaker указание удалить файл, в этом случае working_directory снова будет указывать на область, где хранятся Включенные файлы.

Стоит отметить, что в общем случае вам не нужно использовать это и достаточно просто указать само имя файла (плюс любую дополнительную информацию о пути), чего вполне достаточно. Например, это:

=======

working_directory

working_directory can actually return two different values depending on what you are using it for: 

  • If you are writing a file to disk, working_directory points to the area of local storage that is reserved for your game on the target device (in Windows this is "%LOCALAPPDATA%/gamedir/" where the "gamedir" is the directory with the name of your game).
  • However, when reading from disk, working_directory can be either the local storage (game_save_id) or the area where the Included Files are stored.

 The working_directory variable will return the path including the final backslash.

For example, if you have a default *.txt file included with your game and read it into memory using working_directory, then that file will be taken from the area where the Included Files are stored. If you then choose to write that information to a text file using working_directory, this will write the file to the target platform's local storage, where it will be read from the next time unless you instruct GameMaker to delete the file, in which case working_directory will once more point to the area where the Included Files are stored.

It is worth noting that in general you do not need to use this and simply specify the file name itself (plus any additional path information), which is sufficient. For example, this:

Example:

This will open an INI file from the working directory of the game (creating it if it does not already exist). This could be the local storage, or the area where the included files are depending on whether the file being looked for exists in either place.
