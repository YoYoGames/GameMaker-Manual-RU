md5_string_utf8

В криптографии MD5 (Message-Digest algorithm 5) - это широко используемая криптографическая хэш-функция со 128-битным хэш-значением, которая применяется в самых разных приложениях безопасности. Она также часто используется для проверки целостности файлов и строк. Эта функция принимает на вход строку UTF-8 (которая имеет переменное количество байт на символ) и возвращает 32-символьный шестнадцатеричный хэш MD5, уникальный для этой строки. Таким образом, вы можете сгенерировать безопасный ключ, который можно хранить и использовать для проверки целостности информации, отправляемой (или получаемой) на внешний сервер (например).

ПРИМЕЧАНИЕ: Существует два формата кодировки MD5 - UTF-8 и unicode. Оба формата предусмотрены для облегчения взаимодействия с различными настройками сервера, но чаще всего используется unicode.

ПРИМЕЧАНИЕ: MD5 не является полностью безопасным и может быть взломан. См. эту страницу для получения дополнительной информации.

Синтаксис:

md5_string_utf8(string)

АргументТип Описание
string The string to hash.

Возвращается:

Пример:

var hash, str;
str = json_encode(hiscore_map);
hash = md5_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

Приведенный выше код кодирует карту DS в строку JSON. Затем генерируется хэш MD5 и сохраняется в ini-файле, чтобы впоследствии его можно было использовать для проверки целостности JSON при получении той же информации от сервера. JSON затем отправляется.
