Web и HTML5

GameMaker предоставляет различные функции для взаимодействия с интернетом или сервером, и их можно использовать на всех доступных целевых платформах. К таким функциям относятся:

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

Помимо общих функций, перечисленных выше, целевой HTML5 имеет некоторые дополнительные веб-функции, которые специально разработаны для использования играми, ориентированными на него:

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

Помимо этих функций для целевого модуля HTML5, существует также специальная переменная JavaScript, которую можно использовать в тех редких случаях, когда содержимое папки html5game хранится в совершенно другом месте (URL). Обычно этот путь жестко закодирован в файле JavaScript и является относительным к месту, откуда вызывается JavaScript, но вы можете установить переменную g_GameMakerHTML5Dir перед вызовом сценария игры, и игра будет запущена из вновь определенного места. Например:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

Также стоит отметить, что вы можете пометить функции сценария с помощью префикса gmcallback_, что означает, что они не будут обфусцированы при компиляции проекта для HTML5. Это очень удобно при использовании перечисленных выше функций clickable_* (более подробную информацию можно найти здесь).
