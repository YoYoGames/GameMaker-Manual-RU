clickable_add

С помощью этой функции вы можете создать пользовательскую, кликабельную иконку, которая действует как "кнопка" вне игровой канвы, но в самом окне браузера. Функция возвращает значение ID, которое должно быть сохранено в переменной для использования во всех последующих функциях кнопки, которые ссылаются на новую кликабельную иконку. Эта функция очень полезна для предотвращения создания браузером всплывающего окна при нажатии на кнопку, поскольку все это делается в самом GameMaker.

Сначала необходимо задать позицию, которая является относительной по отношению к (0,0) окна браузера (левый верхний угол), а затем дать текстуре страницы запись для sprite, которую вы хотите видеть у значка кнопки (см: sprite_get_tpe()). Затем вы можете указать либо URL-адрес для открытия, либо скрипт (из браузера активов GameMaker) для открытия или запуска. Параметр "target", который вы указываете, совпадает со стандартным значением JavaScript "name" при использовании метода open() (имейте в виду, что все значения, кроме '_self', могут привести к блокировке браузера или к тому, что браузер спросит пользователя, разрешить ли ему это), а аргумент "params" совпадает с параметром JavaScript "specs" для управления свойствами нового окна/вкладки (не все браузеры могут поддерживать все функции). Если вы используете иконку для запуска внутреннего сценария, эти два последних аргумента можно оставить пустыми строками "".

Действительными целями являются:

Цель Описание
_blank Открывает связанный документ в новом окне или вкладке.
_self Открывает связанный документ в том же фрейме, в котором он был щелкнут (по умолчанию).
_родитель Открывает связанный документ в родительском фрейме.
_top Открывает связанный документ в полном тексте окна.

Доступными параметрами являются:

Параметры Описание
'height=[px]' Высота окна, минимальное значение - 100.
'width=[px]' Ширина окна, минимальное значение - 100.
'left=[px]' Левая позиция окна.
'top=[px]' Верхнее положение окна (только в IE).
'location=[boolean]' Отображать или нет поле адреса (по умолчанию 1).
'menubar=[boolean]' Отображать или нет строку меню (по умолчанию 1).
'resizable=[boolean]' Можно ли изменять размеры окна (по умолчанию 1).
'scrollbars=[boolean]' Отображать или нет полосы прокрутки (по умолчанию 1).
'status=[boolean]' Добавлять или нет строку состояния (по умолчанию 1).
'titlebar=[boolean]' Отображать или нет строку заголовка. Этот параметр игнорируется, если вызывающее приложение не является HTML-приложением или доверенным диалоговым окном (по умолчанию 1);
'toolbar=[boolean]' Отображать или нет панель инструментов браузера (по умолчанию - да).


Также стоит отметить, что если передаваемый вами аргумент URL является именем функции скрипта (т.е. не начинается с "http://") И скрипт называется gmcallback_*, то вы можете получить прямую передачу в html5-событие. Имена функций скриптов GameMaker не обфусцируются, если они начинаются с gmcallback_.

Синтаксис:

clickable_add(x, y, tpe, url, target, params);

Аргумент Тип Описание
x Реально The x position within the window.
y Реально The y position within the window.
tpe Ввод страницы текстуры The texture page entry for the sprite to be used.
url Строка The URL (website address) to link to, or the script to be run.
target Строка This is the target area to open the URL in (see description).
params Строка The various parameters for the icon (see description).

Возвращается:

Clickable ID

Пример:

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

Приведенный выше код создает кликабельную DOM-иконку в позиции (32, 32) страницы, на которой запущен игровой холст. Иконка использует sprite, указанный на странице текстуры как "spr_MS_Home", и при нажатии на иконку откроется новое окно для указанного URL и с определенными свойствами для этого окна.
