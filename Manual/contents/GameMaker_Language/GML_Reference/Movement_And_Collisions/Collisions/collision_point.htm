точка_столкновения

Collision_point проверяет точку, указанную аргументами x1,y1, на столкновение с любым экземпляром объекта, указанного аргументом "obj". Эта проверка может быть точной или нет, но для включения точных столкновений объект или экземпляр, который вы проверяете, должен также иметь включенные точные столкновения для своего sprite. Если нет, то проверка по умолчанию основывается на граничных коробках. На следующем изображении показано, как это работает:

Помните, что для учета точных столкновений и объект sprite, и функция столкновения должны быть отмечены как точные. Следует также отметить, что возвращаемое значение функции может быть id любого из экземпляров, считающихся столкнувшимися, поэтому если три экземпляра пересекаются в данной точке, то любой из их id может быть возвращаемым значением функции.

Обратите внимание, что вместо индекса объекта вы можете указать экземпляр id для проверки конкретного экземпляра, или ключевые слова экземпляра all, или other (в зависимости от события и текущей области видимости кода).

Синтаксис:

collision_point(x, y, obj, prec, notme);

Аргумент Тип Описание
x Реально The x coordinate of the point to check.
y Реально The y coordinate of the point to check.
obj Идентификаторактива или экземпляраобъекта The object to check for instance collisions.
prec Булево Whether the check is based on precise collisions (true, что медленнее) или его граничная область в целом (false, быстрее).
notme Булево Whether the calling instance, if relevant, should be excluded (true) или нет (false).

Возвращается:

Instance ID or noone

Пример:

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10S;
}

Здесь мы проверяем точку в позиции объекта, имеющего код объекта "obj_Cursor". Если такая точка есть, то мы добавляем 10 к переменной score.
