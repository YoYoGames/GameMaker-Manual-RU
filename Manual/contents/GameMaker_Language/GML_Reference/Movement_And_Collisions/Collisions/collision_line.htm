линия столкновения

Collision_line проверяет линию от точки x1,y1 до точки x2,y2 на столкновение с любым экземпляром объекта, указанного аргументом "obj". Эта проверка может быть точной или нет, но для включения точных столкновений объект или экземпляр, который вы проверяете, должен также иметь включенные точные столкновения для своего sprite. Если нет, то проверка по умолчанию основывается на граничных коробках. На следующем изображении показано, как это работает:

Помните, что для учета точных столкновений и объект sprite, и функция столкновения должны быть отмечены как точные. Следует также отметить, что возвращаемое значение функции может быть id любого из экземпляров, считающихся столкнувшимися вдоль линии, поэтому если три экземпляра пересекают определенную линию, то любой из их id может быть возвращаемым значением функции.

Обратите внимание, что вместо индекса объекта вы можете указать экземпляр id для проверки конкретного экземпляра, или ключевые слова экземпляра all, или other (в зависимости от события и текущей области видимости кода).

Синтаксис:

collision_line(x1, y1, x2, y2, obj, prec, notme);

Аргумент Тип Описание
x1 Реально The x coordinate of the start of the line.
y1 Реально The y coordinate of the start of the line.
x2 Реально The x coordinate of the end of the line.
y2 Реально The y coordinate of the end of the line.
obj Идентификаторактива или экземпляраобъекта The object to check for instance collisions.
prec Булево Whether the check is based on precise collisions (true, что медленнее) или его граничная область в целом (false, быстрее).
notme Булево Whether the calling instance, if relevant, should be excluded (true) или нет (false).

Возвращается:

Instance ID or noone

Пример:

with (obj_Enemy)
{
    if collision_line(100, 400, 100, 600, id, false, false) instance_destroy();
}

Этот код заставляет все экземпляры "obj_Enemy" проверить линию от 100x, 400y до 100x, 600y на наличие столкновения с собой, и если таковое имеется, то они уничтожаются.
