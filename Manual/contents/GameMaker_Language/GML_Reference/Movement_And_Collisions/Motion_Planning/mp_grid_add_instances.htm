mp_grid_add_instances

Эта функция использует маску столкновений экземпляра (определяется свойствами sprite или индексом маски вызывающего экземпляра) для пометки ячеек как запрещенных или нет в сетке MP, где запрещенные ячейки не могут быть пересечены ни одной из функций поиска пути. В функции можно указать, учитывать ли точные столкновения или нет, и в зависимости от этого будут меняться отмеченные запрещенные ячейки. Это изображение иллюстрирует такое поведение:

Два вышеприведенных экземпляра были добавлены в сетку MP с помощью функции mp_grid_add_instances() с установленным значением true. Как вы можете видеть, зеленый экземпляр пометил запрещенными только те клетки сетки, которых он "касается", благодаря тому, что его маска sprite также установлена на precise. Однако второй экземпляр пометил и другие клетки, которые , похоже, не касаются. Это происходит потому, что маска sprite не установлена на точную, а значит, даже если вы используете точную настройку в функции, будет рассматриваться только граничная область sprite. То же самое произойдет, если установить точный аргумент функции на false - даже экземпляры с точной маской sprite будут добавлены в сетку на основе их граничных полей.

Синтаксис:

mp_grid_add_instances(id, obj, prec);

Аргумент Тип Описание
id MP Grid ID Index of the mp_grid that is to be used
obj Идентификаторактива или экземпляраобъекта Object index, or instance id, of the instances to be added into the mp_grid
prec Булево Whether the check is based on pixel-perfect collisions (true = slow) or its bounding box in general (false = fast).

Возвращается:

N/A

Пример:

mp_grid_add_instances(grid, obj_Wall, true);

Приведенный выше код добавит все экземпляры "obj_Wall" в mp_grid, проиндексированную в переменной "grid", используя точную маску столкновений, а не ограничивающую рамку.
