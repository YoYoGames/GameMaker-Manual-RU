mp_grid_create

С помощью этой функции можно создать mp_grid для функций планирования движения. Она возвращает индекс, который должен быть сохранен в переменной для использования во всех других вызовах функций MP-сетки. Координаты x и y указывают на положение левого верхнего угла сетки, hcells и vcells - на количество горизонтальных и вертикальных ячеек, которые будут созданы в сетке, а ширина и высота ячеек - на размер ячеек в пикселях. Вы можете создать и поддерживать несколько структур сетки в один момент времени, если они нужны вам для разных вещей, но имейте в виду, что каждая сетка будет требовать памяти, и чем больше ячеек, тем больше памяти она потребует. Также следует учитывать, что функции MP достаточно требовательны к процессору, поэтому чем чаще вы их вызываете и чем выше разрешение ячеек, тем больше вероятность падения производительности.

Синтаксис:

mp_grid_create(left, top, hcells, vcells, cellwidth, cellheight);

Аргумент Тип Описание
left Реально Starting x coordinate of the mp_grid in the room
top Реально Starting y coordinate of the mp_grid in the room
hcells Реально Number of horizontal cells that the mp_grid will contain
vcells Реально Number of vertical cells that the mp_grid will contain
cellwidth Реально The width (in pixels) of each individual cell of the mp_grid
cellheight Реально The height (in pixels) of each individual cell of the mp_grid

Возвращается:

MP Grid ID

Пример:

global.grid = mp_grid_create(0, 0, room_width / 32, room_height / 32, 32, 32);

Приведенный выше код создает глобальную переменную "grid", затем генерирует mp_grid и присваивает ее индекс (id) этой переменной для использования во всех дальнейших вызовах функции mp_grid. Сетка покрывает комнату с разрешением ячеек 32x32 пикселя. Это означает, что, например, в комнате размером 640x480 сетка будет содержать 300 ячеек: 20 горизонтальных ячеек (640 / 32), умноженные на 15 вертикальных (480 /32), дают 300 ячеек.
