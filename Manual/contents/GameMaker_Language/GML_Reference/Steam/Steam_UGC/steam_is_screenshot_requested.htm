steam_is_screenshot_requested

Эта функция опрашивает API Steam, чтобы узнать, была ли нажата клавиша для создания скриншота игры. Функция возвращает true только на один шаг (игровой тик), когда клавиша нажата, и возвращает false во всех остальных случаях.

Обратите внимание, что если клавиша скриншота нажата, эта функция вернет true только один раз для каждого шага, когда она нажата, и вернет false для всех последующих вызовов в пределах одного шага. Например, если скриншот запрашивается в текущем кадре, и вы вызываете эту функцию в событии Step, чтобы узнать это, вы получите true; однако если вы вызовете ее снова в Draw GUI, чтобы проверить, был ли запрошен скриншот, вы получите false, поскольку функция уже была "использована" в событии Step. Чтобы использовать возвращаемое значение функции несколько раз в одном кадре, рекомендуется хранить его в переменной и считывать вместо повторного вызова функции.

ПРИМЕЧАНИЕ: Эта функция не делает снимок экрана. Она только сигнализирует о том, что клавиша была нажата, и вы должны использовать функции GameMaker Studio 2 экран_сохранить() или screen_save_part() для сохранения локальной копии загружаемого файла.

Синтаксис:

steam_is_screenshot_requested();

Возвращается:

Boolean

Пример:

if steam_is_screenshot_requested()
{
    var file = "Catch_The_Haggis_" + string(global.scrn_num) + ".png");
    screen_save(file)
    steam_send_screenshot(file, window_get_width(), window_get_height());
    global.scrn_num += 1;
}

Приведенный выше код будет опрашивать API Steam на предмет запроса скриншота, и если он был, то будет сгенерировано уникальное имя для файла изображения, сделан скриншот, и файл будет отправлен на страницу сообщества Steam для пользователя.
