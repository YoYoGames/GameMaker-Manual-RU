fps_real

В GameMaker есть два основных способа, которые можно использовать для определения скорости, с которой работает ваша игра. Это game speed (как указано в Опциях игры) и fps (кадров в секунду). Эти значения часто путают, но, по сути, одно из них - это количество игровых шагов, которые GameMaker должен выполнить за секунду (скорость игры), а другое - количество шагов процессора, которые GameMaker фактически выполняет за секунду (реальная частота кадров в секунду), и это значение обычно намного выше, чем скорость игры, но будет снижаться по мере усложнения игры и использования большей вычислительной мощности для поддержания заданной комнатной скорости.

Эта переменная, доступная только для чтения, возвращает текущий fps в виде целочисленного значения. Обратите внимание, что функция будет обновляться только один раз на каждом шаге вашей игры, поэтому может показаться, что значение "прыгает" от одного к другому, но это вполне нормально.

ПРИМЕЧАНИЕ: В HTML5-цели эта переменная будет возвращать значение, кратное частоте обновления монитора, поскольку GameMaker должен полагаться на браузер для определения времени и отправки.

Синтаксис:

fps_real

Возвращается:

Пример:

if debug_mode
{
    draw_text(32, 32, "FPS = " + string(fps_real));
}

Приведенный выше код проверит, находится ли игра в режиме отладки, и если да, то отобразит на экране текущий реальный fps.
