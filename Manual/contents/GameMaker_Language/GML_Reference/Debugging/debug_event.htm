отладочное_событие

Эта функция генерирует пользовательское отладочное событие, которое будет отображаться в графическом представлении отладчика, когда игра выполняется в режиме отладки. Если вам нужно, чтобы сообщения отображались не в режиме отладки, используйте функцию show_debug_message().

Функция также принимает две зарезервированные строки, чтобы помочь выполнить отладку с помощью внешних приложений, таких как Visual Studio. Этими строками являются:

  • "OutputDebugOn" - Включает вызов OutputDebugString для цели Windows, что означает, что весь отладочный вывод - все, что вы видите в окне Output - может быть просмотрен Visual Studio или сторонними приложениями.
  • "OutputDebugOff" - Отключает поведение, описанное выше.
  • "BreakOnError" - Эта опция предназначена только для сборок Windows YYC и означает, что проекты не будут отображать обычный экран ошибок кода, если среда выполнения обнаружит ошибку, а вместо этого просто продолжат работу и потерпят крах. Это позволяет вам видеть трассировку стека внутри Visual Studio при отладке.
  • "ResourceCounts" - Перечисляет все активные в данный момент ресурсы, такие как структуры данных, источники времени, поверхности и т.д.
  • "DumpMemory" - Предоставляет информацию о текущем использовании памяти.

Синтаксис:

debug_event(string)

Аргумент Тип Описание
string Строка The custom debug event string to use.

Возвращается:

N/A

Пример:

if !surface_exists(global.EffectsSurface)
{
    debug_event("Recreating Effects Surface");
    global.EffectsSurface = surface_create(room_width, room_height);
}

Приведенный выше код проверяет, существует ли поверхность, и если нет, то в графическом представлении отладчика запускается событие отладки (игра должна быть запущена в режиме отладки, чтобы это было видно) и поверхность создается заново.
