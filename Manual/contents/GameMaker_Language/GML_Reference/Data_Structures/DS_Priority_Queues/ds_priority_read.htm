ds_priority_read

С помощью этой функции вы можете воссоздать сохраненный приоритет DS (тот, который ранее был записан в виде строки с помощью функции ds_priority_write()). Вы должны сначала создать новый приоритет DS для чтения строки, и если приоритет DS уже существует и в нем сохранена информация, то она будет очищена перед чтением. Эта функция имеет жизненно важное значение при создании механизмов сохранения/загрузки для вашей игры.

Обратите внимание, что если указанная строка DS была написана в среде GameMaker: Studio 1.2.x (или старше), следует указать дополнительный аргумент "legacy", установив его в значение true, поскольку после этого формат строки изменился.

Синтаксис:

ds_priority_read(id, str, [legacy]);

Аргумент Тип Описание
id Идентификатор приоритета DS The id of the data structure to write.
str Строка The string to read from.
legacy Булево OPTIONAL Может быть либо true, либо false, либо полностью отсутствовать.

Возвращается:

N/A

Пример:

p_queue = ds_priority_create();
ini_open("save.ini");
var str = ini_read_string("P_Queues", "0", "");
if (str != "")
{
    ds_priority_read(p_queue, str);
}
ini_close();

Приведенный выше код создает приоритетную очередь и сохраняет индекс в переменной "p_queue". Затем он открывает файл ini и считывает из него строку, проверяя, что сначала строка не возвращается пустой. Затем эта строка считывается во вновь созданный приоритет DS.
