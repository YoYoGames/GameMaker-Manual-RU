ds_queue_read

С помощью этой функции можно воссоздать сохраненную очередь DS (ту, которая ранее была записана в виде строки с помощью функции ds_queue_write()). Сначала вы должны создать новую очередь DS для чтения строки, и если очередь DS уже существует и в ней сохранена информация, то она будет очищена перед чтением. Эта функция имеет жизненно важное значение при создании механизмов сохранения/загрузки для вашей игры.

Обратите внимание, что если указанная строка DS была написана в среде GameMaker: Studio 1.2.x (или старше), следует указать дополнительный аргумент "legacy", установив его в значение true, поскольку после этого формат строки изменился.

Синтаксис:

ds_queue_read(id, str [, legacy]);

Аргумент Тип Описание
id Идентификатор очереди DS The id of the data structure to read into.
str Строка The string to read from.
legacy Булево OPTIONAL Может быть либо true, либо false, либо полностью опущен.

Возвращается:

N/A

Пример:

queue = ds_queue_create();
ini_open("save.ini");
var str = ini_read_string("Queues", "0", "");
if (str != "")
{
    ds_queue_read(queue, str);
}
ini_close();

Приведенный выше код создает очередь и сохраняет индекс в переменной "queue". Затем он открывает файл ini и считывает из него строку, проверяя, что сначала строка не возвращается пустой. Затем эта строка считывается в только что созданную очередь DS.
