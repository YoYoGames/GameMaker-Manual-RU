ds_set_precision

При сравнении значений, например, при поиске на карте или сортировке списка, GameMaker должен решить, когда два значения равны. Для строк и целых чисел это понятно, но для реальных чисел из-за ошибок округления с плавающей запятой кажущиеся равными числа могут легко стать неравными. Например, возможно, что (5 / 3) * 3 не будет равно 5! Чтобы избежать этого, во всех функциях работы с вещественными числами используется значение точности, и когда разница между двумя числами меньше этой точности, они считаются равными. По умолчанию для всех функций структур данных используется точность 0.0000001, если она не изменена этой функцией.

ПРИМЕЧАНИЕ: Эта точность используется во всех структурах данных, но не в других сравнениях в GML!

Синтаксис:

ds_set_precision(prec);

Аргумент Тип Описание
prec Реально The precision value (default 0.0000001)

Возвращается:

N/A

Пример:

ds_set_precision(0.0001);

Приведенный выше код изменит настройку точности по умолчанию для всех функций структур данных.
