тип_части_спрайта

Эта функция может быть использована для установки типа частиц, чтобы использовать пользовательский sprite из игровых активов. Вы можете выбрать sprite и затем указать GameMaker использовать sprite и анимировать его или нет, а если вы выберете анимированный, то вы также можете выбрать, чтобы под-изображения менялись с интервалом так, чтобы анимация начиналась и заканчивалась в начале и конце времени жизни каждой созданной частицы (таким образом, если у вас время жизни установлено на 15 шагов минимум и 30 шагов максимум, независимо от того, какое значение имеет время жизни, анимация будет сделана в соответствии с ним, причем она будет работать быстрее для более коротких времен жизни и медленнее для более длинных). Вы также можете выбрать случайное суб-изображение, так что если sprite не анимирован, то для создания sprite будет случайно выбрано другое изображение, или если частица анимирована, то анимация начнется в случайной точке.

Синтаксис:

part_type_sprite(ind, sprite, animate, stretch, random);

Аргумент Тип Описание
ind Идентификатор типа частицы The index of the particle type to change.
sprite Актив спрайта The index of the sprite to set the particle type to.
animate Булево Whether to follow the sprite's animation normally (true) or not (false).
stretch Булево Whether to stretch the sprite's animation to match the particle lifespan (true) or not (false).
random Булево Whether to choose a random sub-image (true) or not (false).

Возвращается:

N/A

Пример:

global.p1 = part_type_create();
part_type_sprite(global.p1 , spr_Coins, 1, 0, 0);
part_type_size(global.p1, 1, 3, 0, 0);
part_type_scale(global.p1, 1, 1);
part_type_colour1(global.p1, c_white);
part_type_alpha2(global.p1, 1, 0);
part_type_speed(global.p1, 2, 4, 0, 0);
part_type_direction(global.p1, 0, 180, 0, 0);
part_type_gravity(global.p1, 0.20, 270);
part_type_orientation(global.p1, 0, 0, 0, 0, 1);
part_type_blend(global.p1, 1);
part_type_life(global.p1, 15, 60);

Приведенный выше код устанавливает тип частицы, указанный в глобальной переменной "p1", на форму sprite, указанную в "spr_Coins", а затем устанавливает различные другие свойства частицы.
