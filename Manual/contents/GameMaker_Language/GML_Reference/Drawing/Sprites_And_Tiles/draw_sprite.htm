рисовать_спрайт

Эта функция рисует заданное sprite и суб-изображение в позиции внутри игровой комнаты. Для sprite вы можете использовать переменную экземпляра sprite_index чтобы получить текущий sprite, который назначен экземпляру, выполняющему код, или вы можете использовать любой другой актив sprite. То же самое относится и к суб-изображению, поскольку оно также может быть задано переменной экземпляра image_index которая установит субизображение, выбранное для текущего экземпляра sprite (обратите внимание, что вы можете нарисовать другой sprite и все равно использовать значение субизображения для текущего экземпляра), или вы можете использовать любое другое значение для этого, чтобы нарисовать конкретное субизображение выбранного sprite. Если значение больше, чем количество субизображений, то GameMaker будет автоматически перебирать номера для выбора соответствующего изображения (например, если рисуемый sprite имеет 5 субизображений, пронумерованных от 0 до 4, и мы установили значение индекса равным 7, то функция нарисует субизображение 3, пронумерованное 0). Наконец, позиция x и y - это положение в комнате, в котором будет рисоваться sprite, и оно центрируется на смещении sprite по x и y.

ПРИМЕЧАНИЕ Эта функция может работать не так, как ожидается, при использовании скелетной анимации sprites, и вы можете обнаружить, что функция рисует только первый кадр позы по умолчанию. Вместо нее следует использовать функции draw_skeleton_*.

Синтаксис:

draw_sprite(sprite, subimg, x, y);

Аргумент Тип Описание
sprite Актив спрайта The index of the sprite to draw.
subimg Реально The sub-image (frame) of the sprite to draw (image_index or -1 correlate to the current frame of animation in the object).
x Реально The x coordinate of where to draw the sprite.
y Реально The y coordinate of where to draw the sprite.

Возвращается:

N/A

Пример:

draw_sprite(sprite_index, image_index, x, y);
draw_sprite(spr_Halo, 0, x, y-32);

Это нарисует экземпляр, назначенный sprite (sprite_index) с текущим суб-изображением в позиции x и y экземпляра в комнате. Затем будет нарисован первый кадр sprite с индексом "spr_Halo" в том же положении x и y, но на 32 пикселя "выше".
