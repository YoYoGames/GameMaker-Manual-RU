рисовать_путь

С помощью этой функции вы можете заставить GameMaker нарисовать путь на экране. Путь будет нарисован в виде простой линии и может быть как относительно вызывающего экземпляра, так и в абсолютной позиции, в которой он был создан в редакторе путей или в коде. Эта функция чрезвычайно полезна при отладке динамических путей (например, созданных для экземпляров с помощью функции mp_grid_path() функцией).

ПРИМЕЧАНИЕ: Если вы хотите нарисовать фигуру с помощью шейдера, вы должны знать, что большинство шейдеров ожидают следующих входных данных: вершина, текстура, цвет. Однако при использовании этой функции передаются только данные вершин и цвета, поэтому шейдер может ничего не нарисовать (или нарисовать, но неправильно). Если вам нужно рисовать фигуры таким образом, то шейдер должен быть настроен с учетом этого.

Синтаксис:

draw_path(path, x, y, absolute);

АргументТип Описание
path The path to draw
x The x coordinate of where the path is drawn
y The y coordinate of where the path is drawn
absolute Whether the path is drawn at the absolute position (true) или относительное положение (false)

Возвращается:

Пример:

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
    draw_path(path, x, y, false);
}

Приведенный выше код использует функцию mp_grid_path для создания пути и сохраняет его в переменной "path". Если путь успешно создан, то он рисуется на экране в позиции относительно экземпляра, выполняющего код.
