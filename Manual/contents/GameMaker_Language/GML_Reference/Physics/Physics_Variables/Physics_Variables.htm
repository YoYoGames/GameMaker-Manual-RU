Физические переменные

Как и в "традиционной" системе столкновений и движения, в системе физики также есть ряд переменных экземпляра, назначаемых каждому экземпляру объекта при его создании. Некоторые из этих свойств могут быть определены пользователем в GML, но многие из них доступны только для чтения, а многие применимы только в событии столкновения.

Следующие переменные существуют во всех объектах с поддержкой физики, и их можно читать и записывать (т.е. изменять) с помощью кода из экземпляров и скриптовых функций:

  • phy_active
  • phy_angular_velocity
  • phy_angular_damping
  • phy_linear_velocity_x
  • phy_linear_velocity_y
  • phy_linear_damping
  • phy_speed_x
  • phy_speed_y
  • phy_position_x
  • phy_position_y
  • phy_position_xprevious
  • phy_position_yprevious
  • phy_rotation
  • phy_fixed_rotation
  • phy_bullet

Некоторые переменные являются переменными только для чтения, что означает, что информация, которую они содержат, не может быть изменена с помощью кода экземпляра или скриптовых функций (но она может быть изменена внутри GameMaker, поэтому они являются переменными, а не константами). В следующем списке представлены переменные, доступные только для чтения, которые существуют во всех объектах с поддержкой физики:

  • phy_speed
  • phy_com_x
  • phy_com_y
  • phy_dynamic
  • phy_kinematic
  • phy_inertia
  • phy_mass
  • phy_sleeping

Наконец, существует ряд временных переменных, которые существуют в объектах с поддержкой физики и создаются только в событии столкновения. Эти переменные подобны перечисленным выше и доступны только для чтения:

  • phy_collision_points
  • phy_collision_x
  • phy_collision_y
  • phy_col_normal_x
  • phy_col_normal_y
