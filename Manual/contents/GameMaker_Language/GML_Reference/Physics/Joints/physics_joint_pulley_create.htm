physics_joint_pulley_create

Шарнир шкива используется для соединения двух экземпляров в физическом мире таким образом, что перемещение одного из них непосредственно влияет на движение другого. Эти шарниры сначала закрепляются в мировом пространстве в двух точках, каждая из которых связана с экземпляром в его центре масс. Это можно изменить, задав локальные координаты привязки x/y относительно начала координат экземпляра, что означает, что фактическое соединение экземпляра с точкой привязки физического мира может быть смещено куда-то, кроме его начала координат. Следующее изображение иллюстрирует это:

Если вы посмотрите на inst2, то увидите, что его локальная точка привязки была смещена на 20 пикселей влево от начала координат (Помните! Мировые точки привязки определяются с помощью координат комнаты, а локальные точки привязки определяются относительно начала координат экземпляра). Вы также можете задать коэффициент для шарнира шкива, который указывает одной или другой стороне двигаться быстрее: значение меньше 1 заставляет первый экземпляр двигаться быстрее (например, 0,5 заставит его двигаться вдвое быстрее), а значение больше 1 заставляет второй экземпляр двигаться быстрее (например, 2 заставит его двигаться вдвое быстрее).

Как и со всеми суставами, если вы установите значение "col" true, то два экземпляра могут взаимодействовать и сталкиваться друг с другом, но только если у них есть события столкновения, однако если установлено значение false, то они не будут сталкиваться независимо ни от чего.

Синтаксис:

physics_joint_pulley_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, l_anchor1_x, l_anchor1_y, l_anchor2_x, l_anchor2_y, ratio, col)

Аргумент Тип Описание
inst1 Идентификатор экземпляра The first instance to connect with the joint
inst2 Идентификатор экземпляра The second instance to connect with the joint
w_anchor1_x Реально The x coordinate where the first part of joint is anchored, within the game world
w_anchor1_y Реально The y coordinate where the first part of the joint is anchored, within the game world
w_anchor2_x Реально The x coordinate where the second part of the joint is anchored, within the game world
w_anchor2_y Реально The y coordinate where the second part of the joint is anchored, within the game world
l_anchor1_x Реально The local x coordinate where joint 1 is anchored to the first instance
l_anchor1_y Реально The local y coordinate where joint 1 is anchored to the first instance
l_anchor2_x Реально The local x coordinate where joint 2 is anchored to the second instance
l_anchor2_y Реально The local y coordinate where joint 2 is anchored to the second instance
ratio Реально Set the velocity ratio between two instances
col Булево Whether the two instances can collide (true) or not (false)

Возвращается:

Physics Joint ID

Пример:

var t_fix, inst1, inst2;
t_fix = physics_fixture_create();
physics_fixture_set_circle_shape(t_fix, 20);
inst1 = instance_create_layer(150, room_height - 90, "Instances", obj_Block);
inst2 = instance_create_layer(300, room_height - 90, "Instances", obj_block);
physics_fixture_bind(t_fix, inst1);
physics_fixture_bind(t_fix, inst2);
physics_joint_pulley_create(inst1, inst2, 150, room_height - 140, 300, room_height - 140, 0, 0, 0, 0, 2, 0);
physics_fixture_delete(t_fix);

Приведенный выше код создает и определяет новое приспособление, создает два экземпляра "obj_Block", а затем связывает это приспособление с ними. Затем определяется соединение шкива между этими экземплярами, без смещения для соединений, с соотношением 2:1 (это означает, что экземпляр 2 будет двигаться быстрее). Никаких столкновений между двумя экземплярами в шкиве не происходит.
