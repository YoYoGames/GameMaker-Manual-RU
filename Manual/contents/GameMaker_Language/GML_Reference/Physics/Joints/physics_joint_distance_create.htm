физическое_расстояние_создать

Одним из самых простых соединений является соединение расстояний, которое гласит, что расстояние между двумя точками на двух экземплярах должно быть постоянным. При задании соединения расстояния два экземпляра уже должны быть созданы и иметь назначенное крепление, затем определяются две опорные точки в комнатных координатах. Первая опорная точка связана с экземпляром 1, вторая опорная точка связана с экземпляром 2, а расстояние между этими точками подразумевает длину ограничения расстояния. На рисунке ниже показано, как это работает:

Как видно, точки привязки задаются в виде координат помещения, поэтому при их определении необходимо соблюдать осторожность, особенно если экземпляры создаются одновременно с суставами, а не размещаются в помещении через редактор помещений. Также следует понимать, что суставы создаются независимо от размера sprite экземпляров или прикрепленных к ним приспособлений. Поэтому, если вы создадите дистанционный шарнир не в начале координат экземпляра, он все равно будет действовать и ограничивать два экземпляра относительно позиции, в которой он был создан. Если установить значение "col" на true, то два экземпляра могут взаимодействовать и сталкиваться друг с другом, но только если у них есть события столкновения, однако если установить значение false, то они не будут сталкиваться независимо ни от чего.

Синтаксис:

physics_joint_distance_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, col)

Аргумент Тип Описание
inst1 Идентификатор экземпляра The first instance to connect with the joint
inst2 Идентификатор экземпляра The second instance to connect with the joint
w_anchor1_x Реально The first x coordinate for the joint, within the game world
w_anchor1_y Реально The first y coordinate for the joint, within the game world
w_anchor2_x Реально The second x coordinate for the joint, within the game world
w_anchor2_y Реально the second y coordinate for the joint, within the game world
col Булево Whether the two instances can collide (true) or not (false)

Возвращается:

Physics Joint ID

Пример:

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id=instance_create_layer(x+300, y, "Instances", obj_Rudder);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_distance_create(id, o_id, x + 50, y, o_id.x - 50, o_id.y, 0);
physics_fixture_delete(mainFixture);

Приведенный выше код создает и определяет новое приспособление, а затем создает экземпляр "obj_Rudder". Затем приспособление назначается экземпляру, выполняющему код, а также вновь созданному экземпляру, и между ними создается соединение. Наконец, приспособление удаляется, поскольку оно больше не нужно.
