буферная_загрузка

Эта функция используется для загрузки буфера, который был ранее сохранен с помощью buffer_save() функции, а также любые Включенные файлы или файлы, загруженные извне. Она вернет идентификатор нового буфера, который создается функцией как "растущий" буфер (см. здесь) с выравниванием по байтам 1. Этот идентификатор должен храниться в переменной и использоваться во всех последующих вызовах функций к этому буферу. Если по какой-либо причине загрузка не удалась, функция вернет -1, а в окне вывода компилятора появится сообщение о том, что загрузка не удалась.

ПРИМЕЧАНИЕ Важно, чтобы вы удаляли любые динамически созданные ресурсы, подобные этому, из памяти, когда они вам больше не нужны, чтобы предотвратить утечку памяти, поэтому, когда вы закончите работу с созданным буфером, вы должны освободить его снова, используя buffer_delete().

Примечания для конкретной платформы

=======

buffer_load

This function loads a buffer that was previously saved using the buffer_save functions, as well as any Included Files or other files stored externally.

It returns a new buffer which is created by the function as a "grow" buffer (see here) with a byte alignment of 1. The buffer should be stored in a variable and used in all further function calls to this buffer. If the load fails for whatever reason, the function will return -1 and a message will be shown in the compiler output window saying that the load has failed.

  • На HTML5 любые буферы, загруженные из локального хранилища, будут декодированы с использованием base64 (поскольку они сохраняются как строки base64), однако использование этой функции для Included Files или любых внешних файлов не будет использовать декодирование base64, поскольку не ожидается, что они будут в формате base64.
