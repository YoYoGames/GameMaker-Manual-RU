buffer_async_group_end

Эта функция завершает определение асинхронной группы буферов. Вы должны предварительно вызвать функцию buffer_async_group_begin() для инициирования группы, а затем вызвать функцию buffer_save_async() для каждого файла, который вы хотите сохранить (или buffer_load_async() для загрузки буферов). Наконец, вы вызываете эту функцию, которая начнет сохранение файлов. Функция вернет уникальное значение ID для сохранения, которое затем может быть использовано в событии Asynchronous Save / Load для разбора результатов из async_load карты DS.

Синтаксис:

buffer_async_group_end();

Возвращается:

Async Request ID

Функция buffer_async_group_end() может быть вызвана из любого события, и поскольку она асинхронна, обратный вызов может быть почти мгновенным или занять несколько секунд. Вызов функции прост и выглядит следующим образом:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
save_id = buffer_async_group_end();

Приведенный выше код запускает буферную группу, затем устанавливает ее для асинхронного сохранения 4 файлов. Затем определение группы завершается (в этот момент начнется сохранение), сохраняя идентификатор вызова функции в переменной "save_id". Когда сохранение будет завершено, сработает событие асинхронного сохранения/загрузки, и вы сможете разобрать карту async_load на предмет правильного ID функции, как показано ниже:

if ds_map_find_value(async_load, "id") == saveid
{
    if ds_map_find_value(async_load, "status") == false
    {
        show_debug_message("Save failed!");
    }
}

Приведенный выше код сначала проверит id созданной карты DS, затем проверит статус обратного вызова, опубликовав отладочное сообщение, если возникли какие-либо проблемы.
