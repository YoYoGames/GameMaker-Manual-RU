buffer_async_group_end

Эта функция завершает определение асинхронной группы буферов. Вы должны предварительно вызвать функцию buffer_async_group_begin() для инициирования группы, а затем вызвать функцию buffer_save_async() для каждого файла, который вы хотите сохранить (или buffer_load_async() для загрузки буферов). Наконец, вы вызываете эту функцию, которая начнет сохранение файлов. Функция вернет уникальное значение ID для сохранения, которое затем может быть использовано в событии Asynchronous Save / Load для разбора результатов из async_load карты DS.

Синтаксис:

buffer_async_group_end();

Возвращается:

Async Request ID

Расширенный пример:

Функция buffer_async_group_end() может быть вызвана из любого события, и поскольку она асинхронна, обратный вызов может быть почти мгновенным или занять несколько секунд. Вызов функции прост и выглядит следующим образом:

=======

buffer_async_group_end

This function finishes the definition of a buffer async group.

You must have previously called the function buffer_async_group_begin to initiate the group, then call the function buffer_save_async for each file that you wish to save out (or buffer_load_async to load buffers). Finally you call this function, which will start the saving of the files. The function will return a unique ID value for the save, which can then be used in the Asynchronous Save / Load event to parse the results from the async_load DS map.

 

Syntax:

buffer_async_group_end();

 

Returns:

Async Request ID

 

Example:

The buffer_async_group_end function can be called from any event, and since it is asynchronous the callback can be almost instantaneous or could take several seconds. Calling the function is simple and would look something like this:
