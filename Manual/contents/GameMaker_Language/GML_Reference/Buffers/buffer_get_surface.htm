buffer_get_surface

С помощью этой функции вы можете записать информацию с поверхности в заданный буфер. Буфер должен быть создан ранее и представлять собой выровненный по 1 байту буфер, достаточно большой, чтобы хранить данные для поверхности, которую вы собираетесь записать (если не уверены, используйте растущий буфер или смотрите пример внизу этой страницы).

Информация для поверхности всегда будет записываться в начало буфера (независимо от позиции поиска), но вы можете выбрать значение смещения (в байтах) для перемещения позиции записи от начала буфера.

Обратите внимание, что функция записывает каждый пиксель поверхности в буфер, используя форматирование RGBA на цели Windows, но на других целях оно может быть другим в зависимости от ОС или даже устройства.

ПРИМЕЧАНИЕ: Эта функция изменилась начиная с обновления GameMaker 2.3.1 и далее. В предыдущих версиях использовалась константа buffer_surface_copy, которая теперь устарела.

Синтаксис:

buffer_get_surface(buffer, surface, offset);

=======

buffer_get_surface

This function writes information from a surface to a given buffer.

The buffer must have been created previously and should be a 1-byte aligned buffer large enough to store data for the surface you are going to write (if unsure, use a grow buffer or see the example at the bottom of this page).

The information for the surface will always be written to the start of the buffer (regardless of the seek position) but you can choose an offset value (in bytes) to move the write position from the start of the buffer.

The format of the surface will affect how the surface is written into the buffer. Keep in mind that the same format may be written in a different way on different platforms.

 

Syntax:

buffer_get_surface(buffer, surface, offset);
