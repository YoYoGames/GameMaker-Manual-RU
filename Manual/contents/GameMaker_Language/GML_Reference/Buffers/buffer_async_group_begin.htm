buffer_async_group_begin

Эта функция вызывается, когда вы хотите начать сохранение нескольких буферов в несколько файлов (или загрузку нескольких буферов). Имя группы - это строка, которая будет использоваться в качестве имени каталога, куда сохраняются файлы, и должна использоваться как часть пути к файлу при последующей загрузке файлов обратно в IDE (используя любую из buffer_load() функции). Эта функция предназначена только для использования с функциями buffer_save_async() и buffer_load_async(), и вы также должны завершить асинхронную группу вызовом функции buffer_async_group_end() иначе файлы не будут сохранены/загружены.

Обратите внимание, что для консольных платформ (например, PS4) название группы будет использоваться в качестве описания слота сохранения, и использование этой функции поможет вам избежать отображения пользовательского интерфейса для каждого сохраняемого файла. Также обратите внимание, что при использовании UWP вы можете перемещать сохраненные игры между родными окнами Xbox и Microsoft store, создав группу, которая будет такой же, как группа ERA Xbox, например:

if os_type == os_uwp
{
    buffer_async_group_begin("root/yourgroupname");
}
else
{
    buffer_async_group_begin("yourgroupname");
}

Синтаксис:

buffer_async_group_begin(groupname);

buffer_async_group_begin

This function is called when you want to begin the saving out of multiple buffers to multiple files (or loading multiple buffers).

The groupname is a string and will be used as the directory name for where the files are saved, and should be used as part of the file path when loading the files back into the IDE later (using any of the buffer_load functions). This function is only for use with the buffer_save_async and buffer_load_async functions and you must also end the async group by calling buffer_async_group_end function otherwise the files will not be saved/loaded.

 For the console platforms (like PS4 for example), the groupname will be used as the save slot description, and using this function can help you avoid having the UI show for every file that is being saved out.

 

Syntax:

buffer_async_group_begin(groupname);
