Функции чисел

Вещественные числа в GameMaker считаются числами с плавающей точкой двойной точности - то есть они имеют десятичную точку без фиксированного количества цифр до или после точки - или целыми числами - то есть это целые числа без десятичной точки. Например, 2 - это целое число, а 2,01 - вещественное число с плавающей точкой.

ПРИМЕЧАНИЕ: В цели HTML5 все действительные числа являются двойными.

Это различие между целыми и плавающими числами очень важно при кроссплатформенной разработке, так как точность вычислений, выполненных на компьютере Windows, отличается от точности тех же вычислений на мобильном устройстве. Это означает, что вы должны быть особенно внимательны при выполнении сравнений, например:

if (image_index == 1)
{
    // Do something...
}

В приведенном выше примере, если мы устанавливали image_speed на 0,25 - например, - то после четырех шагов вы можете предположить, что значение image_index будет равно 1, но это может быть значение 1.0000002 из-за того, как работает математика с плавающей запятой, и поэтому оценка не будет true. Такие ошибки могут быть довольно трудно отлаживать, поэтому всегда полезно знать о них и планировать заранее и использовать другие средства проверки значений или использовать соответствующие функции перекрытия или округления (перечисленные ниже) для преобразования проверяемого числа в целое (более подробную информацию о математике с плавающей точкой и о том, почему это является проблемой, смотрите здесь). Например, приведенный выше код можно записать как:

if (floor(image_index) == 1)
{
    // Do something...
}

Также стоит отметить, что при использовании целей YoYo Compiler все выражения и функции оцениваются слева направо, в то время как на всех других целевых платформах они оцениваются справа налево, что означает, что, например, это - даст разные результаты в зависимости от платформы:

val = max(num, ++num, num++);

ПРИМЕЧАНИЕ: Для получения дополнительной информации см. раздел " Порядок оценки".

Вы также можете использовать специальную функцию, доступную в GameMaker, чтобы установить значение эпсилон для математических вычислений с плавающей точкой. Когда вещественное число округляется до ближайшего числа с плавающей точкой, эпсилон (также известный как "машинный эпсилон") формирует верхнюю границу относительной ошибки, и вы можете получить и установить значение эпсилона с помощью следующих функций:

  • math_set_epsilon
  • math_get_epsilon

Все эти функции связаны с использованием случайных чисел и значений:

ПРИМЕЧАНИЕ При использовании случайных функций GameMaker сохраняет одно и то же случайное зерно при каждом запуске игры. Это значительно облегчает отладку, так как гарантируется, что функции random всегда будут возвращать одно и то же значение, однако если вы не хотите, чтобы это происходило, вам необходимо в самом начале игры установить новое случайное зерно, используя либо функцию randomise() или random_set_seed().

  • choose
  • random
  • random_range
  • irandom
  • irandom_range
  • random_set_seed
  • random_get_seed
  • randomise

Это все функции, которые округляют значения каким-либо образом или выбирают одно значение из различных заданных значений:

  • abs
