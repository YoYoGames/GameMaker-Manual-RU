Диалог

Большинству современных компьютеров и мобильных устройств не нравится, когда игра останавливается при отображении диалогового окна. Они склонны интерпретировать это как остановку бегущей строки игры, и могут посчитать это ошибкой и преждевременно закрыть приложение. Для предотвращения этого в GameMaker предусмотрены асинхронные диалоговые функции, которые позволяют запрашивать определенные формы обратной связи или ввода данных от пользователя, но без блокировки программы запуска игры в любой момент, то есть при их вызове игра будет работать в обычном фоновом режиме, пока пользователь отвечает в показанном диалоговом окне. Как только пользователь ответит, будет вызвано асинхронное событие Dialog, чтобы вы могли обработать результаты.

ПРИМЕЧАНИЕ: За исключением shop_leave_rating()эти функции предназначены только для отладки и тестированияи не должны использоваться в выпущенных играх. Для этих целей вы должны создать пользовательский интерфейс для получения ввода от игроков, чтобы иметь полный контроль над тем, как выглядят и ведут себя диалоги.

Ниже приведен список всех доступных функций асинхронного диалога:

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
