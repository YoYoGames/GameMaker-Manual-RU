show_message_async

Эта функция открывает окно и показывает пользователю сообщение, которое вы задаете в функции. Это асинхронная функция, поэтому GameMaker не блокирует устройство, на котором она выполняется, в ожидании ответа, а продолжает выполнять события в обычном режиме. Как только пользователь нажимает кнопку "Ок", запускается асинхронное событие Диалог запускается событие, которое только на время этого события будет иметь ds_map, хранящийся в переменной async_load.

Эта карта будет содержать два ключа, "id" и "status", причем "id" будет значением, которое было возвращено функцией при вызове, а "status" будет либо true для нажатия кнопки "Ок", либо false, если сообщение было отменено (где это возможно, поскольку не все целевые платформы позволяют отменять диалоги).

ПРИМЕЧАНИЕ Эта функция предназначена только для отладки и тестированияи не должна использоваться в выпущенных играх. Для этих целей вы должны создать пользовательский интерфейс для получения ввода от игроков, чтобы иметь полный контроль над тем, как выглядят и ведут себя диалоги.

Синтаксис:

show_message_async(string);

Аргумент Тип Описание
string Строка The message to show to the user.

Возвращается:

Async Request ID

Расширенный пример:

Событие нажатия левой кнопки мыши (например) объекта, который показывает сообщение, будет иметь следующий код:

msg = show_message_async("You will now be taken to the store");

Приведенный выше пример покажет сообщение с заданной строкой. Идентификатор сообщения хранится в переменной "msg" и будет использован в асинхронном событии Dialogs, как показано ниже:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        url_open("https://play.google.com/store");
    }
}

Приведенный выше код сверяет ключ "id" возвращаемого DS Map со значением, хранящимся в переменной "msg". Если они совпадают, то проверяется, была ли нажата кнопка "Ок" (а не закрыто/отменено окно), и если возвращается true, то открывается url.
