display_set_timing_method

Эта функция может использоваться для установки метода синхронизации, который будет использоваться для рендеринга вашей игры. Метод может быть одной из следующих констант:

Постоянная Описание
tm_sleep Используйте значение запаса сна в качестве основного метода синхронизации
tm_countvsyncs Использовать синхронизацию vsync в качестве основного метода синхронизации (по умолчанию для всех поддерживаемых платформ)

Метод синхронизации vsync использует поддержку вертикальной синхронизации целевой платформой для обеспечения привязки для расчетов времени рендеринга в игре, в то время как установка режима sleep margin будет просто пытаться обеспечить, чтобы каждый кадр длился нужное количество времени (скажем, 1/30 или 1/60 секунды) путем ожидания или сна. В целом, стандартный режим vsync дает наиболее плавные результаты, но обратите внимание, что при использовании метода vsync значения маржи сна все еще важны, хотя их влияние уменьшается, и мы рекомендуем оставить значение по умолчанию при использовании этого метода.

По умолчанию на всех платформах, кроме PS4, Ubuntu и HTML5, GameMaker будет использовать метод синхронизации vsync, в то время как на неподдерживаемых платформах может использоваться только синхронизация sleep margin.

Если вы хотите установить предел сна, вы можете сделать это с помощью функции display_set_sleep_margin() и вы можете узнать, какой метод синхронизации используется в данный момент, с помощью функции display_get_timing_method().

Синтаксис:

display_set_timing_method(method);

Аргумент Тип Описание
method Метод синхронизации дисплея Постоянный The timing method to use (see the list of constants, above)

Возвращается:

N/A

Пример:

if display_get_timing_method() != tm_sleep
{
    display_set_timing_method(tm_sleep);
    if display_get_sleep_margin() != 20
    {
        display_set_sleep_margin(20);
    }
}

Приведенный выше код проверит метод синхронизации, и если он не установлен на tm_sleep, то он будет установлен на это значение, а запас сна будет установлен на 20.
