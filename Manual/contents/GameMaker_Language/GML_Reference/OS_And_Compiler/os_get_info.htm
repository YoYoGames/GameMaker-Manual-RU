os_get_info

Эта функция возвращает DS Map с подробной информацией об ОС, на которой запущена игра. Точная возвращаемая информация будет зависеть от ОС и устройства, поэтому рекомендуется протестировать эту функцию на всех платформах, на которые вы хотите ориентироваться, и вывести возвращаемые значения с помощью функции типа show_debug_message(). Обратите внимание, что карта DS не очищается автоматически из памяти, и вы должны использовать эту функцию, когда она больше не нужна. ds_map_destroy() функцию, когда она больше не нужна.

Ниже приведены некоторые подробности о дополнительных данных, которые будут доступны для карты DS для следующих платформ. Следует отметить, что на всех платформах, кроме HTML5, карта будет иметь ключ "is64bit", который будет либо true, либо false в зависимости от того, работает ли бегун в 64-битном режиме или нет.

Windows / UWP

На Windows Desktop и Windows UWP, вы получите некоторые дополнительные данные от DX11 со следующими клавишами, доступными в карте DS (полезно для расширений):

  • udid: Идентификатор, который является уникальным для машины
  • video_d3d11_device: указатель на устройство DX11
  • video_d3d11_context: указатель на контекст DX11
  • video_d3d11_spawnchain: указатель на цепочку подкачки DX11
  • video_adapter_vendorid: строка с идентификатором поставщика адаптера
  • video_adapter_deviceid: строка с идентификатором устройства адаптера
  • video_adapter_subsysid: строка с идентификатором подсистемы адаптера
  • video_adapter_revision: строка с номером ревизии адаптера
  • video_adapter_description: строка с описанием видеоадаптера
  • video_adapter_dedicatedvideomemory: строка с размером видеопамяти
  • video_adapter_dedicatedsystemmemory: строка с размером системной памяти (используется адаптером DX11)
  • video_adapter_sharedsystemmemory: строка с размером общей системной памяти

macOS

Функция возвращает некоторую ограниченную информацию о OpenGL на macOS:

  • udid: Идентификатор, который является уникальным для машины
  • gl_vendor_string: GL_VENDOR из OpenGL, пример - "Intel Inc.".
  • gl_version_string: GL_VERSION из OpenGL, пример - "2.1 INTEL-16.2.16".
  • gl_renderer_string: GL_RENDERER из OpenGL, пример - "Intel(R) Iris(TM) Graphics 6100".

Android

На платформе Android вы получите некоторые дополнительные данные из различных API со следующими ключами, доступными в карте DS:

  • android_tv: возвращается true, если устройство обнаружено как Android TV устройство, или false в противном случае.
  • GL_VERSION: версия OpenGL, о которой сообщает драйвер.
  • GL_VENDOR: поставщик OpenGL, о котором сообщает драйвер.
  • GL_RENDERER: рендерер OpenGL, о котором сообщает драйвер
  • GL_EXTENSIONS: расширения OpenGL, которые доступны по данным драйвера
  • GL_SHADING_LANGUAGE_VERSION: версия GLSL, поддерживаемая данным драйвером OpenGL
  • GL_MAX_TEXTURE_SIZE: максимальный размер текстуры, поддерживаемый данным драйвером OpenGL
  • SDK_INT: значение android.os.Build.VERSION.SDK_INT (см. примечание ниже)
  • RELEASE: значение android.os.Build.VERSION.RELEASE
  • MODEL: значение android.os.Build.MODEL
  • DEVICE: значение android.os.Build.DEVICE
  • MANUFACTURER: значение android.os.Build.MANUFACTURER
  • CPU_ABI: значение android.os.Build.CPU_ABI
  • CPU_ABI2: значение android.os.Build.CPU_ABI2
  • BOOTLOADER: значение android.os.Build.BOOTLOADER
  • BOARD: значение android.os.Build.BOARD
  • VERSION: значение os.version из метода Android System.getProperty.
  • REGION: значения user.region из метода Android System.getProperty
  • VERSION_NAME: значение versionName этого пакета - см. здесь для более подробной информации.
  • PHYSICAL_KEYBOARD: если мы считаем, что физическая клавиатура доступна, то строка "TRUE" в противном случае "FALSE"

ПРИМЕЧАНИЕ: Для получения информации о любых переменных android.os.Build см. документацию Android Developer.

iOS и tvOS

Вот ключи, возвращаемые на сайтах iOS и tvOS (подробная информация приведена в документации для разработчиков Apple):

  • name: строка, как сообщает [[UIDevice currentDevice] name]
  • systemName: строка, как сообщает [[UIDevice currentDevice] systemName]

  • totalDiskSpace: Общее дисковое пространство
  • cpuFrequency: Частота процессора
  • busFrequency: Частота шины
  • cpuCount: Количество процессоров
  • totalMemory: Общая память устройства
  • userMemory: Память пользователя
  • maxSocketBufferSize: Максимальный размер буфера сокета
  • Дополнительные ключи, содержащие информацию о графике OpenGL

PlayStation 4

Информацию о функциях, упомянутых здесь, см. в документации PS4.

Функция возвращает следующие клавиши на консоли PlayStation 4:

  • display_safe_area_ratio: Значение с плавающей точкой, возвращаемое sceSystemServiceGetDisplaySafeAreaInfo().
  • is_neo_mode: Это будет 1, если игра запущена на Neo (PS4 Pro), и 0, если она запущена на обычной PS4.
  • enter_button_assign: Целочисленное значение, возвращаемое sceSystemServiceParamGetInt(SCE_SYSTEM_SERVICE_PARAM_ID_ENTER_BUTTON_ASSIGN, &enterassign); значение 0 означает, что кнопка Circle используется как "назначение", а 1 - что вместо нее используется кнопка X.

PlayStation 5

Функция возвращает следующие клавиши на консоли PlayStation 5:

  • display_safe_area_ratio: То же, что и PS4 (см. выше).
  • display_resolution: "HD", "4K" или "неизвестно".
  • display_dynamic_range: "HDR", "SDR" или "неизвестно".
  • display_refresh_rate: "120 Гц", "60 Гц" или "Неизвестно".

Xbox One

Информация, полученная на Xbox One, в основном такая же, как и на Windows, однако ключи video_adapter_* и udid являются ключами 0 (за исключением video_adapter_description, который является пустой строкой ""). Ключ video_d3d11_swapchain также отсутствует.

Эта платформа также возвращает следующую информацию:

  • is_xbox_one_x: Это будет 1, если игра запущена на Xbox One X, и 0, если она запущена на базовом Xbox One.

Xbox Series X/S

Информация, получаемая на консолях Xbox Series X/S, в основном такая же, как и на Xbox One, однако клавиши video_d3d11_* были заменены на video_d3d12_*.

Вот некоторые ключи, которые возвращает эта платформа в дополнение к тем, что перечислены в разделе Xbox One:

  • video_d3d12_cmdqueue: Указывает на очередь команд DX12
  • video_d3d12_cmdlist: Указывает на список команд DX12
  • video_d3d12_currentrt: Указывает на текущую цель рендеринга DX12

Синтаксис:

os_get_info()

Возвращается:

DS Map ID

Пример:

if (os_type == os_android)
{
    var _info = os_get_info();
    
    if (_info[? "android_tv"])
    {
        global.android_tv = true;
    }
}

Приведенный выше код проверяет, является ли текущая ОС Android, получает информацию об ОС и считывает возвращаемую карту, чтобы проверить, запущена ли игра на телевизоре Android; в этом случае он устанавливает пользовательскую переменную global.android_tv в true.
