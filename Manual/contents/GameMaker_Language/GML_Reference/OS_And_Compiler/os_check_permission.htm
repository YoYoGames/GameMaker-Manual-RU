os_check_permission

С помощью этой функции вы можете проверить, было ли определенное разрешение предоставлено игре пользователем. Вы указываете проверяемое разрешение в виде строки, используя формат "android.permission.<permission>", поэтому для проверки разрешения RECORD_AUDIO (например) вы должны вызвать функцию

os_check_permission("android.permission.RECORD_AUDIO");

Функция вернет одну из трех констант - показанных ниже - чтобы сообщить игре о состоянии разрешения. Более подробную информацию о разрешениях приложений можно найти в документации Android.

ВАЖНО! Эта функция предназначена только для цели Android.

Синтаксис:

os_check_permission(permission)

Аргумент Тип Описание
permission Строка The permission to check (a string)

Возвращается:

Permission State Constant

Состояние разрешения Константа
Постоянная Описание
os_permission_granted Это означает, что разрешение было выдано
os_permission_denied Это означает, что разрешение не было выдано
os_permission_denied_dont_request Это указывает на то, что разрешение либо заблокировано настройками телефона, либо пользователь ранее отклонил запрос и выбрал "Не спрашивать снова".

Пример:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

Приведенный выше код проверяет тип ОС и, если это Android, выполняет проверку разрешений и, если разрешение "INTERNET" еще не было предоставлено, запрашивает его.
