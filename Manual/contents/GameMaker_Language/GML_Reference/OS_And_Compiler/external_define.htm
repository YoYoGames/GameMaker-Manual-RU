external_define

Эта функция может использоваться для определения вызова внешней функции к определенной dll (для Windows) или dylib (для Mac). Этот файл может быть как включенным файлом, так и частью расширения.

Вы указываете имя (и путь) файла, затем имя функции, которую вы хотите определить. Далее необходимо определить используемое соглашение о вызове (см. список констант ниже), а также тип ожидаемого результата (также константа, как указано ниже).

Наконец, вы должны указать количество аргументов, которые может принимать функция (от 0 до 15), и для каждого из аргументов указать его тип. Обратите внимание, что для функций с 4 и более аргументами, все они должны быть типа ty_real.

ПРИМЕЧАНИЕ: Это касается только dll или dylib, которые были добавлены как "Включенные файлы" в GameMaker IDE. Это не будет работать с файлами, добавленными в качестве расширений, поскольку для них требуется определение функций в самом пакете расширения.

Внешние константы типов вызовов
Постоянная Описание
dll_cdecl Это вызов по умолчанию в C, C++
dll_stdcall Это стандартный вызов WinAPI (толькоWindows dll ).
Внешние константы типов данных
Постоянная Описание
ty_real Аргумент вещественного числа
ty_string Строковый аргумент с нулевым окончанием

Синтаксис:

external_define(dll, name, calltype, restype, argnumb, argtype[0], argtype[1], ...argtype[10]);

Аргумент Тип Описание
dll Строка The name of the DLL file (string)
name Строка The name of the function (string)
calltype Константа типа внешнего вызова (dll_*) The calling convention used
restype Константа внешнего типа данных (ty_*) The type of the result to expect
argnumb Реально The number of arguments (0 - 10)
argtype[0 ... 10] Константа внешнего типа данных (ty_*) The types of the arguments being used

Возвращается:

External Function

Пример:

my_funcion = external_define("MyDLL.dll", "MyMin", dll_cdecl, ty_real, 2, ty_real, ty_real);

В приведенном выше примере кода будет определена внешняя функция под названием "MyMin" с двумя аргументами.
