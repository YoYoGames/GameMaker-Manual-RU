os_check_permission

С помощью этой функции вы можете запросить определенное разрешение. Вы указываете запрашиваемое разрешение в виде строки, используя формат "android.permission.<permission>", поэтому, чтобы запросить разрешение RECORD_AUDIO (например), вы должны вызвать функцию

os_request_permission("android.permission.RECORD_AUDIO");

В ответ функция запустит системное асинхронное событие, в котором встроенная async_load DS map будет содержать следующие пары ключ/значение:

  • "type": Это тип события, которое было вызвано, а значением будет строка "permission_request_result".
  • "<разрешение>": Это будет разрешение, запрашиваемое функцией, в виде строки, которая будет содержать одну из следующих констант:
Постоянная Описание
os_permission_granted Это означает, что разрешение было выдано
os_permission_denied Это означает, что разрешение не было выдано

Стоит отметить, что следующие разрешения поддерживаются GameMaker, но считаются Google "опасными" и поэтому должны быть запрошены в явном виде (обратите внимание, что некоторые разрешения могут быть запрошены с помощью Опций игры Android без необходимости использования этой функции):

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

Дополнительную информацию о разрешениях приложений можно найти в документации по Android.

ВАЖНО! Эта функция предназначена только для цели Android.

Синтаксис:

os_request_permission(permission)

Аргумент Тип Описание
permission Строка The permission to request (a string)

Возвращается:

N/A

Пример:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

Приведенный выше код проверяет тип ОС и, если это Android, выполняет проверку разрешений и, если разрешение "INTERNET" еще не было предоставлено, запрашивает его.
