ОС и компилятор

При создании кроссплатформенных игр часто бывает крайне важно получить информацию об операционной системе, на которой работает ваше устройство. Такие вещи, как язык, версия или состояние сети, могут быть использованы для адаптации вашей игры к системе, на которой она работает, и для обеспечения наилучших условий для конечного пользователя, поэтому GameMaker имеет ряд специальных функций, которые можно использовать для получения необходимой информации.

Следующие функции и переменные можно использовать для получения определенных сведений об операционной системе или браузере, в котором запущена ваша игра:

  • os_browser
  • os_device
  • os_type
  • os_version
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

Некоторые функции предварительно компилируются при запуске игры (т.е. они используются для определения того, как будет скомпилирована конечная игра), а некоторые также используются после компиляции игры для получения конкретных сведений о среде выполнения. Здесь вы можете найти полный список этих функций:

  • GM_build_date
  • GM_build_type
  • GM_version
  • GM_runtime_version
  • GM_project_filename
  • gml_release_mode
  • gml_pragma
  • parameter_count
  • parameter_string
  • environment_get_variable

На сайте Windows разрешение планировщика потоков Windows по умолчанию устанавливается на 1 мс при запуске игры. Следующие функции доступны для получения и изменения разрешения планировщика потоков Windows во время выполнения игры:

  • scheduler_resolution_get
  • scheduler_resolution_set
