uwp_check_privilege

С помощью этой функции вы можете проверить, обладает ли данный пользователь выбранной привилегией. Функция вернет true или false в системном асинхронном событии в зависимости от того, включена привилегия или нет, а если вы установите аргумент attempt_resolution в true, а привилегия не включена, она также откроет системный диалог (приостановив игру), чтобы предложить пользователю обновить учетную запись или что-либо еще, необходимое для получения привилегии. Если пользователь включит требуемую опцию, функция вернет true.

Функция требует, чтобы вы использовали один из следующих символов UWP Privilege Constantдля запроса привилегий:

UWP Privilege Constant
Постоянная Описание
uwp_privilege_internet_browsing Проверьте, разрешен ли просмотр Интернета
uwp_privilege_social_network_sharing Проверьте, разрешено ли делиться информацией в социальных сетях
uwp_privilege_share_kinect_content Проверьте, разрешено ли совместное использование контроллера Kinect
uwp_privilege_video_communications Проверьте, разрешена ли видеосвязь
uwp_privilege_communications Проверьте, разрешено ли общение через Интернет
uwp_privilege_user_created_content Проверьте, разрешен ли доступ к содержимому, созданному пользователем
uwp_privilege_multiplayer_sessions Проверьте, разрешены ли многопользовательские онлайн-сессии
uwp_privilege_sessions Проверьте, разрешены ли онлайн-сессии
uwp_privilege_fitness_upload Проверьте, разрешена ли загрузка фитнес-данных

После вызова функции на сайте XBox One она вызовет асинхронное системное событие после получения результата обратного вызова, которое будет иметь специальную карту DS async_load. Затем эта карта должна быть разобрана для следующего ключа:

  • "event_type" - должно иметь значение "check_privilege_result", если событие было вызвано этой функцией

Если тип события относится к этой функции, то при вызове функции на XBox также появятся следующие дополнительные клавиши:

  • "результат" - Будет одним или несколькими из следующих значений UWP Privilege Result Constants (комбинированные значения):
    UWP Privilege Result Constant
    Константа Описание Значение
    uwp_privilege_result_no_issue У пользователя нет проблем с привилегиями 0
    uwp_privilege_result_purchase_required Пользователь должен приобрести что-то дополнительное, обычно подписку, для получения доступа 1
    uwp_privilege_result_aborted Проверка была прервана 2
    uwp_privilege_result_banned Пользователь запрещен 4
    uwp_privilege_result_restricted Пользователю ограничен доступ, обычно с помощью родительского контроля. 8

  • "privilege" - будет одной из констант, перечисленных выше для использования в функции, чтобы вы могли проверить, какую привилегию вы запросили в случае нескольких проверок.

Если проект выполняется на платформе UWP, которая не является XBox One, то событие Async Event не будет запущено, и вместо этого функция немедленно вернет значение -1, 0 или 1:

  • -1 означает -1 пользователь не имеет привилегии
  • 0 означает, что функция не смогла запросить пользователя
  • 1 означает, что пользователь имеет привилегию.

Синтаксис:

uwp_check_privilege(user_id, privilege, attempt_resolution);

Аргумент Тип Описание
user_id Идентификатор пользователя Xbox The User ID pointer to check.
privilege UWP Privilege Constant The privilege constant to check (see description, above).
attempt_resolution Булево Enable attempting a resolution if set to true, или игнорировать, если установлено значение false.

Возвращается:

Real

Пример:

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
{
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
}

Вышеописанное проверяет, вошел ли пользователь в систему, и если вошел, то проверяет разрешение на многопользовательские сессии.
