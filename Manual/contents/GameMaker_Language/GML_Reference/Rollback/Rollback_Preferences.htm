Предпочтения отката

Возможно, вы захотите разрешить каждому игроку выбирать персонажа или настраивать его свойства. Такие свойства можно задать как "предпочтения игрока".

Как использовать предпочтения?

Сначала включите настройки, вызвав rollback_use_player_prefs() / Use Player Prefs. С помощью этой функции вы можете по желанию установить предпочтения по умолчанию для всех игроков.

Как только вы вызовете эту функцию, "автозапуск" будет отключен, то есть игра не начнется, даже если все игроки присоединились, и вам придется начать ее вручную с помощью функции rollback_start_game() / Start Game.

До начала игры вы можете установить настройки для игрока, вызвав команду rollback_set_player_prefs() / Set Player Prefs на клиенте этого игрока. Позже их можно будет прочитать через rollback_get_player_prefs() / Get Player Prefs, или переменную player_prefs в экземпляре игрока.

Событие отката

Когда игрок меняет свои предпочтения, все в игре получают событие отката rollback_player_prefs Rollback Event. Чтобы событие было отправлено, игрок должен быть полностью подключен.

Если предпочтения были установлены перед подключением, то при завершении подключения всем игрокам будут отправлены последние предпочтения, установленные перед подключением.

Начальное состояние

Если вы включили настройки, но настройки по умолчанию не установлены, то игра не начнется, пока не будут установлены настройки для всех игроков.

В этом случае вы можете либо вызвать rollback_start_game() несколько раз в тревоге, чтобы попытаться запустить игру, либо вызывать его каждый раз, когда получено событие отката rollback_player_prefs.

Какие типы я могу использовать?

Предпочтение может быть любого типа, например, число, строка, массив и т.д., но рекомендуется использовать struct, который является самым простым способом хранения коллекции переменных.
