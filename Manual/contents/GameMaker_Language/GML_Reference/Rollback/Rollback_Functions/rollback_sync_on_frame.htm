rollback_sync_on_frame

Эта функция проверяет, все ли игроки синхронизированы на текущем кадре, то есть их состояния основаны не на предсказаниях, а на фактических данных, полученных от них. В этом случае она возвращает true.

Если игроки не синхронизированы на текущем кадре, возвращается false, и предпринимается попытка синхронизировать всех игроков.

Попытка синхронизировать игроков с помощью этой функции может привести к зависанию. Вы должны убедиться, что эта функция не вызывается постоянно в событии Step.

Вместо этого используйте его в качестве переключателя. См. пример внизу страницы.

Переменная rollback_confirmed_frame обновляется автоматически при синхронизации всех игроков.

Синтаксис:

rollback_sync_on_frame();

Возвращается:

Boolean

Пример:

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

Этот код запускается в событии Step объекта игрока.

Он проверяет, нет ли выигравшего игрока, и есть ли у текущего игрока очки, необходимые для победы.

Если оба значения верны, это означает, что игрок должен победить. Однако перед этим проверяется, все ли игроки синхронизированы, чтобы убедиться, что состояние правильное.

Если все игроки синхронизированы, то устанавливается переменная winning_player. В результате первое условие становится ложным, что гарантирует, что функция rollback_sync_on_frame() не будет вызвана снова.

Это означает, что функция используется как переключение, а не работает непрерывно каждый кадр.
