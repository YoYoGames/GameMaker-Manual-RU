откат_определить_игрока

Эта функция определяет объект, который должен использоваться для игроков в многопользовательской игре. Она необязательна, и при ее использовании система будет автоматически создавать экземпляры для игроков, которые подключаются, и уничтожать экземпляры для игроков, которые отсоединяются.

По умолчанию он создает экземпляры игроков в слое "Instances", который должен существовать в вашей игровой комнате. Если вы хотите использовать другой слой, передайте его имя в качестве второго аргумента.

Информацию о создании игры с этой функцией или без нее см. в разделе Определение объекта игрока. Если эта функция используется, она должна выполняться до функции join/create.

Каждый экземпляр, созданный с помощью этой функции, получает переменные экземпляра, перечисленные на этой странице.

Синтаксис:

rollback_define_player(object, [layer_name]);

Аргумент Тип Описание
object Объект Активы The object that will be used for creating player instances.
layer_name Строка OPTIONAL Имя слоя, в котором будут созданы экземпляры игроков. По умолчанию "Instances".

Возвращается:

N/A

Пример:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

Приведенный выше код определяет объект игрока, а затем пытается присоединиться к игре. Если он не присоединился, то вместо этого начинается новая игра, в которой максимум 2 игрока и отключен Sync Test.
