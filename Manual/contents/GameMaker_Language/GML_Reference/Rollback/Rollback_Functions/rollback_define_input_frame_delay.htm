откат_определить_входной_кадр_задержка

Эта функция определяет фиксированную задержку кадров для локальных входов плеера.

По умолчанию ваша игра будет использовать адаптивную задержку ввода, которая основана на задержке между игроками. Это подходит практически для всех игр, однако некоторые специфические типы игр (например, файтинги) могут предпочесть использовать фиксированную задержку ввода вместо адаптивной, которая может меняться в любое время.

Не рекомендуется изменять это значение, если вы не знаете точно, что делаете. Вы можете вызвать эту функцию в любое время до и во время многопользовательской игры.

Указанное значение задержки должно быть в количестве кадров. Если указать -1, игра вернется к использованию адаптивной задержки. Максимальная задержка, которую можно задать, составляет 10 кадров.

Синтаксис:

rollback_define_input_frame_delay(frames);

Аргумент Тип Описание
frames Реально The number of frames to set as the fixed input delay, or -1 чтобы сбросить настройки по умолчанию.

Возвращается:

N/A

Пример:

// 'Options' script
global.frame_delay = 5;

// obj_game: Create event
rollback_define_input_frame_delay(global.frame_delay);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

Вы можете определить значение задержки кадра как глобальную переменную и использовать ее при вызове rollback_define_input_frame_delay() перед запуском игры с откатом.
