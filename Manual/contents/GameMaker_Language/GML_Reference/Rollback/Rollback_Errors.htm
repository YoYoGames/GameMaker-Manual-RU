Ошибки отката

При работе с системой отката могут возникать определенные ошибки. Эти ошибки являются ошибками Runner Errors, поэтому они отображаются во время работы игры:

В следующей таблице перечислены возможные ошибки, которые вы можете получить при работе с системой Rollback, и их описание. В ней также приведено решение для каждого конкретного сообщения об ошибке.

Ошибка Описание
Не удалось получить слой для создания игроков. Вы должны передать имя слоя в функцию rollback_define_player или создать слой Instances. Если объект игрока был определен с помощью функции rollback_define_player, система отката попытается создать объект игрока на слое, имя которого может быть передано в качестве второго параметра (или на слое с именем "Instances", если он не определен). Если слой не может быть найден по его имени, произойдет ошибка. Решением является либо явное задание имени слоя для создания игроков, либо создание слоя с именем "Instances", если имя слоя не задано явно.
При запуске многопользовательских игр необходимо управлять объектами игроков. Если объект игрока был определен с помощью функции rollback_define_player и объект игрока не помечен как управляемый, то возникает эта ошибка. Решение состоит в том, чтобы пометить объект игрока как управляемый в редакторе объектов.
Не удалось получить информацию о пользователе для player_id N. Если система отката не смогла получить от сервера информацию об игроке, выдается эта ошибка. Проверьте в консоли разработчика, не были ли заблокированы какие-либо запросы.
Не удалось вызвать rollback_chat до полного подключения к игре. Попробуйте дождаться события rollback_game_info. Функция чата недоступна, пока игрок не присоединился к комнате. Решение: Отключите чат до получения события rollback_game_info (см. События отката).
rollback_use_manual_start должен быть вызван перед rollback_create_game и rollback_join_game. Эти ошибки возникают, если вызов функции нарушает внутренние ограничения машины состояний. В сообщении об ошибке предлагается решение.
rollback_use_player_prefs должен быть вызван перед rollback_create_game и rollback_join_game.
rollback_use_player_prefs должен быть вызван до rollback_set_player_prefs.
rollback_set_player_prefs должен быть вызван до rollback_start_game.
rollback_define_player должен быть вызван до rollback_create_game и rollback_join_game.
rollback_define_input должен быть вызван перед rollback_create_game и rollback_join_game.
rollback_define_mock_input должен быть вызван перед rollback_create_game и rollback_join_game.
Не считывайте информацию об игроке до начала игры. Вызовите функцию rollback_get_info().
Не считывайте ввод игрока до начала игры. Вызов rollback_get_input().
rollback_define_input_frame_delay должен быть вызван перед rollback_create_game и rollback_join_game
rollback_define_input_frame_delay должен быть вызван перед rollback_create_game и rollback_join_game.
rollback_set_player_prefs не удалось установить предпочтения Сервер отклонил настройки игрока. Обычно это происходит в следующих случаях: 1. Один пользователь пытается сбросить пользовательские настройки, в то время как другой пользователь уже начал игру. В этом случае игра должна использовать последние успешно установленные пользовательские настройки, мы гарантируем, что каждый игрок должен установить некоторые пользовательские настройки перед началом игры. 2. Размер сериализованных префов игрока превышает максимальный размер префов игрока (в настоящее время 800 байт).
rollback_create_game, количество игроков установлено на N, но должно быть в диапазоне от 1 до 4 Функция rollback_create_game была вызвана с неверно заданным количеством игроков. Решением является исправление первого параметра функции rollback_create_game на значение в диапазоне от 1 до 4.
rollback_create_game не удалось установить предпочтения игрока по умолчанию. См. описание для "rollback_set_player_prefs failed to set the preferences".
Многопользовательский откат поддерживается только в цели operagx. Вызывается функция rollback_create_game, а цель не установлена на OperaGX. В настоящее время многопользовательская игра поддерживается только на платформе OperaGX. Решением является установка цели на OperaGX.
Не удалось создать игру (неизвестный тип сессии). Это внутренняя ошибка системы отката. Сообщите о проблеме команде GameMaker.
Не удалось найти player_id на этом экземпляре, вам нужно передать его в rollback_get_input(). Функцияrollback_get_input может принимать в качестве параметра id игрока. Если функции не было предоставлено никаких параметров, она примет поле player_id объекта игрока для получения входных данных. Если параметр player_id не задан, то произойдет ошибка. Если объект игрока был создан с помощью rollback_define_player, то у него должен быть задан player_id. Решение: 1. Явно передавать параметр player id в функцию 2. Создайте игрока с помощью функции rollback_define_player.
Info player_id N находится вне диапазона. rollback_get_info вызывается с неверным player_id (или объект игрока имеет неверное значение в поле player_id). Решение: Исправьте параметр player_id так, чтобы он находился в диапазоне от 1 до 4.
Не удалось найти экземпляр <id> объекта <id> (<имя>), на который ссылались данные отката. Внутренняя ошибка системы отката. Сообщите о проблемекоманде GameMaker.
Не знаю, как создать OBJECT_KIND=<kind>.
RefID <id> не имеет объекта (некорректный ввод?).
Неверный ReferenceKind <id> (неправильный ввод?).
Сопоставление экземпляров включено, но мы попытались десериализовать индекс вне диапазона. <id> не находится в [0, <id>].
Неверное значение RValue при десериализации.
Попытка воскресить экземпляр, который был удален!
Невозможно сериализовать RValue с kind=<id> (<имя>).
Пустое определение входа. rollback_define_input был вызван с пустым входом.
Определение ввода содержит константу <код ключа> дважды, что недопустимо. Код клавиши может быть сопоставлен только с одним входом. Проверьте, не задан ли какой-либо код клавиши более одного раза в определении входа.
Входное определение с меткой <label> не является вещественным значением или массивом вещественных значений. Все значения, описывающие входные величины, должны иметь тип Real.
Пустое определение входа макета для идентификатора игрока N. См. "Определение пустого входа".
Макетное определение содержит нереальное значение. См. "Входное определение с меткой <label> не является вещественным значением или массивом вещественных значений".
