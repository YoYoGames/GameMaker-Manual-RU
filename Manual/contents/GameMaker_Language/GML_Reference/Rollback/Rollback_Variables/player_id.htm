player_id

Когда вы используете rollback_define_player() для определения объекта игрока, система автоматически создает экземпляр этого объекта для каждого игрока. Каждый экземпляр этого объекта получает переменную player_id, в которой хранится ID этого игрока.

Этот идентификатор начинается с 0, который изначально является самим хостом. Все игроки от 1 и выше - это "коллеги", которые присоединились к игре.

Если вы не используете rollback_define_player(), то эта переменная не будет создана ни для каких объектов. В этом случае экземпляры игроков и их идентификаторы должны управляться вручную, как описано в разделе Определение объекта игрока.

Синтаксис:

player_id

Возвращается:

Real

Пример:

// Create event
if (player_id == 0)
{
    x = 300;
}
else if (player_id == 1)
{
    x = 500;
}

Приведенный выше код изменяет положение X экземпляра игрока на основе его ID.

Вы также можете использовать этот идентификатор в качестве индекса массива, например, для хранения оценок всех игроков:

// Step event
var _coin = instance_place(x, y, obj_coin);
if (_coin != noone)
{
    obj_game.scores[player_id]++;
    instance_destroy(_coin);
}

// Draw event
var _my_score = obj_game.scores[player_id];
draw_text(x, y, _my_score);

Приведенный выше код события Step увеличивает счет игрока в массиве в obj_game, когда он сталкивается с монетой. Затем экземпляр монеты уничтожается.

Приведенный выше код события Draw извлекает счет игрока из того же массива и рисует его значение.
