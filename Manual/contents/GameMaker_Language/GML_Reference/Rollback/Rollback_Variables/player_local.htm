player_local

Когда вы используете rollback_define_player() для определения объекта игрока, система автоматически создает экземпляр этого объекта для каждого игрока. Каждый экземпляр этого объекта получает переменную player_local, которая сообщает, является ли данный экземпляр игрока локальным или нет.

Если вы не используете rollback_define_player(), то эта переменная не будет создана ни для каких объектов.

Вы можете использовать эту переменную для отображения того, какой игрок является локальным. Не рекомендуется использовать эту переменную таким образом, чтобы она влияла на логику игры, которая должна быть одинаковой для локальных и удаленных игроков.

Синтаксис:

player_local;

Возвращается:

Boolean

Пример:

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

Приведенный выше код запускается в событии Draw и проверяет, является ли экземпляр игрока локальным. В этом случае над экземпляром рисуется стрелка sprite, указывающая игроку, каким экземпляром он управляет.
