Использование инспектора

Инспектор позволяет просматривать и редактировать активы, не полагаясь исключительно на рабочие пространства.

Например, обычно вы можете редактировать код события в полноэкранном режиме и часто переходить обратно в рабочую область для редактирования свойств объекта.

С помощью инспектора можно просматривать детали объекта, оставаясь в полноэкранном режиме:

Это позволяет изменять свойства объекта и удалять/изменять события при редактировании его кода в полноэкранном режиме, а также устанавливать родительские права, определения переменных и многое другое.

Хотя в данном примере рассматриваются только объекты, Инспектор может гораздо больше и позволяет редактировать множество различных видов активов (например, Sprites, шрифты, анимационные кривые и т.д.).

Элементы редактора

Помимо активов, в редакторе можно также осматривать элементы, такие как слои в комнате, дорожки в последовательности и т.д.

Инспектор позволяет применить фильтр или эффект к слою комнаты, чего нельзя сделать в самом редакторе комнат:

ПРИМЕЧАНИЕ Когда вы применяете фильтр/эффект к слою, не являющемуся FX, с помощью Инспектора, он всегда применяется как "однослойный" эффект, т.е. его эффект будет применяться только к этому конкретному слою, а не к другим слоям, расположенным ниже него.

Кнопка "Открыть редактор"

Если актив выбран, вы можете нажать "Открыть редактор" в Инспекторе, чтобы открыть редактор актива.

Например, если сделать это с объектом, откроется редактор объектов, где вы можете изменить его свойства, добавить события и т.д.

Однако при редактировании чего-либо внутри помещения, например, экземпляра или слоя, инспектор будет фокусироваться только на редакторах внутри редактора помещения.

Как показано выше, выделение экземпляра в помещении и нажатие кнопки "Открыть редактор" в инспекторе откроет редактор экземпляра.

Аналогично, если выбрать слой Path и нажать "Открыть редактор", фокус переместится на панель слоев, где можно редактировать свойства слоя, вместо того, чтобы открыть редактор активов Path.

Multi-Select

Инспектор позволяет одновременно просматривать и редактировать свойства нескольких активов и элементов (например, слоев помещений и дорожек последовательности).

Выберите нужные активы или элементы, удерживая CTRL / CMD и щелкая левой кнопкой мыши. Выбранные элементы появятся в списке "мультивыбора" в Инспекторе:

Нажмите X рядом с элементом, чтобы удалить его из выбора.

Редактирование значений

Если выбранные вами элементы имеют одинаковое значение для какого-либо свойства, это общее значение появится в этом поле.

Однако если элементы имеют разные значения для какого-либо свойства, редактор просто покажет - вместо этого значения.

Вы можете изменить любое из этих полей. При изменении значения оно будет применено ко всем выбранным элементам.

Выбор внутри списка

Вы можете выбрать любой элемент в списке мультивыбора для просмотра и редактирования его свойств без необходимости сбрасывать выбор.

Щелкните по пустой области в списке, чтобы вернуться к виду мультивыбора.

Обратите внимание, что вы не можете выбрать несколько элементов в этом списке.

Многочисленные инспекторы

Вы можете открыть несколько окон инспектора в любое время, и каждое из них может быть связано с определенным элементом на сайте IDE.

Для того чтобы при каждом щелчке на элементе все открытые инспекторы не менялись, показывая новый выбор, необходимо сначала заблокировать каждое окно инспектора по очереди.

Для этого нужно открыть инспектор, выбрать элемент IDE для проверки, а затем нажать кнопку блокировки на инспекторе, чтобы заблокировать его на элементе (показано на рисунке ниже). После блокировки инспектора можно открыть другой и повторить процесс.

Обратите внимание, что заблокированное окно инспектора не блокирует редактирование любых значений для инспектируемого элемента. Он просто фиксирует инспектор на выбранном элементе, и вы можете продолжать редактировать его в любое время, даже если исходный элемент не является основным фокусом IDE.

С помощью функции блокировки вы можете одновременно открыть несколько инспекторов, каждый из которых будет заблокирован на определенном активе, как показано в примере ниже:
