Контроль источников

Управление контролем исходных текстов (SCM) - это название метода работы с подверсиями или резервными копиями ваших проектов через репозиторий и локальный источник. По сути, решение SCM - это независимый программный пакет, который контролирует все аспекты поддержания, изменения и сравнения версий вашего проекта в процессе работы над ним. Это особенно полезно для тех, кто работает в команде и должен иметь возможность контролировать, кто что делает, и не беспокоиться о потере данных или внесении изменений, которые впоследствии придется отменить, но и отдельные пользователи могут воспользоваться этой мощной, но гибкой системой.

GameMaker использует Git в качестве SCM, который должен быть установлен и настроен отдельно, прежде чем SCM можно будет использовать в рамках GameMaker. См. документацию Git для получения более подробной информации относительно Git.

В строке менюGameMaker появится выпадающее меню Source Control, которое будет заполнено следующими опциями после активации Source Control для вашего проекта (подробности см. в разделе Настройка плагина Git выше):

Здесь мы опишем каждый из доступных вариантов (большинство из них более подробно описаны в разделах учебника выше):

  • Создать репозиторий проекта: Эта опция позволяет создать локальный репозиторий в том же каталоге, где находятся файлы проекта.
  • Клонировать репозиторий: Эта опция позволит вам клонировать хранилище из источника в новое место назначения.
  • Зафиксировать изменения: С помощью этой опции вы можете отобразить измененные файлы в вашем проекте и затем зафиксировать их в репозитории.
  • Push Changes: После выполнения фиксации или слияния эта опция используется для переноса изменений в основной репозиторий.
  • Pull Changes: С помощью этой опции вы можете обновить локальный репозиторий, извлекая измененные файлы из главного репозитория.
  • Просмотреть историю: Откроется окно истории, где можно просмотреть всю историю версий проекта и выбрать откат определенных путей файлов или целых ревизий.
  • Показать конфликты: Это откроет окно Conflicts и перечислит все конфликтующие файлы, которые могут существовать в проекте, позволяя вам разобраться с ними либо через GameMaker IDE, либо с помощью специального инструмента слияния.
