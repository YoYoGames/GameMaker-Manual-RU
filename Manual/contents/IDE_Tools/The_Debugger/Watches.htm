Часы

Идея отладчика заключается в том, чтобы позволить вам "наблюдать" за каждой деталью вашей игры. Для этого используются "окна наблюдения", которые по умолчанию открываются в окне Output, прикрепленном IDE Вы можете щелкнуть на каждой из различных вкладок, чтобы увидеть содержащиеся в них часы, некоторые из которых могут обновляться в реальном времени (если включена отладка в реальном времени на панели инструментов отладчика), в то время как другие требуют, чтобы отладчик приостановил игру.

Когда вы наблюдаете за переменными в любом из окон, они могут быть одного из нескольких типов данных. Однако из-за того, как GameMaker хранит определенные данные, отладчик не всегда может точно знать, что именно хранит переменная. Это происходит потому, что внутренние значения ID для структур данных или экземпляров являются целыми числами , и поэтому отладчик не всегда знает, является ли целочисленное значение, хранящееся в переменной, значением структуры данных или экземпляра, или это просто целочисленное значение. По этой причине вы можете щелкнуть правой кнопкой мыши на любом значении переменной и получить всплывающее окно со списком доступных типов данных. которыми может быть эта переменная:

Строки и массивы не допускают такой возможности, и стоит отметить, что даже если структура данных хранится в переменной экземпляра, ее ID номер является глобальным, поэтому если у вас есть четыре экземпляра и каждый из них создает список ds_list (например), каждый список будет иметь ID от 0 до 3, и не все они будут равны 0.

Выбрав тип данных, переменная отобразит записи в структуре, просто щелкнув по маленькому значку рядом с ее названием. Это позволит раскрыть структуру Вы также можете изменить некоторые значения для показанных переменных, просто дважды щелкнув на значении и присвоив ему новое значение.

ПРИМЕЧАНИЕ: вы не можете переназначить массивы или структуры данных таким образом, но вы можете изменить некоторые из их содержимого (вы не можете изменить значения "ключа" в картах, а также значение приоритета для очередей приоритетов).

Местные жители

В окне Locals отображаются свойства текущего шага, которые включают имя текущего события (или сценария), выполняемого в настоящее время смещение в коде виртуальной машины (VM) для текущего сценария (аббревиатура "PC" означает "счетчик программ"), экземпляр "self", выполняющий блок кода, и "другой" экземпляр в событии. Последняя часть действительна только в том случае, если событие является событием столкновения или в коде используется "with". Если экземпляр находится в столкновении, "other" покажет другой экземпляр (и его переменные) в столкновении, но если это with, то экземпляр "Self" становится тем, который выполняет код, а "Other" - это экземпляр. with В остальное время "other" будет просто показывать те же данные, что и "Self".

Обратите внимание, что экземпляры "Self" и "Other" можно развернуть, щелкнув на , чтобы увидеть список всех локальных переменных и переменных экземпляра, которые в настоящее время объявлены для просматриваемого блока кода. Если вы щелкните правой кнопкой мыши на значении, вы можете установить его тип , чтобы увидеть детали структур данных.

Watches

The whole idea of the Debugger is to permit you to "watch" every detail of your game. This is done using "watch windows" which, by default, are opened in The Output Window docked at the bottom of the IDE. You can click on each of the different tabs to see the watches that they contain, some of which can be made to update in real time (when real time debugging  is enabled from the Debugger toolbar) while others will require the debugger to have paused the game.

When you're watching variables in any of the windows, they can be one of multiple data types. For a couple of them GameMaker is able to provide more information about the variable's contents (its members or entries): instances, structs, arrays and data structures. In this case the debugger shows a small  icon next to the variable name. Clicking it will expand the structure's contents, and you can also change some of the values for the variables shown by simply double clicking  on the value and giving it a new one.

 You cannot re-assign arrays or data structures in this way, but you can change some of their contents (you cannot change the "key" values in maps, nor the priority value for priority queues).

Below you can find a list of each of the different watch windows along with an explanation of what they are for and how to use them:

Locals

The Locals window shows the current step properties, which includes the name of the current event (or script) being run, the offset in the Virtual Machine (VM) code for the current script (the acronym "PC" is for "program counter"), the "self" instance running the code block, and the "other" instance in the event. This last part is only ever valid when the event is a collision event or there is a "with" used in the code. If the instance is in collision, "other" will show the other instance (and its variables) in the collision, but if it is a with then the "Self" instance becomes that which is running the code, while "Other" is then the instance that actually contains the with. At all other times, "other" will simply show the same data as "Self".

 The "Self" and "Other" instances can be expanded by clicking on  to see a list of all local and instance variables that are currently declared for the code block being stepped through.

 

Globals

The Globals window will show a list of all the currently declared global variables and their values.
