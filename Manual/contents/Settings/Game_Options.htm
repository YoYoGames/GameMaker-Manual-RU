Параметры игры

Параметры игры разделены на два раздела: Основные параметры и Настройки платформы. Основные параметры начинаются с раздела " Общие параметры", где вы можете задать определенные параметры, которые будут влиять на игру независимо от платформы, на которой она будет работать.

Окно состоит из следующих разделов:

  • Язык сценариев по умолчанию: Выберите, должны ли события объектов и сценарии в текущем проекте использовать GML Code или GML Visual, или всегда спрашивать при создании нового сценария.
  • Game Frames Per Second: Это значение устанавливает ограничение на количество кадров в секунду, которое GameMaker должен пытаться поддерживать в процессе работы вашей игры. Это значение представляет собой количество раз, которое игровой цикл будет выполняться за одну секунду (один игровой цикл считается одним "кадром"), и по умолчанию установлено на значение 60, хотя часто используется и 30. Существует много споров о том, какая скорость комнаты является идеальной, но на самом деле ответ зависит от типа игры, которую вы делаете, размера комнаты, количества экземпляров и интенсивности кода. Например, если у вас большая комната и скорость комнаты 30, игра может выглядеть "отрывистой", так как чем больше область, на которую вы смотрите, тем легче глазу воспринимать низкую скорость комнаты, но если игра интенсивная, то установка скорости комнаты на 120 может привести к "лагам", так как fps падает из-за объема выполняемой обработки. Поэтому будьте осторожны с этим значением и старайтесь всегда регулировать его в соответствии с потребностями вашей игры.
  • Default Draw Colour: Цвет по умолчанию для рисования текста или форумов (и других вещей), когда цвет не был задан с помощью кода.
  • Default Draw Colour Вне области комнаты: Цвет по умолчанию, который будет использоваться для очистки фона любой области за пределами игровой комнаты.
  • Генерировать изображения проекта: При нажатии на эту кнопку откроется инструмент Генератор изображений проекта.
  • Дата начала проекта: Дата начала проекта.
  • Время использования проекта: Продолжительность времени, прошедшего с момента начала проекта.
  • GUID проекта: Это уникальный идентификатор, который создается для каждого проекта в GameMaker. Если вы хотите изменить его, вы можете нажать кнопку Create New GUID, чтобы создать новый идентификатор.
  • Генерировать мипмапы для отдельных страниц текстур: Эта опция заставит GameMaker создавать мипмапы для тех sprites, которые отметили опцию Seperate Texture Page в настройках текстур редактора Sprite. Включение mipmapping означает, что оригинальная страница текстуры для sprite будет масштабирована и отфильтрована в несколько разрешений в файле текстуры. Обычно эта функция выключена, но если вы используете представления для увеличения или уменьшения больших областей игровой комнаты или работаете с 3D-игрой, то вы можете включить ее и затем использовать функции Mipmapping Functions для настройки того, как игра работает с различными уровнями mipmap. Обратите внимание, что если вы не отмечаете sprites как использующую отдельные текстурные страницы, вы все равно можете включить мипмаппинг для каждой группы текстур в менеджере групп текстур. По умолчанию эта опция выключена.
  • Включить контроль исходных текстов: Если вы хотите, чтобы проект находился под SCM (Source Control Management), то вам необходимо установить этот флажок. По умолчанию он выключен (дополнительную информацию о контроле исходных текстов см. здесь).
  • Автор: Имя автора игры (или издателя, или студии).
  • Последнее изменение проекта: В этом разделе просто отображается дата/время последнего изменения проекта (не редактируется).
  • Принять лицензию Spine: Здесь вы можете установить флажок, чтобы вам была показана лицензия Spine Runtime License и принять ее. Это позволит вам использовать Spine sprites в вашей игре (эта лицензия также будет показана при первой попытке добавить Spine sprite в ваш проект, если вы предварительно не отметили лицензию здесь).
  • Режим совместимости столкновений: Это указывает GameMaker использовать устаревшую систему столкновений, применявшуюся до версий 2022 года (в которой использовались целочисленные значения вместо плавающих). Для получения дополнительной информации читайте раздел Режим совместимости столкновений. По умолчанию этот режим отключен, и рекомендуется отключать его для новых проектов.
  • Включить поведение копирования при записи для массивов: Это включает устаревшее поведение копирования при записи массивов для текущего открытого проекта. Для получения дополнительной информации читайте Копировать при записи. По умолчанию эта функция отключена, и рекомендуется не включать ее для новых проектов.

Раздел Основные параметры также содержит меню Информация о шаблоне.

В браузере активов будут доступны и другие игровые опции в зависимости от целевых платформ, доступных для имеющейся у вас лицензии, и не все из них могут быть доступны или видны.

Настройки платформы

Существуют следующие варианты игр для конкретных платформ:

  • Opera GX
  • Windows
  • macOS
  • Ubuntu
  • Windows UWP
  • HTML5
  • Android
  • iOS
  • tvOS

ПРИМЕЧАНИЕ Различные варианты консольных игр не перечислены здесь из-за различных юридических ограничений, поддерживаемых различными компаниями. Если у вас есть лицензия на консоль, то вы можете получить эту информацию в службе поддержки YoYo Games, но сначала вы должны связаться с YoYo Games, указав свой ID консоли, и запросить разрешение на просмотр различных разделов.

Некоторые проекты GameMaker могут также показывать опции игры для платформы Amazon Fire. Это устаревшие опции, доступные только для чтения, поскольку GameMaker теперь поддерживает платформу Fire напрямую, используя игровые опции Android. Если ваш проект все еще показывает Amazon Fire, вы можете получить информацию об этих опциях на следующей странице:

  • Amazon Fire (Legacy)
