Конфигурации

В связи с требованиями кроссплатформенной разработки и необходимостью того, чтобы один продукт был доступен и адаптирован для различных клиентов, устройств или платформ, GameMaker включает конфигурации для упрощения работы. Используя эту систему, вы можете быстро переключаться между различными заставками, иконками и прилагаемыми файлами и в любой момент протестировать или экспортировать только те из них, которые подходят для целевой платформы или клиента, и все это на основе одного и того же базового кода.

Чтобы иметь возможность справляться с различными требованиями клиентов и платформ, вы можете использовать редактор конфигурации, который доступен из меню Extras в браузере активов:

Все игры, созданные в GameMaker, начинаются с конфигурации "по умолчанию", которая указана в редакторе, когда вы впервые его открываете:

Для настройки конфигураций рекомендуется сначала подготовить конфигурацию "по умолчанию", включив в нее все настраиваемые активы проекта. Для этого просто пройдитесь по следующему списку опций и установите в них то, что требуется по умолчанию:

  • Параметры расширения: Установите различные поля "Копировать в..." на необходимые платформы для конфигурации. Любые "Прокси" файлы, созданные здесь, будут иметь свою "Целевую" платформу, связанную с конфигурацией.
  • Редактор спрайтов: Установите группу текстур, в которую вы хотите добавить sprites.
  • Редактор наборов плиток: Установите группу текстур, в которую будут добавлены наборы плиток.
  • Редактор шрифтов: Установите группу текстур, в которую нужно добавить шрифты.
  • Менеджер групп текстур: Здесь вы можете задать любые свойства для каждой конфигурации, включая родительскую группу, к которой принадлежит группа текстур.
  • Менеджер аудиогрупп: Здесь можно настраивать звуки, которые назначаются различным аудиогруппам...
  • Включенные файлы: Установите в поле "Копировать в..." необходимые платформы для конфигурации.

После того как вы установили конфигурацию по умолчанию, вы можете добавить новую конфигурацию в редактор конфигурации, щелкнув правой кнопкой мыши на папке конфигураций и выбрав Add Config, или нажав кнопку "Create" в нижней части окна. Новая конфигурация будет отображаться как "дочерняя" от конфигурации по умолчанию - все пользовательские конфигурации являются дочерними от конфигурации по умолчанию - и вы можете дать ей соответствующее имя.

Чтобы отредактировать конфигурацию, необходимо либо дважды щелкнуть на ней , либо воспользоваться правой кнопкой мыши , выбрав пункт меню "Set Config", который выделит конфигурацию в редакторе, после чего можно приступать к ее настройке. Для этого просто перейдите к перечисленным выше опциям и измените их на желаемые настройки для вновь созданной конфигурации. GameMaker автоматически сохранит эти значения для конфигурации, и вы сможете переключаться между конфигурациями в любое время, просто используя опцию меню RMB или дважды щелкнув на .

Дальнейшие конфигурации могут быть добавлены либо как дочерние элементы конфигурации по умолчанию, либо как дочерние элементы конфигураций, определенных пользователем, которые вы создали, в зависимости от того, какая конфигурация была выделена в редакторе при ее создании. На изображении примера ниже видно, что у нас определено три пользовательских конфигурации, две из которых являются дочерними для конфигурации по умолчанию, а еще одна - дочерняя для конфигурации, определенной пользователем:

Стоит отметить, что вы можете использовать любую заданную конфигурацию с любой заданной целевой платформой при компиляции или тестировании ваших проектов. Для этого необходимо выбрать нужную конфигурацию в разделе Config в Target Manager:

Вы также можете открыть редактор конфигурации в любое время из менеджера целей, нажав на кнопку меню в правом верхнем углу (как выделено на рисунке выше).

После того как вы зададите конфигурацию для использования таким образом, при компиляции с целью тестирования или создания конечного пакета активов будет использоваться выбранная конфигурация. Более подробную информацию об окне Цели см. в разделе Компиляция.
