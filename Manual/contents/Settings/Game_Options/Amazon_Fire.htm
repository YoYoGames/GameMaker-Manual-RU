Варианты игр для Amazon Fire (наследие)

ВАЖНО Опции Amazon Fire видны только для старых проектов и доступны только для чтения. Amazon Fire и все другие цели Android теперь настраиваются в Опциях игры для Android.

В этом разделе описаны различные доступные вам опции, которые определяют, как будут компилироваться ваши игровые проекты Amazon Fire. Различными разделами являются:

Общие сведения

Первая часть раздела General посвящена информации о продукте и настройкам сборки. Вы должны заполнить информацию о продукте, которая включает в себя отображаемое имя проекта, информацию о его пакете, которая состоит из домена, компании и продукта. Информация о пакете может состоять только из букв A-Z и цифр 0-9, без символов и других специальных знаков, а также не должна содержать заглавных букв. Эти данные будут использованы для создания идентификатора пакета в формате "com.company.myappname".

ПРИМЕЧАНИЕ: Если вы уже настроили параметры игры на цели Android, вы можете нажать кнопку "Sync from Android" в верхней части, чтобы перенести соответствующую информацию с цели Android на Amazon Fire.

Затем необходимо установить Build Tools, а также Target, Minimum и Compile SDK версии.

ПРИМЕЧАНИЕ: Существует также опция для использования версии "Support Library". Начиная с GameMaker v2.2.4 и выше, этот параметр больше не требуется, и любые значения, используемые здесь, будут игнорироваться. Этот параметр сохраняется для использования в старых проектах, которым требуется более ранняя версия исполнения, чем 2.2.4.

Инструменты сборки (и библиотека поддержки, если требуется) используются, когда GameMaker собирает вашу игру для Android/Amazon Fire и должны быть установлены на соответствующие значения, основанные на инструментах, которые вы установили с помощью Android Studio.

  • Target SDK указывает на то, что вы протестировали свое приложение на версии (вплоть до включительно), которую вы здесь указали. Это просто для того, чтобы дать ОС Android представление о том, как она должна работать с вашим приложением с точки зрения возможностей ОС. Для всех практических целей большинство приложений захотят установить Target SDK на последнюю выпущенную версию API (как используется параметром Compile SDK). Это позволит вашему приложению выглядеть как можно лучше на самых последних устройствах Android. Обратите внимание, что с ноября 2019 года Google сделал API уровня 28 обязательной целью.
  • Минимальная версия SDK - это минимальный уровень API, на котором будет работать ваш проект. С ноября 2019 года Google сделал уровень API 16 абсолютно минимально необходимым, но если вы добавите расширения, то они могут потребовать более высокий минимальный уровень API.
  • Версия SDK для компиляции - это версия API, под которую компилируется проект. Это означает, что вы можете использовать функции API Android, включенные в эту версию API. Если вы попытаетесь использовать, например, функции API 16, но установите Compile SDK 15, то получите ошибку компиляции. Однако если вы установите Compile SDK на 28, вы сможете запустить приложение на устройстве с API 26 (а также на всех предыдущих версиях).

Вы можете выбрать один из предопределенных целевых параметров, нажав на кнопку Pre-populate SDK values to chosen API level, которая откроет список различных API на выбор. При выборе любого из них все файлы SDK будут заполнены соответствующими значениями, которые вы можете использовать или редактировать по мере необходимости.

ПРИМЕЧАНИЕ: Для правильной компиляции проектов необходимо установить соответствующие API в Android SDK Manager или через Android Studio.

Вторая часть раздела "Общие сведения" охватывает различные ориентации , в которых может быть запущена ваша игра для наилучшего восприятия пользователем. Доступны следующие варианты:

  • Портрет
  • Перевернутый портрет
  • Ландшафт
  • Перевернутый пейзаж

Наконец, у вас есть следующие различные опции:

  • Включить поддержку Bluetooth/iCade: Это добавит поддержку геймпадов bluetooth или iCade в ваш игровой проект. По умолчанию эта функция включена.
  • Запустить анализ кода Lint: Когда эта опция отмечена, инструменты сборки будут запускать lint-анализ кода проекта в процессе сборки, обеспечивая более глубокую проверку ошибок и показывая больше информации в окне вывода по мере компиляции проекта (также будет создан LOG-файл, путь к которому будет показан в конце компиляции в окне вывода). Обратите внимание, что не все ошибки, отмеченные lint-анализом, обязательно являются проблемными для вашей игры, и включение этой функции может означать, что ваша игра больше не компилируется, даже если она выключена, она будет компилироваться и запускаться нормально. По сути, эта опция нужна только для отладки, и по умолчанию она выключена.
  • Место установки: Здесь вы можете установить место установки по умолчанию для вашей игры. Можно установить значение Automatic, в этом случае игра будет установлена в место, выбранное пользователем на его устройстве, или значение Prefer External, которое всегда будет пытаться установить игру на внешнее хранилище устройства.

Здесь также можно установить параметр Amazon Fire Sleep Margin. Эта опция связана с уменьшением заикания при запуске игры на определенных системах. В принципе, если ваша игра работает быстрее, чем ваша комнатная скорость, GameMaker будет "спать" оставшееся время, но этот сон может быть довольно неточным, и в итоге вы можете спать дольше, чем нужно, в результате чего ваш кадр занимает больше времени, что вызывает заикание. Чтобы обойти эту проблему, мы можем спать меньше времени, а затем зациклиться на оставшемся времени, чтобы сделать его более точным - хотя проблема с зацикливанием заключается в том, что это приводит к увеличению загрузки процессора, что, в свою очередь, может привести к повышению температуры процессора и увеличению расхода батареи. По умолчанию это значение установлено на 4, и в 99,99% случаев вас это устроит, но для устройств с низким уровнем производительности или для устройств с большим количеством запущенных фоновых процессов это может быть не идеальным решением, и может потребоваться значение от 5 до 10. Обратите внимание, что эта конфигурация зависит от конкретного устройства, и то, что работает на вашем устройстве, может не подойти другому пользователю, поэтому, если вы сомневаетесь, оставьте значение 4.

Графика

Здесь вы можете изменить следующие детали, связанные с тем, как будет отображаться ваша игра, при этом доступны следующие опции:

  • Интерполировать цвета между пикселями: Включает линейную интерполяцию, которая, по сути, "сглаживает" пиксели. Для четкой пиксельной графики ее следует выключить, но если у вас есть красивые альфа-смешения и сглаженные края графики, ее лучше оставить включенной. По умолчанию эта функция включена.
  • Глубина цвета экрана: Здесь можно установить глубину цвета для рендеринга в 16 или 24 бита. Если установить значение 24 бит, это повлияет на совместимость с некоторыми старыми устройствами, а также увеличит объем необходимой памяти ashmem (это общая память на устройствах AmazonFire).
  • Масштабирование: Здесь можно выбрать масштабирование холста рисунка с сохранением соотношения сторон экрана устройства - добавление "прокладок" по краям для соответствия экрану - или растягивание холста рисунка в соответствии с экраном устройства.

Наконец, есть возможность задать размер страницы текстуры. По умолчанию (и наиболее совместимый) размер - 2048x2048, но вы можете выбрать любой размер от 256x256 до 8192x8192. Также есть кнопка Предварительный просмотр которая сгенерирует страницы текстур для этой платформы и затем откроет окно, чтобы вы могли увидеть, как они выглядят. Это может быть очень полезно, если вы хотите увидеть, как структурированы текстурные страницы, и предотвратить создание текстурных страниц большего (или меньшего) размера, чем необходимо. Более подробную информацию о текстурных страницах можно найти здесь.

ПРИМЕЧАНИЕ: Помните, что чем больше размер страницы текстуры, тем меньше совместимость вашей игры с различными браузерами и устройствами.

Иконки

В этом разделе вы можете добавить различные изображения иконок, которые понадобятся вашему проекту для различных страниц магазина и устройств. Все изображения иконок должны быть созданы как 24-битные файлы *.png соответствующих размеров.

Стоит отметить, что в GameMaker есть инструмент Project Image Generator, который можно использовать для автоматического создания всех изображений, необходимых для различных целевых платформ, для которых компилируется ваша игра. Если вы используете этот инструмент, вам следует пересмотреть созданные изображения, чтобы убедиться, что они соответствуют вашим требованиям. Инструмент генерирует как значки, так и изображения (см. раздел ниже).

Изображения

Здесь можно задать цвет границы, который будет использоваться в качестве фона для заставки, не заполняющей весь экран. Вы также можете добавить заставки в конечный файл игры, как для ландшафтного , так и для портретного режима, которые будут отображаться во время загрузки игры на устройство. Этот экран должен быть 24-битным *.png файлом, и рекомендуется, чтобы он был такого же размера, как первая комната (или вид) вашей игры. Если вам требуется, чтобы заставка отображалась в течение определенного времени, вы также можете задать его здесь в диапазоне от 0 до 10 секунд (время по умолчанию 0 означает, что она будет отображаться только в течение загрузки актива). Вы также можете выбрать способ отображения заставки из следующих вариантов:

  • Сохранить соотношение сторон: Это позволит сохранить соотношение сторон изображения по сравнению с экраном устройства, добавляя "подложку" по краям, где это необходимо.
  • Crop To Fit: Это позволит сохранить изображение в соотношении 1:1 с экраном устройства и обрезать (или подрезать) края по мере необходимости.
  • Растянуть: Это растягивает изображение, чтобы заполнить экран устройства (и может исказить отображаемое изображение).

Наконец, вы можете установить изображение баннера, которое также является 24-битным *.png.

Разрешения

На этой вкладке вы можете изменить некоторые разрешения, которые ваша игра может запросить в манифесте Amazon Fire. GameMaker автоматически назначает разрешения по мере необходимости, исходя из функций, используемых игрой, однако вы можете обнаружить, что для вашей игры требуется разрешение, которое не было правильно назначено (например, при использовании пользовательских расширений), в этом случае отметьте нужное разрешение.

В приведенном ниже списке указаны доступные разрешения:

  • WRITE_EXTERNAL_STORAGE: Позволяет вашей игре записывать данные на внешнее хранилище.
  • READ_PHONE_STATE: Разрешить доступ к состоянию телефона только для чтения.
  • ACCESS_NETWORK_STATE: Разрешить вашей игре доступ к информации о сетях.
  • INTERNET: Разрешить вашей игре открывать сетевые сокеты.
  • BLUETOOTH: Позволяет вашей игре подключаться к сопряженным устройствам Bluetooth.
  • RECORD_AUDIO: Позволяет вашей игре записывать звук с микрофона.

ПРИМЕЧАНИЕ: Если вы не уверены, что они вам нужны, вам, вероятно, следует просто оставить их не отмеченными по умолчанию и позволить GameMaker разобраться с разрешениями для вашей игры.

В разделе Amazon Fire Permissions также есть область, где вы можете вводить разрешения в теги Android-приложений в разделе Androidmanifest.xml. Полный список доступных тегов можно найти в документации Android здесь:

  • Разработчик Android: Теги приложения

Стоит отметить, что для целевого API 28 и выше, вам больше не будет разрешено использовать внешние веб-вызовы с помощью "http" - вместо этого требуется "https" - если вы не добавите следующий тег:

android:usesCleartextTraffic = "true"
