Варианты игр для iOS

В этом разделе описаны различные доступные вам опции, которые определяют, как будут компилироваться ваши игровые проекты iOS. Различными разделами являются:

Общие сведения

Первая часть раздела "Общие" включает в себя настройки сборки для вашей игры. Для начала вы устанавливаете ориентации , в которых может быть запущена ваша игра для наилучшего восприятия пользователем. Доступны следующие варианты:

  • Портрет
  • Перевернутый портрет
  • Ландшафт
  • Перевернутый ландшафт

Затем вы можете перейти в раздел " Поддержка устройств ", где доступны следующие опции для настройки типа устройства:

  • iPhone или iPod Touch: iPhone 6 и выше или устройства iPod Touch
  • iPad: устройства iPad
  • Оба: и iPhone/iPod, и iPad

Второе, что нужно сделать, это заполнить информацию о продукте вашей игры, включая ее отображаемое имя, идентификатор пакета и информацию о версии. Обратите внимание, что для корректной работы финальной сборки игры идентификатор пакета должен быть в правильном формате обратного URL, например: com.[Company].[GameName]. Здесь вы также можете указать идентификатор команды, который вы хотите использовать для подписи финального приложения, которое GameMaker создает для вас. Заданный здесь идентификатор будет заменять идентификатор команды, который вы указали в настройках iOS.

Далее можно установить или снять флажок Отложить вывод индикатора дома. Если эта опция отмечена, то домашняя панель всегда будет отображаться при низком уровне альфа, и первый свайп пробудит панель, а второй свайп перенесет вас домой. Если опция не отмечена, то бар не будет отображаться в игре, но первый свайп по краю приведет вас домой. По умолчанию опция не отмечена.

Наконец, у вас есть возможность сгенерировать Podfile и Podfile.lock. Эти файлы необходимы, если вы добавляете CocoaPods в расширение для iOS с помощью редактора расширений. Эти файлы содержат специальные макросы, необходимые GameMaker для настройки всех перечисленных CocaoPods при создании игры. Нажатие на кнопки создаст файлы с предварительно сгенерированными макросами, а также откроет проводник файлов, если вы захотите отредактировать их вручную. Также есть кнопка, которая приведет вас к следующему руководству:

  • iOS и tvOS: использование какао-подборок

Графика

Здесь вы можете изменить следующие детали, связанные с тем, как будет отображаться ваша игра, с помощью следующих опций:

  • Интерполировать цвета между пикселями: Включает линейную интерполяцию, которая, по сути, "сглаживает" пиксели. Для четкой пиксельной графики ее следует выключить, но если у вас есть красивые альфа-смешения и графика со сглаженными краями, ее лучше оставить включенной. По умолчанию эта функция выключена.
  • Половина текстур iPad 1: iPad 1 имеет гораздо меньше текстурной памяти, чем более современные устройства, поэтому вы можете включить эту опцию, чтобы уменьшить размер текстур наполовину, если вы хотите ориентироваться на это устройство или добиться максимальной совместимости.
  • Масштабирование: Здесь можно выбрать масштабирование холста рисунка с сохранением соотношения сторон экрана устройства - добавление "прокладок" по краям для соответствия экрану - или растягивание холста рисунка в соответствии с экраном устройства.

Наконец, есть возможность задать размер страницы текстуры. По умолчанию (и наиболее совместимый) размер - 2048x2048, но вы можете выбрать любой размер от 256x256 до 8192x8192. Есть также кнопка Preview , которая генерирует страницы текстур для этой платформы и открывает окно, чтобы вы могли увидеть, как они выглядят. Это может быть очень полезно, если вы хотите увидеть, как структурированы текстурные страницы, и предотвратить создание текстурных страниц большего (или меньшего) размера, чем необходимо. Более подробную информацию о текстурных страницах можно найти здесь.

ПРИМЕЧАНИЕ: Помните, что чем больше размер страницы текстуры, тем меньше совместимость вашей игры с различными браузерами и устройствами.

Иконки

Этот раздел позволяет добавить различные иконки, необходимые для различных устройств iPhone и iPad и различных разделов iOS App Store. Эти значки должны быть созданы как 24-битные *.png изображения с размером, указанным для каждого из них.

Стоит отметить, что в GameMaker есть инструмент Project Image Generator, который можно использовать для автоматического создания всех изображений, необходимых для различных целевых платформ, для которых компилируется ваша игра. Если вы используете этот инструмент, вам следует пересмотреть созданные изображения, чтобы убедиться, что они соответствуют вашим требованиям. Инструмент генерирует как значки, так и изображения (см. раздел ниже).

ПРИМЕЧАНИЕ При импорте старого проекта может появиться следующее предупреждение:Это происходит потому, что текущая версия iOS целевой программы изменила формат необходимых значков с момента загрузки проекта. Вам придется заново создать все пиктограммы, поскольку они были заменены на общие пиктограммы GameMaker (для этого можно быстро воспользоваться упомянутым выше инструментом Project Image Generator).

Изображения

В этом разделе вы можете добавить отдельные графические изображения в качестве стартовых экранов (изображение, которое будет отображаться на короткое время во время загрузки вашего приложения) для каждой из ориентаций устройства, а также цвет границы, который будет использоваться в качестве фона для стартовых экранов, не заполняющих весь экран. Эти изображения должны быть созданы как отдельные 24-битные изображения *.png без прозрачности и соответствующего размера. Обратите внимание, что это не считается заставкой, и у Apple есть довольно строгие рекомендации по ее настройке и использованию. Дополнительную информацию см. на странице разработчика " Руководство по человеческому интерфейсу ".

Вы также можете задать способ отображения экрана запуска из следующих вариантов:

  • Сохранить соотношение сторон: Это позволит сохранить соотношение сторон изображения по сравнению с экраном устройства, добавляя "подложку" по краям, где это необходимо.
  • Crop To Fit: Это позволит сохранить изображение в соотношении 1:1 с экраном устройства и обрезать (или подрезать) края по мере необходимости.
  • Растянуть: Это растягивает изображение, чтобы заполнить экран устройства (и может исказить отображаемое изображение).
