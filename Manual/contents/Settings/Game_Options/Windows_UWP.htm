Параметры игры в Windows UWP

В этом разделе описаны различные доступные вам опции, которые определяют, как будут компилироваться ваши игровые проекты UWP (Universal Windows Platform). Различными разделами являются:

Общий

В общем разделе опций UWP можно задать информацию о продукте:

  • Отображаемое имя: Указывает дружественное имя приложения, которое будет отображаться пользователям (эта строка может быть локализована).
  • Имя пакета: Это имя созданного вами пакета, которое должно совпадать с именем, используемым для ключа разработчика, созданного вами для игры (см. раздел "Упаковка" ниже).
  • Отображаемое имя пакета: Это "короткое имя" пакета, которое в идеале должно совпадать с отображаемым именем.
  • Отображаемое имя издателя: Это имя вашей компании или человека, которого вы хотите ассоциировать как издателя игр.
  • Версия: Номер версии вашей игры.

После настройки вы можете выбрать различные ориентации , в которых может быть запущена ваша игра для наилучшего восприятия пользователем. Доступны следующие варианты:

  • Ландшафт
  • Портрет
  • Перевернутый пейзаж
  • Перевернутый портрет

По умолчанию все они будут выбраны, и вам следует снять выделение с тех, которые вам не нужны.

После установки параметров ориентации вы можете выбрать различные разрешения , которые потребуются вашему UWP-приложению, из следующих вариантов:

  • Включить разрешение на доступ в Интернет: Помечает игру как имеющую доступ к Интернету. По умолчанию эта функция выключена.
  • Включить разрешение микрофона: Помечает, что игра имеет доступ к микрофону устройства. По умолчанию эта функция выключена.
  • Включить песочницу IAP: Помечает игру как использующую "песочницу" для покупок в приложениях. По умолчанию эта функция выключена.

Семейства устройств

Здесь вы можете выбрать семейства устройств, на которые будет ориентирован экспорт Windows UWP при создании исполняемого файла. По умолчанию отмечена опция "Поддерживать универсальное семейство устройств", что означает, что приложение будет поддерживать все устройства и семейства. Чтобы ограничить приложение поддержкой только определенных семейств устройств, следует снять флажок "Универсальное" и отметить соответствующие семейства устройств. Также можно поддерживать семейство устройств Universal, но ограничить другие семейства устройств только определенными версиями платформы - в этом случае нужно оставить отмеченным семейство устройств Universal, но также отметить другое семейство устройств, для которого вы хотите поддерживать другую версию платформы, и ввести данные о версии для этого семейства в соответствующие текстовые поля. О том, как работает эта логика, читайте на этой странице и на следующей странице поддержки Microsoft:

  • Семейство целевых устройств: Примеры

В общем случае вы не захотите трогать целевую и минимальную версии платформы и оставите их значения по умолчанию, но если у вас возникли проблемы со сборкой - в частности, если вы не можете установить SDK Windows, установленный по умолчанию, - то вам нужно изменить эти значения на те, которые у вас установлены. Если вы не уверены, какие значения нужно установить, создайте новый UWP-проект в Visual Studio и посмотрите, какие значения он устанавливает.

Для получения дополнительной информации о семействах устройств см. следующий документ Microsoft:

  • Обзор семейств устройств

Графика

Параметры графики - это параметры, которые необходимо настроить, чтобы определить, как ваша игра будет использовать видеокарту вашего целевого UWP-устройства. Для изменения включены следующие параметры:

  • Интерполировать цвета между пикселями: Включает линейную интерполяцию, которая, по сути, "сглаживает" пиксели. Для четкой пиксельной графики ее следует выключить, но если вы хотите получить красивые альфа-смешения и сглаженные края графики, ее лучше оставить включенной. По умолчанию она выключена.
  • Отображать курсор: Когда эта опция включена, в вашей игре будет отображаться курсор ОС по умолчанию. Если эта опция отключена, то при наведении мыши на окно игры курсор не будет виден, если только вы не создали его в рамках проекта игры (по умолчанию он включен).
  • Start Fullscreen: Если флажок установлен, игра будет запускаться в полноэкранном режиме, в противном случае она будет запускаться в оконном режиме. По умолчанию эта опция выключена.
  • Разрешить полноэкранное переключение: Если этот параметр установлен, пользователь может переключаться из полноэкранного режима в оконный и обратно, используя стандартные сочетания клавиш Ubuntu OS. По умолчанию эта функция выключена.
  • Масштабирование: Здесь вы можете выбрать сохранение соотношения сторон (таким образом, комната 4:3 будет иметь формат буквы на экране 16:9) или полное масштабирование (растягивание изображения на весь экран).

    ВНИМАНИЕ! Выключение поверхности приложения отключит все параметры масштабирования, установленные в разделе Windows Game Options, пока она не будет снова включена. Дополнительные сведения см. в разделе Поверхность приложения.

Наконец, есть возможность задать размер страницы текстуры. По умолчанию (и наиболее совместимый) размер - 2048x2048, но вы можете выбрать любой размер от 256x256 до 8192x8192. Также есть кнопка Предварительный просмотр которая сгенерирует страницы текстур для этой платформы и затем откроет окно, чтобы вы могли посмотреть, как они выглядят. Это может быть очень полезно, если вы хотите увидеть, как структурированы текстурные страницы, и предотвратить создание текстурных страниц большего (или меньшего) размера, чем необходимо. Более подробную информацию о текстурных страницах можно найти здесь.

ПРИМЕЧАНИЕ: Имейте в виду, что чем больше размер страницы текстур, тем менее совместима ваша игра на ПК с более низкими техническими характеристиками.

Изображения

В первой части раздела "Изображения" у вас есть возможность установить различные тайлы , которые ваша игра будет использовать на различных устройствах, на которых может работать UWP. Вы можете задать следующие параметры (и все они должны быть файлами формата *.png ):

  • Маленькое изображение плитки - это изображение логотипа, которое появляется рядом с названием приложения в строке поиска и других областях пользовательского интерфейса. Его размер должен составлять 44 x 44 пикселя.
  • Изображение средней плитки должно быть размером 71x71 пиксель.
  • Следующее изображение плитки - это плитка с логотипом магазина , ее размер должен составлять 50 x 50 пикселей.
  • Широкоеизображение плитки - это изображение, которое появляется, когда плитка отображается в широком формате, и должно иметь размер 310 x 150 пикселей. Если это изображение не предоставлено, плитка может отображаться только в квадратном формате и не может принимать уведомления, основанные на широких типах шаблонов. Так как пользователь может решить, какой формат использует плитка, рекомендуется включить плитку с широким изображением. Также обратите внимание, что если предоставлено широкое изображение, плитка может изначально отображаться в широком формате.
  • Большая плитка изображения представляет собой формат большой живой плитки и должна иметь размер 310 x 310 пикселей. Обратите внимание, что это будет использоваться только на настольных ОС.

Следующий раздел предназначен для определения логотипа игры. По сути, это иконка, которую ваша игра будет использовать в экосистеме UWP, и стоит отметить, что эти изображения могут быть прозрачными .png файлами, в этом случае они будут нарисованы поверх цвета фона, указанного в этом разделе. Этот цвет фона также применяется к цвету кнопок в любых диалоговых окнах, принадлежащих приложению, и к странице описания приложения в Магазине на устройствах.

После настройки логотипа вы можете установить заставку для игры. Это экран, который будет отображаться при запуске игры, и он не может быть больше 620 x 300 пикселей, как и логотип, это может быть прозрачный файл .png, в этом случае для него будет использоваться выбранный вами цвет фона.

Стоит отметить, что в GameMaker есть инструмент Project Image Generator, который можно использовать для автоматического создания всех изображений, необходимых для различных целевых платформ, для которых компилируется ваша игра. Если вы используете этот инструмент, вам следует пересмотреть созданные изображения, чтобы убедиться, что они соответствуют вашим требованиям.

Упаковка

GameMaker предоставляет ключ разработчика по умолчанию для разработки Windows UWP, но рекомендуется сгенерировать свой собственный и указать его здесь. Для этого создайте пустой проект в Visual Studio, затем сохраните в надежном месте копию автоматически сгенерированного файла *.pfx, который будет включен в пустой проект (это и есть ключ разработчика). Затем вы должны связать этот файл *.pfx здесь.

После ссылки на файл необходимо нажать на связанную с ним кнопку Install , а затем следовать инструкциям в открывшейся командной строке. Если этого не сделать, то использование кнопок Run/Debug (или нажатие F5) приведет к неудаче, и ваша игра не сможет быть протестирована на платформе UWP (однако, если вы используете Create Executable, процесс установки сертификации будет запущен автоматически, если *.pfx еще не зарегистрирован).

ПРИМЕЧАНИЕ: Когда дело дойдет до загрузки вашего приложения в магазин, вам нужно будет получить правильный *.pfx для вашего названия. Это делается через ассоциацию вашего приложения с магазином Windows Store в Visual Studio, что заставляет Visual Studio генерировать новый *.pfx специально для приложения. Стоит также отметить, что вы можете связать любое приложение с магазином Windows Store в разделе Visual Studio, то есть вы можете связать пустой проект с приложением Windows Store и просто скопировать сгенерированный *.pfx в безопасное место, которое затем можно использовать в UWP Game Options. Вы можете найти инструкции о том, как это делается, в базе знаний YoYo Games.

В этом разделе также необходимо указать имя издателя, которое было использовано для генерации сертификата подписи. Перед вводом всегда следует указывать "CN=", например: CN=YoYoGames.

Xbox Live

Раздел Xbox Live в опциях UWP-игры - это место, где вы можете активировать свой UWP-проект для использования на консоли Xbox One. Активация проекта для Xbox позволяет выполнять следующие действия при запуске проекта на Xbox One:

  • Вход в Xbox Live из приложения
  • Использовать идентификатор пользователя Xbox Live каким-либо образом в проекте (например, показывать геймерский тег или фотографию геймера)

Однако прежде чем вы сможете выполнить эти действия, вам необходимо убедиться, что вы нажали на флажок Включить Xbox Live, а если вы являетесь участником программы Microsoft Creators Program, вам также следует отметить опцию Creators Program App. Если вы участвуете в программе Creators Program, вам нужно будет добавить идентификатор названия и идентификатор конфигурации службы (более подробную информацию о них можно получить в Microsoft).

Обратите внимание, что в настоящее время для тестирования функциональности Xbox Live в вашем проекте необходимо использовать Create Executable, а не Run, так как для вывода UWP Xbox необходимо использовать сертификат подписи Windows Store, который вы предоставите в качестве части информации о пакете (см. выше), а не временный сертификат, поставляемый с GameMaker, который будет использоваться, когда вы нажмете Run.

После включения XBox Live вы можете использовать функции XBox Live в своем проекте, но учтите, что для базовых функций вам нужно быть участником программы Creator Program, а для расширенных функций вам понадобится учетная запись ID@XBox.
