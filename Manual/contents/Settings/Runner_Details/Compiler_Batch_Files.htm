Пакетные файлы/скрипты компилятора

Компилятор GameMaker позволяет размещать Batch-файлы (Windows) или Shell-скрипты (на сайтах macOS и Linux) в каталоге вашего проекта, а также в каталоге для каждого расширения в вашем проекте.

Размещение

Файл сценария может быть помещен в корень каталога вашего проекта, рядом с файлом .yyp проекта.

Файл сценария также может быть помещен в каталог расширения, который может выглядеть следующим образом: <каталог_проекта>/extensions/<папка_расширения>/.

Версии расширения файла сценария будут выполняться первыми, а его корневая версия будет выполняться последней. Порядок выполнения расширений нельзя контролировать, поэтому разработчик расширений должен убедиться, что сценарии одного расширения не зависят от сценариев другого расширения.

Исполнение

Файлы сценариев с определенными именами, помещенные в любой из описанных выше каталогов, выполняются в различные моменты процесса компиляции:

Имя пакетного файла Windows macOS/Linux Имя сценария оболочки Описание
pre_run_step.bat pre_run_step.sh Выполняется перед началом развертывания игры
post_run_step.bat post_run_step.sh Он выполняется, когда игра подготовлена и готова к запуску; после выполнения скрипта игра запускается
pre_build_step.bat pre_build_step.sh Выполняется перед запуском компилятора активов для создания игры
post_build_step.bat post_build_step.sh Выполняется после запуска компилятора активов для создания игры
pre_package_step.bat pre_package_step.sh Этот шаг выполняется перед заключительным этапом упаковки, когда все файлы готовы, но окончательный ZIP-файл или пакет для хранения вот-вот будет создан.
post_package_step.bat post_package_step.sh Выполняется после завершения последнего этапа упаковки и подготовки окончательного ZIP-файла или упаковки для магазина.
post_textures.bat post_textures.sh Этот шаг выполняется после того, как сгенерированы текстуры для вашей игры. Вы можете использовать этот шаг для оптимизации текстур игры вручную.

Этот скрипт получает переменную окружения TexturesDir, которая хранит путь к сгенерированным текстурам.

Эти файлы сценариев поддерживаются при сборке на всех целевых платформах.

Отключение сценариев для расширения

Если вы используете несколько расширений, выполняющих скрипты, например, Steamworks и GDK, то вам нужно отключить Steamworks при экспорте в GDK, и наоборот.

Об этом см: Как отключить расширения?
