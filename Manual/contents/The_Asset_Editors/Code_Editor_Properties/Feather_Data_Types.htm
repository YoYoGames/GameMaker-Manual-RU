Типы данных пера

Feather использует типы данных для интеллектуальной проверки синтаксиса при написании кода, гарантируя, что вы не используете неправильный тип данных для переменной или параметра функции.

Он также позволяет указывать типы данных для параметров и возвращаемых значений собственных функций сценария, используя комментарии JSDoc.

Теги @param и @return JSDoc позволяют указать любой из следующих типов:

ПРИМЕЧАНИЕ Типы JSDoc не чувствительны к регистру, то есть Id.DsList и id.dslist - это одно и то же.

ПРИМЕЧАНИЕ * В колонке " Примеры спецификаторов " перечислены некоторые примеры спецификаторов, но не все. Например, в группах Id, Asset и Constant может быть больше возможных типов, которые здесь не перечислены.

Вы можете увидеть тип Any* в IDE при использовании Feather, что указывает на то, что тип идентификатора (который может быть переменной, параметром, возвращаемым значением и т.д.) не может быть определен Feather в данный момент. Когда для этого идентификатора будет добавлено больше кода, он может принять определенный тип.

Спецификаторы

Спецификатор добавляется после базового типа данных с помощью точки ., чтобы указать точный тип данных в этой группе.

Такие типы, как Id.DsList, Asset.GMObject и Constant.Color, используют спецификаторы.

Типы коллекций

Такие типы, как Array и Id.DsList, которые представляют собой структуры данных, содержащие несколько значений, могут указывать один тип данных для всего своего содержимого.

Для этого используются типы коллекций, которые добавляются к типу с помощью угловых скобок <>.

Например, массив, содержащий строки, будет Array<String>, а DS List, содержащий массив, содержащий Reals, будет Id.DsList<Array<Real>>.
