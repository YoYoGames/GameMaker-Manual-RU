Редактор сценариев

В этом разделе рассматривается редактор сценариев (также называемый текстовым редактором), в котором вы вводите весь код, необходимый для запуска вашей игры. Этот редактор используется для создания пользовательских скриптовых активов, а также для кодирования событий объектов, для добавления кода создания комнат и для многих других вещей.

Этот раздел просто описывает работу редактора сценариев, но не рассказывает о том, как создавать активы сценариев и как использовать GML Code или GML Visual для создания сценариев. Для этого, пожалуйста, обратитесь к следующим разделам:

  • Скрипты кода GML
  • Визуальные сценарии GML

При создании актива сценария вам может быть предложено выбрать между GML Visual и GML Code. Дополнительную информацию см. в разделе Язык GameMaker.

Редактор сценариев - это место, где вы пишете свой код GameMaker Language, который является встроенным языком программирования, используемым в GameMaker (подробнее см. раздел " Язык GameMaker" ). Как только вы освоите GameMaker и захотите использовать его в полной мере, рекомендуется начать изучение этого языка, так как он значительно расширяет ваши возможности при создании игр.

Актив сценария можно переименовать, щелкнув правой кнопкой мыши на нем в браузере активов и выбрав команду Rename (Переименовать), но учтите, что имя сценария должно соответствовать правилам сценариев для активов, поэтому оно может быть только буквенно-цифровым, начинаться с буквы, а единственным символом, который оно может содержать, является символ "_" под баром. Редактор сценариев в других местах не может быть переименован и будет иметь имя, специфичное для того, что редактируется, например, событие объекта.

Вкладка сценариев / событий

Редактор сценариев открывается в окне с вкладками в верхней части, что позволяет разместить несколько сценариев или событий в одном окне (хотя это поведение можно изменить в настройках, чтобы для каждого сценария или события открывалось новое окно). Вы можете щелкнуть на вкладке и затем перетащить ее, чтобы изменить порядок вкладок, или - если вы предпочитаете - вы можете вытащить ее из текущего окна и разместить в нем. можно вытащить вкладку из текущего окна и поместить ее на рабочую область, чтобы создать новое окно для этого сценария (или добавить его в другое окно), а также можно увеличить редактор сценариев до максимума, чтобы создать новую рабочую область. Если вы перетащите вкладку сценария из IDE, то будет создан новый IDE для размещения этого ресурса сценария, и его можно использовать так же, как и главное окно.

Обратите внимание, что если вы редактируете код из события объекта в редакторе сценариев и вы максимизировали редактор сценариев или разместили его в отдельном окне или рабочей области, то меню редактора , вызываемое правой кнопкой мыши, будет содержать некоторые дополнительные опции:

  • Восстановить в рабочее пространство: Это выведет редактор сценариев из максимизированного состояния/рабочего пространства и повторно привяжет его к объекту в рабочем пространстве.
  • Перейдите к объекту: Это приведет вас к рабочей области, на которой находится объект с кодом, и сфокусирует внимание на объекте.
  • Добавить /Открыть событие: Это позволяет добавить новое событие к объекту, к которому принадлежит текущий скрипт, и откроет новую вкладку кода в редакторе скриптов для добавленного события. Если выбранное событие уже содержит код, то он будет будет открыта в новой вкладке.

Информацию об остальных опциях меню правой кнопки мыши см. в разделе " Редактирование" ниже.

При выборе этого параметра функция поиска будет выделять только те строки, которые соответствуют всей введенной строке. Например, при выключенном параметре поиск по слову "random" покажет все слова, содержащие эту строку - например, irandom(), или randomise() - в то время как при включении этой функции будет показана только функция random().
Когда эта опция включена, вы говорите GameMaker проверить не только содержимое строки поиска, но и регистр. Например, если у вас есть sprite под названием "spr_Dog" и вы выполняете поиск по запросу " dog" с выключенным переключателем, то строка sprite будет выделена, однако если переключатель включен , то не будет, поскольку "Dog" больше не считается тем же самым, что и "dog".

Обратите внимание, что "следующий" термин считается следующим после текущей позиции курсора, и вы можете переходить к разным терминам, используя кнопки со стрелками в правом верхнем углу окна поиска.

Главный редактор кода

Основной редактор кода - это место, где вы будете писать весь свой код для создания актива сценария или заполнения события объекта. Фактическое редактирование кода рассматривается в разделе " Редактирование " далее на этой странице.

Информация

Информационная строка в нижней части IDE показывает номер текущей строки, на которой вы находитесь, и позицию в строке. Здесь же вы можете увидеть помощника кода, который представляет собой строку текста, показывающую функцию, которую вы редактируете в данный момент, а также с аргументами, которые она требует. По мере заполнения функции в редакторе аргументы будут подсвечиваться, чтобы показать, какой из них вы сейчас редактируете.

Обратите внимание, что любые необязательные аргументы в функциях сценария будут показаны со скобками [] вокруг них. Если вы использовали JSDoc Script Comments в активе сценария, тогда информация, которую вы предоставили, также будет показана здесь.
