Предпочтения плагина

Параметры плагинов используются для управления любыми дополнительными плагинами, которые были добавлены в GameMaker IDE. По умолчанию здесь есть один раздел, посвященный плагину Source Control для интеграции проектов GameMaker с Git.

Контроль источников (Git)

В этом разделе рассматриваются настройки, которые необходимы для работы плагина Git SCM с GameMaker.

  • Путь к исполняемому файлу Git: Это местоположение исполняемого файла Git, который GameMaker использует для всех команд Source Control. Как показано на рисунке выше, он будет расположен в папке cmd вашей установки Git.

Ниже вы можете настроить инструмент Merge Tool и Diff Tool для Source Control с этими параметрами(необязательно):

  • Слияние (расположение инструмента): здесь указывается полный путь к файлу инструмента слияния Git.
  • Слияние (опции инструмента): В этом поле можно добавить любые дополнительные команды, которые будут выполняться при использовании инструмента слияния.
  • Diff (расположение инструмента): Здесь вы указываете полный путь к файлу инструмента Git diff.
  • Diff (опции инструмента): В этом поле вы можете добавить любые дополнительные команды, которые будут выполняться при использовании инструмента diff.

Finally there are two more options that affect how Git is set up in new projects: 

  • Add skeleton .git defaults to new projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to new projects that you create. It is enabled by default.
  • Add skeleton .git defaults to imported projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to projects that you import from a .yyz file. It is enabled by default.
