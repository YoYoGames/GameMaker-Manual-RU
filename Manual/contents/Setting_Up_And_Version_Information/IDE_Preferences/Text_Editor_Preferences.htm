Предпочтения текстового редактора

Параметры текстовых редакторов используются для определения того, как выглядят и воспринимаются различные редакторы кода/скриптов. На главной странице есть следующие параметры:

  • Задержка проверки синтаксиса (мс): Время задержки (в миллисекундах) перед запуском программы проверки синтаксиса. Значение по умолчанию - 2000 мс.
  • Задержка завершения кода (мс): Время задержки (в миллисекундах).
  • Открывать скрипты в полноэкранном редакторе: Если отметить этот параметр, ресурсы сценария будут автоматически открываться в новом рабочем пространстве. Если флажок не установлен, они будут открываться в окне в текущей рабочей области. По умолчанию эта опция не отмечена.
  • Открывать шейдеры в полноэкранном редакторе: Если отметить этот параметр, ресурсы шейдеров будут автоматически открываться в новом рабочем пространстве. Если флажок не установлен, они будут открываться в окне в текущей рабочей области. По умолчанию эта опция не отмечена.
  • Открывать события объектов в полноэкранном редакторе: Если установить этот флажок, редакторы кода событий объектов будут автоматически открываться в новой рабочей области. Если флажок снят, они будут открываться в окне, прикрепленном к редактору объектных событий. По умолчанию эта опция не отмечена.
  • Открывать сценарии событий в виде вкладок в том же окне: При добавлении кода или GML Visual к событию, окно кода прикрепляется к окну события, и когда эта опция отмечена, все последующие события, открытые для редактирования, будут добавляться в то же окно в виде вкладок сверху. Если снять этот флажок, каждое событие будет открываться как собственное уникальное окно с цепочкой. Эта опция отмечена по умолчанию.
  • Открытие связанных скриптов в виде вкладок в одном окне: Когда вы имеете несколько связанных скриптов и открываете их, по умолчанию они будут сгруппированы в одном окне в виде вкладок. Если снять этот флажок, они будут открываться каждый в своем собственном окне/рабочем пространстве. Эта опция установлена по умолчанию.
  • Панорамировать текстовый редактор в поле зрения при фокусировке: Когда вы открываете текстовый редактор из события объекта, выбор этой опции автоматически панорамирует рабочую область, чтобы редактор оказался в пределах границ. По умолчанию эта опция выключена.
  • Ширина окна кода по умолчанию: установите ширину (в пикселях) окна кода по умолчанию при открытии. Значение по умолчанию - 900.
  • Высота окна кода по умолчанию: Установите высоту (в пикселях) окна кода по умолчанию при открытии. Значение по умолчанию - 600.
  • Минимальная ширина окна кода: Установите минимальную ширину (в пикселях), допустимую для окна кода. Значение по умолчанию - 300.
  • Минимальная высота окна кода: Установите минимальную высоту (в пикселях), допустимую для окна кода. Значение по умолчанию - 200.

Помимо вышеперечисленных опций есть две подкатегории, Редактор кода и Визуальный редактор, описанные ниже.

Редактор кода

Параметры цвета используются для изменения того, как редактор кода и сценариев отображает различные тексты. Каждый пункт имеет несколько подпунктов, позволяющих определить, как именно должен выглядеть каждый отдельный фрагмент текста, как показано на рисунке выше. В общем, вы можете задать цвет, шрифт, размер и стиль для каждого из различных вариантов. Далее вы также можете задать более общий стиль для таких вещей, как выделение строк, кодовых скобок и т.д..

Под цветовыми предпочтениями находятся различные параметры стиля. Эти параметры используются для настройки поведения редактора кода/скрипта, при этом доступны следующие опции:

  • Сохранять табуляцию: Если флажок установлен, то нажатия клавиш табуляции будут сохраняться в редакторе как символы табуляции - ' \t' - в то время как снятый флажок означает, что табуляции преобразуются в пробелы. По умолчанию выключено.
  • Размер табуляции: Количество пробелов, на которое при нажатии клавиши "Tab" должен быть сделан отступ от кода. По умолчанию - 4.
  • Показать пробельные символы: Отображать " ." для показа пробельных символов (см. рисунок ниже). По умолчанию выключено.
  • Показывать номера строк: Показывать ли номера строк слева внизу или нет. По умолчанию включено.
  • Показать линии табуляции: Отображение " ...." для показа пространства с вкладками. По умолчанию выключено.
  • Включить цветовое кодирование: Включите или отключите цветовое кодирование для редактора. Если код выключен, он не будет стилизован, а если включен, то будет стилизован с использованием форматирования, приведенного выше в разделе Цвет. По умолчанию включено.
  • Лимит строк документа: задает максимальное количество строк для любого данного документа редактора. Значение по умолчанию равно 0 (0 или меньше по сути является бесконечным), а любое другое значение будет ограничивать допустимое количество строк.
  • Выравнивание отступа на новых строках: Включите или отключите автоотступ для вашего кода. По умолчанию включено, что позволит сохранить все табуляции, установленные в строке, при добавлении новой строки.
  • Автоматический отступ после скобки: Включите или отключите автоотступ для кода при добавлении скобок {}. По умолчанию включено, что приводит к табуляции следующей строки кода при добавлении новой скобки { и новой строки.
  • Включить завершение кода: Включить или отключить окно завершения кода. Когда вы пишете код и эта опция включена, вы получите всплывающее окно, показывающее возможные функции, которые вы можете использовать, основываясь на текущем вводимом тексте. По умолчанию включено.
  • Автоматическое добавление функциональных скобок: Включите или отключите автоматическое добавление скобок () при использовании автозаполнения. При написании кода, когда эта функция включена, IDE будет автоматически добавлять две скобки (), которые требует функция, помещая курсор в них, если функция принимает аргументы, или помещая курсор после них, если не принимает. Эта функция работает, только если функция была выбрана из автозаполнения. По умолчанию включено.
  • Undo / Redo Stack Limit: Здесь вы можете установить предел стека Undo/Redo. Все, что меньше 0, по сути, бесконечно (ограничено доступной памятью), в то время как положительные значения ограничат стек этим числом, удаляя любое действие в хвосте стека при достижении предела. По умолчанию -1.
  • Выделить совпадения выделения: При этом будет слегка выделена любая часть кода, которая дублируется в других местах. Так, например, если вы выделите функцию, все экземпляры этой функции в текущем окне кода также будут выделены. По умолчанию включено.
  • Цветовое кодирование/завершение кода для устаревших функций: Это позволит выделить любой устаревший участок кода. По умолчанию включено.

Визуальный редактор

Параметры Комментарии управляют различными опциями цвета и форматирования для любых комментариев, которые были добавлены к вашим визуальным сценариям GML. К этим параметрам относятся:

  • Цвет: Цвет по умолчанию для текста, записываемого в окно (по умолчанию ARGB #FF53E270).
  • Имя шрифта: Имя шрифта, который необходимо использовать (доступные шрифты зависят от установленных в вашей системе).
  • Стиль: Используемый стиль шрифта, например, жирный или курсив (обратите внимание, что не во всех окнах можно использовать одинаковые стили, а также это зависит от используемого шрифта).
  • Размер: Размер шрифта для данного окна вывода.
  • Сглаживание: Здесь вы можете установить режим сглаживания для рендеринга шрифтов выходных окон. Вы можете позволить IDE принимать решения, выбрав Use Default, или принудительно включить или выключить , выбрав одну из этих опций.
