Выделение

Этот раздел содержит настройки для изменения цветов, используемых для подчеркивания части кода, чтобы отметить ошибку, предупреждение или предложение:

  • Цвет ошибок: Цвет, используемый для ошибок.
  • Цвет предупреждения: Цвет, используемый для предупреждений.
  • Цвет предложения: Цвет, используемый для предложений.

Серьезность сообщения

Этот раздел содержит все правила, которые Feather использует для проверки вашего кода, и позволяет настроить степень серьезности каждого правила.

Профиль

В этом выпадающем окне вы можете выбрать один из нескольких предустановленных профилей для правил GM*. Доступны следующие профили:

  • None: Этот профиль не будет показывать никаких сообщений об ошибках.
  • SyntaxErrors: Если вы используете этот профиль, будут показаны синтаксические ошибки, наряду с теми, что показаны в профиле None.
  • TypeErrors: Этот профиль также показывает ошибки типа в дополнение к профилю SyntaxErrors .
  • Все: Этот профиль показывает все, что может анализировать Feather.

Правила ГМ*

Вы можете установить правило как "Ошибка", "Предупреждение" или "Предложение", что влияет на то, как код, нарушающий правило, будет сообщаться вам в редакторе кода и в окне сообщений Feather.

Вы можете установить для правила значение "Игнорировать", которое будет игнорировать все случаи нарушения этого правила.

Все правила, начинающиеся с "GM1000", проверяют ваш синтаксис на возможные фатальные ошибки, а правила, начинающиеся с "GM2000", являются правилами лучшей практики, которые помогают вам предотвратить ошибки в вашей игре.

Правила именования

Этот раздел позволяет задать правила именования для активов, переменных и различных других частей GML Code.

Правило GM2017 в разделе "Серьезность сообщения" должно быть включено, чтобы правила именования вступили в силу. По умолчанию оно отключено.

Настройки в этом разделе следующие:

  • Блокирующий список идентификаторов: Это список идентификаторов, разделенных пробелами, которые будут игнорироваться для всех правил именования.

    Например, у вас есть правило использовать obj_ в качестве префикса для объектов, но вы хотите, чтобы менеджер объектов и сеть оставались как есть и не использовали префикс obj_.

    В этом случае вы напишете manager network в поле Indentifier Blocklist. Feather будет игнорировать эти объекты и любые другие идентификаторы (переменные, перечисления, параметры и т.д.) с такими же именами.
  • <идентификатор> Правило именования: Вы можете установить правила именования для каждого типа идентификаторов в списке, таких как активы, макросы, перечисления, параметры функций, переменные и т.д.

    В каждом выпадающем списке правил именования есть следующие опции:
    • Префикс: Текст, который должен появиться перед именем идентификатора, который может быть obj_ для объектов, _ для локальных переменных и т.д. в зависимости от ваших предпочтений.
    • Суффикс: Текст, который должен появиться после имени идентификатора.
    • Стиль именования: Выберите стиль именования для идентификатора. Вы можете выбрать следующие варианты:
      • Без ограничений: Для данного типа идентификаторов форматирование не применяется.
      • Верхний регистр (UpperCamelCase): MyHealth, ObjGrappleHook, MaxInventoryItemsAllowed и т.д.
      • lowerCamelCase: myHealth, objGrappleHook, maxInventoryItemsAllowed, etc.
      • ALL_UPPER: MY_HEALTH, OBJ_GRAPPLE_HOOK, MAX_INVENTORY_ITEMS_ALLOWED, etc.
      • all_lower: my_health, obj_grapple_hook, max_inventory_items_allowed, etc.
