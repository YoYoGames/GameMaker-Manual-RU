Визуальные предпочтенияGML

Сайт Визуальный GML Preferences используются для определения того, как выглядит и ощущается визуальный редактор. Различные доступные опции перечислены ниже вместе с описанием того, что каждая из них делает.

  • Расположение панели инструментовGML Visual по умолчанию: По умолчанию панель инструментов GML Visual (где хранятся все различные библиотеки действий) располагается справа от редактора, но вы можете использовать эту опцию, чтобы она располагалась по умолчанию слева.
  • Режим видимости обзора узлаGML Visual: "Обзор узла" - это краткий описательный текст, который появляется рядом с определенными действиями в рабочей области действий, например, это действие Destroy Instance:

    Вы можете включить их всегда, включить только если действие свернуто (настройка по умолчанию) или вообще отключить.
  • Автоматическая реакция на преобразование GML Visual в GameMaker Language: Когда вы конвертируетеGML Visual событие или сценарий в код GML, вы по умолчанию получите сообщение, предупреждающее о том, что вы собираетесь сделать, и спрашивающее, хотите ли вы продолжать. Установка этого параметра в значение "OK" приведет к тому, что изменение будет выполнено без показа сообщения.
  • Автоматическая реакция на удаление узла GML Visual: Когда вы удаляете GML Visual узел события, вы по умолчанию получите сообщение, предупреждающее вас о том, что вы собираетесь сделать, и спрашивающее, хотите ли вы продолжить. Установка этого параметра в "OK" просто завершит удаление без показа сообщения.
  • Максимальный размер для миниатюр действий панели инструментов GML Visual: В нижней части панели инструментов действий есть ползунок для изменения размера GML Visual значков. Этот параметр будет использоваться для определения максимально допустимого размера (по умолчанию 64px).
  • Минимальный размер для миниатюр действий панели инструментов GML Visual: В нижней части панели инструментов действий имеется ползунок для изменения размера GML Visual значков. Этот параметр будет использоваться для определения минимально допустимого размера (по умолчанию 16px).
  • Размер по умолчанию для миниатюр действий панели инструментов GML Visual: Этот параметр будет использоваться для определения размера по умолчанию для всех значков действий в панели инструментов (по умолчанию 32px).
  • Максимальное количество недавно использованных действий для отображения: В верхней части панели инструментов действий находится библиотека, в которой хранятся действия, которые вы использовали в последнее время. С помощью этой опции можно изменить максимальное количество последних действий, которые будут отображаться (по умолчанию 5).
  • Толщина цепи, соединяющей действия GML Visual: Определяет толщину графической цепочки, связывающей различные действия в GML Visual блоке кода (по умолчанию 3px).
  • Время опускания мыши для начала перетаскивания узлов (мс): Когда вы щелкаете на узле и перетаскиваете его, вы изменяете положение узла в рамках кода действия. Однако если вы на мгновение задержите мышь, а затем перетащите, вы переместите положение узла в пределах рабочей области, но не в порядке кода. Этот параметр устанавливает количество миллисекунд, которое необходимо выждать при нажатой кнопке мыши, прежде чем узел будет перемещен (не перепозиционирован). По умолчанию - 350.
  • Скрыть панель обзора узла GML Visual: Здесь вы можете выбрать, скрыть ли панель обзора редактора кода действий. По умолчанию она отключена, и ее включение уберет из редактора текстовый список цепочек действий.
