Предпочтения компиляции

Параметры компиляции касаются определенных опций, связанных с выводом результатов при компиляции вашей игры для тестирования или создания пакета, а также определенных областей процесса компиляции. Опции включают в себя:

  • Автоматически открывать окно синтаксической ошибки при ее обнаружении: Включить или отключить уведомление о синтаксических ошибках при компиляции. По умолчанию этот параметр выключен.
  • Показывать окно компиляции во время компиляции: По умолчанию окно компиляции будет показано при компиляции проектов, даже если док компиляции закрыт (он будет открыт для компиляции). Если снять флажок, это поведение будет отключено, и окно компиляции будет видно только в том случае, если оно открыто.
  • Показывать команды компилятора в выводе компиляции: Этот флажок установлен по умолчанию и заставляет компилятор показывать все соответствующие команды в окне компиляции. Если снять этот флажок, вывод станет меньше и будет содержать меньше информации.
  • Показывать подробный вывод компилятора: Установка этого флажка заставит компилятор показывать дополнительные команды, связанные с процессом компиляции. Обычно это требуется только при отладке и может быть запрошено службой поддержки, если у вас возникнут проблемы. По умолчанию он выключен (и это рекомендуемая настройка).
  • Использовать замену для путей к дискам: Если этот параметр отмечен, GameMaker будет создавать - и удалять, когда это больше не требуется, - виртуальные диски для использования в качестве замены полных путей к файлам. Эти виртуальные диски обходят все потенциальные проблемы, связанные с тем, что пути к файлам слишком длинны для ОС (особенно при компиляции для Android или использовании расширений), поэтому рекомендуется включить эту опцию. Однако если у вас возникают проблемы с созданием GameMaker таких виртуальных дисков, вы можете отключить эту опцию. По умолчанию эта опция включена.
  • Показывать пакет в проводнике по завершении сборки: Установка этого флажка заставит GameMaker открыть окно проводника ОС в том месте, где был создан окончательный исполняемый пакет. По умолчанию эта опция включена.
  • Очищать окно вывода при запуске компиляции: По умолчанию окно компилятора будет очищать все предыдущие сообщения из окна вывода компилятора в начале каждой сборки, однако снятие этого флажка сохранит предыдущие журналы компилятора. Эта опция включена по умолчанию и является рекомендуемой конфигурацией, поскольку большие журналы компилятора могут замедлить время компиляции.
  • Автоматически останавливать выполнение перед упаковкой: По умолчанию, если ваша игра уже запущена и вы пытаетесь создать исполняемый файл, вам будет показано сообщение об ошибке и потребуется остановить игру вручную перед созданием исполняемого файла. Вы можете включить эту опцию, чтобы всегда пропускать это сообщение об ошибке и автоматически останавливать игру, если она запущена, когда выбран пункт "Создать исполняемый файл". По умолчанию эта опция выключена.
  • Максимальное количество параллельных заданий компиляции: Это значение устанавливает максимальное количество ядер процессора для компиляции проекта. Значение по умолчанию равно 8, и в целом его не нужно изменять, но если вы работаете на ноутбуке или машине низкого класса, возможно, вам захочется уменьшить это значение. По умолчанию установлено значение 8.
