Предпочтения Android

Предпочтения Android разделены на различные разделы, перечисленные ниже:

В верхней части предпочтений вы можете установить максимальный размер кучи памяти JDK (по умолчанию 2 ГБ). Далее необходимо настроить различные пути , чтобы GameMaker знал, где искать различные инструменты, необходимые для создания конечного исполняемого пакета и тестирования вашей игры. Здесь есть три раздела для Android SDK, Android NDK и Java JDK. Эти пути должны быть заполнены автоматически, но если у вас возникнут проблемы, вы можете нажать кнопку справа, чтобы открыть проводник файлов и вручную выбрать местоположение каждого элемента. Подробную информацию о том, какие версии требуются для каждого из элементов SDK, можно найти здесь. Пути должны проверяться по мере их применения, и в сообщении будет показано "Found" или сообщено о каких-либо ошибках (в этом случае вам следует пересмотреть всю информацию, чтобы убедиться в ее правильности).

Ниже представлены различные варианты упаковки:

  • Всегда делать полную установку APK: Отметив этот флажок, GameMaker удалит все предыдущие версии вашей игры с вашего устройства, а также все связанные с ней файлы (например, *.ini ) с вашего тестового устройства перед установкой новой версии игры.
  • Установить на пакет: Когда эта опция отмечена, компиляция конечного исполняемого пакета *.apk также автоматически установит его на выбранное в данный момент целевое устройство.

Наконец, мы переходим к хранилищу ключей. Это файл, который будет использоваться для "подписи" всех ваших приложений Android, поэтому заполните все данные правильно и помните, что этот файл будет необходим для создания и обновления всех ваших приложений Android в будущем, а это значит, что после установки он должен быть сохранен. рекомендуется сделать скриншот используемых настроек и создать резервную копию конечного файла.. Если вы потеряете этот файл, вы не сможете обновить ни одну из существующих игр, загруженных в магазин.

Для создания нового файла keystore необходимы следующие данные:

  • Filename: Это имя файла хранилища ключей.

ПРИМЕЧАНИЕ: Это не ваше имя или название компании, а имя генерируемого файла! Если у вас уже есть файл keystore из предыдущих проектов, вы можете нажать здесь кнопку браузера файлов и направить GameMaker к ранее созданному файлу keystore (или нажать кнопку Import ). В этом случае вы должны заполнить остальную необходимую информацию (точно так же, как и при первом создании файла), но вы не должны нажимать кнопку Generate Keystore ! Это приведет к созданию нового файла keystore, который перезапишет предыдущий.

  • Общее имя: Обычно это ваше имя.
  • Пароль: Ваш пароль безопасности для файла keystore, который должен состоять не менее чем из шести символов.
  • Псевдоним: Это другое имя, которое может быть использовано для файла keystore и должно отличаться от имени, введенного выше.
  • Alias Password: Ваш псевдонимный пароль безопасности для файла keystore, который должен состоять не менее чем из шести символов.

    ПРИМЕЧАНИЕ Если версия вашего Android Studio 4.2 или выше, вы должны использовать один и тот же пароль для полей "Пароль" и "Псевдонимный пароль". Использование разных паролей и псевдонимов приведет к сбою сборки.
  • Организационное подразделение: Отдел в компании, частью которого вы являетесь.
  • Организация: Название вашей компании.
  • Местонахождение: Название города или населенного пункта, в котором вы находитесь.
  • Страна: Стандартный двухбуквенный код страны, в которой вы находитесь.

Когда эта информация будет заполнена и вас все устроит, нажмите кнопку с надписью Generate Keystore for GameMaker, чтобы создать необходимый файл keystore. Это может занять некоторое время, но после завершения работы его, как правило, не нужно будет менять снова. Обратите внимание, что вы можете нажать кнопку Import , чтобы импортировать ранее созданный файл keystore (см. раздел Filename , выше), а также получить SHA1-хэш файла keystore, нажав кнопку Generate Key Hash . При нажатии этой кнопки будут сгенерированы два специальных хэш-ключа безопасности, непосредственно связанных с вашим файлом keystore, которые требуются некоторым веб-сайтам для корректной работы (например, Facebook или Google Play для хэша Sha1). Если вам это понадобится, просто нажмите кнопку, а затем скопируйте и вставьте созданные хэш-ключи.
